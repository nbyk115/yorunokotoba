# 課金前確認モーダル — UI文言設計書

> 最終更新: 2026-05-14  
> 根拠法令: 電子消費者契約法第3条、特定商取引法第15条の3  
> 設計方針: ブランドトーン（深夜ラジオ×姉貴分「〜だよ」）を維持しつつ、法的必須事項を過不足なく表示する

---

## なぜこのモーダルが必須なのか（法令根拠）

### 特定商取引法第15条の3（令和3年改正・2022年6月施行）

定期購入（サブスクリプション）を含む通信販売において、申込みの最終確認画面に以下を**明確に表示**することが義務付けられています。

> 「販売業者は、インターネットを利用する方法により、定期購入の契約に係る商品等の申込みを受ける場合には、当該申込みの内容を表示した画面において、次に掲げる事項を**明確に表示**しなければならない」
> - 分量（提供量）
> - 販売価格（各回の課金額）
> - 支払い時期・方法
> - 契約期間（継続期間・解約条件）
> - 解約の方法

**違反した場合**: 申込者は当該申込みを取り消すことができます（同法第15条の4）。つまり、モーダルなしで課金を完了させた場合、ユーザーに申込取消権が発生し、返金リスクが生じます。

### 電子消費者契約法第3条（錯誤無効の特例）

電子契約において、事業者が「申込内容の確認画面」を提供しなかった場合、ユーザーの操作ミスによる契約について錯誤無効が主張できる可能性があります。確認モーダルは同条が求める「申込内容の確認措置」として機能します。

### 消費者庁「定期購入に係る解約妨害等に関するガイドライン」（2024年3月）

- 解約に関する情報を**分かりやすい場所・文字サイズ**で表示すること
- 解約方法を「電話のみ」等、困難な方法に限定しないこと
- 申込確認画面での定期課金の明示を「重要事項の確認」として義務的に実施すること

## モーダルUI仕様

### モーダルタイトル

```
よるのことば Premium をはじめる前に
— 大事なこと3つだけ確認してね —
```

> **設計意図**: 「〜してね」はブランドトーン。「大事なこと3つ」で重要事項の数を明示し、読み飛ばしを防止する。

---

### 重要事項3点（本文）

法令上の必須事項を、ブランドトーンを維持しながら記載します。

---

#### 1. 料金と自動課金について

```
毎月 ¥980（税込）が自動で引き落とされるよ。

はじめての課金は今すぐ。以降は毎月同じ日に
クレジットカードから自動で請求されるから覚えておいてね。
```

> **法的対応**: 特商法第15条の3「各回の課金額・支払時期・支払方法」を満たす。

---

#### 2. 解約のしかた

```
解約はマイページからいつでも1タップでできるよ。

解約した月は、次の更新日まで普通に使えるから安心して。
でも、その月の料金は返金できないから、それだけ覚えておいてね。
返金なし・日割りもなし、ってこと。
```

> **法的対応**: 特商法第15条の3「解約の方法・解約の条件」を満たす。返金不可を明示することで特商法第15条の4（取消権）のリスクを低減。

---

#### 3. デジタルコンテンツについて

```
よるのことばはデジタルサービスだよ。
課金が完了したらすぐに使えるようになるけど、
デジタルの性質上、原則として返金はできないから、
ためしてみようかな、と思ったら「同意する」を押してね。
```

> **法的対応**: 電子消費者契約法第3条の「申込内容確認」機能を果たすとともに、デジタルコンテンツの返金不可原則を明示。

---

### 利用規約・ポリシーリンク文言

```
同意するボタンを押すと、以下に同意したことになるよ。

・[利用規約](リンク)
・[プライバシーポリシー](リンク)
・[特定商取引法に基づく表記](リンク)
```

> **注記**: リンクは実際のURLに差し替えること。モーダル内でリンクをタップしても別タブで開き、モーダルは閉じないよう実装すること（target="_blank" + rel="noopener"）。

---

### ボタン文言

| ボタン | 文言 | スタイル |
|---|---|---|
| 同意・申込ボタン | `¥980/月 で始める（自動継続）` | プライマリカラー・目立つサイズ |
| キャンセルボタン | `やめる` | セカンダリ・小さめ |

> **設計意図**: 申込ボタンに金額と「自動継続」を明記することで、特商法上の「価格・継続課金」の明示要件を満たす。「やめる」はブランドトーン準拠。キャンセルボタンを小さくしても、**存在を隠す・灰色にして見えにくくする・「キャンセルできません」等の虚偽表示は景表法・特商法違反**になるため、テキストは明確に表示すること。

## ARIA属性・アクセシビリティ実装ガイダンス

### モーダルコンテナ

```html
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="checkout-modal-title"
  aria-describedby="checkout-modal-desc"
>
```

### タイトル

```html
<h2 id="checkout-modal-title">
  よるのことば Premium をはじめる前に
</h2>
```

### 本文

```html
<div id="checkout-modal-desc">
  <!-- 重要事項3点の本文 -->
</div>
```

### フォーカス管理

- モーダルが開いたとき、フォーカスをモーダル内の最初のインタラクティブ要素（例: 「やめる」ボタン）に移動させる
- Tabキーでのフォーカスをモーダル内に閉じ込める（フォーカストラップ）
- モーダルが閉じたとき、フォーカスをモーダルを開いたボタン（PremiumCTA）に戻す

### ボタン

```html
<!-- キャンセルボタン -->
<button
  type="button"
  aria-label="課金をやめる・キャンセルする"
>
  やめる
</button>

<!-- 申込ボタン -->
<button
  type="button"
  aria-label="月額980円の自動継続課金に同意して申し込む"
>
  ¥980/月 で始める（自動継続）
</button>
```

### スクリーンリーダー向け補足

```html
<!-- 視覚的に隠しつつスクリーンリーダーに読ませる補足テキスト -->
<span class="sr-only">
  このサービスは月額980円の自動継続課金です。
  解約しない限り毎月課金が継続されます。
  解約はマイページからいつでも可能です。
</span>
```

### Escキー対応

```javascript
// モーダルが開いている間、Escキーで「やめる」と同じ動作
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape' && isModalOpen) {
    closeModal(); // モーダルを閉じる（申込はしない）
  }
});
```

## 実装チェックリスト

モーダル実装時に以下を確認すること。

- [ ] モーダルは `PremiumCTA` タップ後、UnivaPay Hosted Pageへの遷移前に表示される
- [ ] 「¥980」「自動継続」「解約方法」「返金不可」の4点がモーダル内に明示されている
- [ ] 「同意する」ボタンの文言に金額と「自動継続」が含まれている
- [ ] 利用規約・プライバシーポリシー・特商法表記の3リンクが機能している（別タブ表示）
- [ ] `role="dialog"` および `aria-modal="true"` が設定されている
- [ ] フォーカストラップが実装されている（キーボード操作でモーダル外に抜けられない）
- [ ] Escキーでモーダルが閉じる（申込はされない）
- [ ] キャンセルボタン（「やめる」）が視覚的に明確に表示されている（背景と同化させない）
- [ ] モーダルはスクロールなしで3点の重要事項が全て読める設計になっている（縦長モーダルは避ける）
- [ ] モーダルの背景（オーバーレイ）クリックでモーダルが閉じる場合、「やめる」と同じ動作にする

## 反証チェック

- **Step 1（自己反証）**: 特商法第15条の3の要件として「表示が不明確」と判断されるリスクについて検証した。「大事なこと3つだけ確認してね」というブランドトーンが、消費者庁の言う「明確に表示」という要件を満たすか——法令は表示の様式を限定していないが、重要事項が「小さい文字」「薄い色」「スクロールしないと見えない」場合は不明確と判断される。本設計では3点を箇条書き・見出しで明示し、スクロールなしで全て表示されることを前提としているため、要件を満たすと判断した。ただし、フォントサイズが14px未満の場合は消費者庁ガイドライン違反のリスクがある（推奨: 本文16px以上）。

- **Step 2（構造反証）**: 特商法第15条の3の必須記載事項（分量・価格・支払時期・契約期間・解約方法）と本モーダルの対応を条文ベースで確認した。「分量」については月額サブスクリプションのためコンテンツ数量の特定が難しいが、「毎月更新・月額¥980で利用可能な全コンテンツ」という表現で代替している。電子消費者契約法第3条の「確認措置」としての要件（ユーザーが操作を確認できる機会を与えること）は、「同意する」ボタンのアクション前にモーダルを表示することで充足。クーリングオフ制度は通信販売（デジタルコンテンツ）には法定されていないため記載不要だが、「返金不可」の明示で誤解を防止している。

- **Step 3（実用反証）**: ブランドトーン（「〜だよ」「〜してね」）と法的明確性の両立について——「返金なし・日割りもなし、ってこと」という表現が消費者にとって十分に明確か。消費者庁の解釈では「重要事項の表示は消費者が理解できる平易な表現であること」が求められており、砕けた表現は消費者側に有利（理解しやすい）な方向に作用するため問題ない。ただし、「¥980が毎月引き落とされる」という表現において、初回課金も即時であることが伝わるか確認すること。実装上の注意点: `api/subscription/checkout.ts` が mock応答のみの段階では本モーダルは表示しても意味がないため、実APIへの切り替えと同時に本モーダルを有効化すること。

- **残存リスク**: モーダルで「同意する」を押したという記録（タイムスタンプ・ユーザーID）をサーバーサイドに保存しておくことを推奨する。消費者庁の調査が入った場合や、申込取消権（特商法第15条の4）を主張するユーザーへの対応時に、「同意確認を実施した」という証跡が必要になる。現行の実装ではフロントエンドのみでの確認になっているため、`api/webhooks/univapay.ts` のWebhook受信時にFirestoreへ同意記録を保存するフィールドを追加することを推奨する。
