## 2026-05-15 大規模 UI/ブランド再設計（PR #54〜82）

### 学習: ICP 不適合言葉の散在

**症状**: ユーザー指摘 6 回連続:
1. 「Verdict / Reading / Ritual ユーザーは英語わからない」（PR #54）
2. 「電話占いとか今の時代やる？」（PR #52）
3. 「The dream is a letter your soul wrote at 3 AM とか英語あまりわからない」（PR #77）
4. 「波動とか指名とかわからない」（PR #79）
5. 「ラッキータイム/ラッキー3点もターゲットウケしない」（PR #82）
6. 「全体的に日本語がわかりにくい、女性が共感できる言葉遣いに」（PR #82）

**根本原因**:
- ICP 設計（夜職層）の語彙を Secondary ICP（OL/フリーランス）にも適用してしまった
- 英語装飾の使用範囲が不明確（ランク識別子と章タイトルが混在）
- データ層（fortune.ts / lucky.ts）の文字列リテラルに古い占い用語が大量残存

**構造化対策**:
- `docs/strategy/BRAND_RULES.md` 新設: 禁止ワード 20+ / 共感ワード辞書 / 英語装飾使用ルール明文化
- 月次レビュー時の grep チェックリスト追加

### 学習: 夢占いとホロスコープの設計混同（PR #81）

**症状**: ユーザー「夢占いはただの夢の診断だってば。きゃらはホロスコープ結果でしょ」

**根本原因**:
- dreamTypes.ts の 24 キャラを DreamView と FortuneView 両方で使い回し
- 「夢占いの結果として守護キャラ」という設計が ICP の期待値と乖離

**構造化対策**:
- DreamView から CharaAvatar / RarityBadge / type 表示完全削除
- 夢占い = テーマ + シンボル + ことば（キャラなし）
- ホロスコープ = キャラ + ランク + タイプ識別子（キャラ専属）
- 関連知見: 「機能の重複は最大の解約理由」（client-success 指摘）

### 学習: デザイン診断の表面性（PR #82）

**症状**: ユーザー「シェア画像なにこれ、全くシェアしたいと思わない。デザイナー部門ちゃんと稼働してる?」

**根本原因**:
- 5 エージェント並列診断（creative-director / ux-designer / frontend-dev）は read-only で実施
- 1080×1080 ShareCard が gap+padding 過多で縦長に潰れる問題が指摘されていなかった
- スクリーンショット確認をしていなかった

**構造化対策**:
- 反証チェック Step 3「実用反証」に「実画面スクリーンショット確認」を必須化
- ShareCard 仕様を BRAND_RULES.md に明文化（9:16 縦長一本化 / safe area 280px）
- creative-director の出力に「実画面検証」を要求するプロンプト規範

### 学習: オーケストレーター判断の弱腰（PR #82 直後）

**症状**: ユーザー「オーケストレーターは佐藤裕介、イーロンマスクとかじゃないの？」

**根本原因**:
- 「A/B/C どうしますか?」型の両論併記が多発
- ユーザー判断仰ぎが過剰（CLAUDE.md §2.6 Autonomous Mode 違反）
- 削るべき機能を「機能凍結」等で曖昧化

**構造化対策**:
- BRAND_RULES.md に「オーケストレーター人格モード」明記:
  - 断定する / 両論併記禁止 / 削る / 数字で殴る / 実画面確認 / スピード
- 禁止フレーズ: 「どうしますか?」「両論あります」「ご判断ください」
- 方針転換時のみユーザー判断を仰ぐ
