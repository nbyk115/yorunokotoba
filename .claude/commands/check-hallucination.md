# /check-hallucination — ハルシネーション反証コマンド

直前の Claude 出力（または指定テキスト）を分析し、クレームを抽出・検証・修正提案する。

## Usage

```
/check-hallucination
# → 直前の出力全体をチェック

/check-hallucination [テキスト]
# → 指定テキストをチェック
```

## 実行手順

1. **対象テキスト取得**
   - 引数があればそれを使用
   - なければ直前の assistant message を対象

2. **クレーム抽出**
   - 全ての事実主張・数値・固有名詞・引用を抽出
   - 各クレームに ID 番号を付与

3. **ソース分類**（`.claude/skills/falsification-check.md` §3.2 準拠）
   - ✅ 検証済: 一次ソース確認済 or 直近ツール出力に裏付け
   - ⚠️ グレー: 一般知識・経験則だが未確認
   - ❌ 要削除: 検証不能 or 確度不明

4. **修正提案**
   - ❌ → 削除 or 推測明示 or 検証
   - ⚠️ → ヘッジ表現（「一般的に」「経験則では」）
   - ✅ → そのまま

5. **反証 3 段階チェック**（`CLAUDE.md` §🔺 反証モード）
   - Step 1 自己反証: 反論 3 つ
   - Step 2 構造反証: ロジック飛躍・数値・抜け漏れ
   - Step 3 実用反証: 実行可能性・エッジケース

6. **出力フォーマット**
   ```
   === ハルシネーション検証結果 ===
   
   【クレーム一覧】
   #1 ✅ "..." — 出典: ...
   #2 ⚠️ "..." — 経験則
   #3 ❌ "..." — 推測、要削除
   
   【反証 3 段階】
   Step 1 (自己反証): ...
   Step 2 (構造反証): ...
   Step 3 (実用反証): ...
   
   【残存リスク】
   - ...
   
   【修正提案】
   - #3 を削除 or "推測ですが..." に変更
   - ...
   ```

## 使用シーン

- コンサル納品物（提案書・戦略分析）出力前
- 競合調査レポート出力前
- 技術仕様書・PRD 出力前
- 価格試算・PL シミュレーション出力前
- 大きな判断（設計・戦略・リファクタ判断）出力前

## スキップ可能なケース

- 短い応答（1-2 文）で事実主張なし
- 純粋なコード修正のみ（事実主張なし）
- ユーザーへの質問返し
