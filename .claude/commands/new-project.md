# /new-project — 案件受付・インテークコマンド

新規案件を受け付け、適切なエージェントチームに振り分けるまでを一気通貫で行う。

## 実行フロー

### Step 1: インテーク（依頼の受付）

以下の質問を順番にユーザーに確認する。全項目を埋めてから次のステップへ。

```
【案件インテーク】

1. 依頼の一文要約:
   「___をしたい」

2. 最終的にクライアントに渡す成果物は何か？
   □ 戦略提案書
   □ LP/Webサイト
   □ 市場調査レポート
   □ コード・プロダクト実装
   □ マーケティング施策設計
   □ 海外展開計画
   □ その他（自由記述）

3. 期限:
   □ 今すぐ（当日）
   □ 今週中
   □ 今月中
   □ 期限なし

4. クライアント/依頼者:
   社名 or 担当者名:
   業種:
   規模（従業員数 or 売上概算）:

5. 予算感（任意）:
   □ ~50万
   □ 50~200万
   □ 200万~
   □ 未定

6. 既存情報・参考資料:
   （URLや既存ファイルのパスがあれば記載）
```

### Step 2: 自動ルーティング判定

インテーク情報をもとに、CLAUDE.mdのスマートルーティング判定ツリーを適用し、最適なエージェントチームを提案する。

**出力フォーマット:**
```
【ルーティング判定】
起点エージェント: [エージェント名]
連携エージェント: [エージェント名, エージェント名]
参照スキル: [スキル名 §N]
推奨フロー: [パターン番号と名称]
```

### Step 3: プロジェクトファイル生成

`.claude/projects/YYYY-MM-DD_[クライアント名]_[案件概要].md` を生成する。

テンプレート: `.claude/projects/template.md` を使用。

### Step 4: キックオフ確認

```
【確認】
以下の内容で案件を開始してよいですか？

案件名: ___
担当チーム: ___
最初のアウトプット: ___
品質ゲート: ___

[Yes → 起点エージェントを起動]
[No → インテークに戻る]
```

---

## 品質ゲート定義

各フェーズ完了時に人間の確認を求める。自動で次フェーズに進まない。

| フェーズ | 確認内容 | 確認者 |
|---|---|---|
| Phase 1完了 | 調査結果・仮説の妥当性確認 | クライアント or PM |
| Phase 2完了 | 戦略方針・Go/No-Go判定 | 意思決定者 |
| Phase 3完了 | 成果物ドラフトのレビュー | クライアント |
| Phase 4完了 | 最終納品物の承認 | クライアント |

**品質ゲート通過の条件:**
- 反証チェック結果が添付されている
- 数値・根拠が明示されている
- 次フェーズへのハンドオフ情報が整っている

---

## 使い方

```
/new-project
```

または案件概要を添えて:

```
/new-project SaaS企業のLP改善、CVR3倍を目標に
```
