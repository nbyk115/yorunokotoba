# /peer-review — クロスエージェントレビュー

別部門のエージェントが成果物を交差レビューする。
同じ部門の人間だけでは気づかない盲点を潰し、品質を掛け算にする。

---

## 使い方

```
/peer-review [file=レビュー対象ファイル] [reviewer=レビュアーエージェント]
```

reviewerを省略した場合、対象ファイルの種別から自動選定する。

---

## レビュアー自動選定ルール

| 対象成果物の種別 | 自動選定レビュアー | レビュー観点 |
|---|---|---|
| 戦略提案書・市場分析 | `kpi-analytics` | 数値の妥当性・PL根拠 |
| 戦略提案書・市場分析 | `legal-compliance-checker` | リスク・規制見落とし |
| コード・実装 | `tech-lead` | アーキテクチャ・セキュリティ |
| コード・実装 | `fullstack-dev` | バグ・エッジケース |
| LP・クリエイティブ | `brand-guardian` | ブランド整合 |
| LP・クリエイティブ | `growth-hacker` | CVR・ファネル |
| マーケ施策 | `kpi-analytics` | KPI設計・計測漏れ |
| 海外展開計画 | `legal-compliance-checker` | 現地規制リスク |
| 海外展開計画 | `kpi-analytics` | グローバルPL妥当性 |
| 提案書・スライド | `brand-guardian` | トーン・品質 |

## クロスレビューの実行フロー

### Step 1: レビュアーによる事前読み込み

レビュアーエージェントとして振る舞い、対象ファイルを読む。

### Step 2: 3観点でのレビュー出力

```
【クロスレビュー結果】
レビュアー: [エージェント名]
対象: [ファイルパス]
実施日: YYYY-MM-DD

✅ 問題なし（維持すべき強み）:
1.
2.

⚠️  要改善（具体的な修正指示）:
1. [問題] → [修正案]
2. [問題] → [修正案]

🔴 重大な問題（納品前に必ず修正）:
1. [問題] → [修正案]

総合判定: PASS / CONDITIONAL PASS / REJECT
条件付きPassの場合の条件:
```

### Step 3: 元エージェントへの差し戻し（問題あり時）

`/handoff to=[元エージェント]` で修正依頼を生成:
- ハンドオフファイルに「クロスレビュー結果」を添付
- 重大な問題はREJECTとして差し戻し

### Step 4: 修正後の最終承認

元エージェントが修正完了 → レビュアーが最終確認 → PASSで次フェーズへ

## 部門間クロスレビューの推奨組み合わせ

### 「数値の壁」— 全戦略アウトプットに適用
```
戦略系エージェント（strategy-lead / competitive-analyst）のアウトプット
  → kpi-analytics がレビュー
  観点: 「このPL試算の前提は何か」「数字の根拠は何か」「外れた場合どうなるか」
```

### 「法務の壁」— 新規事業・海外展開に適用
```
戦略・GTMアウトプット
  → legal-compliance-checker がレビュー
  観点: 「この施策に法的リスクはないか」「規制見落としはないか」
```

### 「ブランドの壁」— 全クリエイティブ・コンテンツに適用
```
creative系アウトプット（LP / コンテンツ / キャンペーン）
  → brand-guardian がレビュー
  観点: 「ブランドガイドライン準拠か」「禁止表現は含まれていないか」
```

### 「CVRの壁」— LP・コンバージョン系に適用
```
LP / Webデザイン
  → growth-hacker がレビュー
  観点: 「ファーストビューで離脱しないか」「CTAは機能するか」「A/Bテスト案はあるか」
```

### 「技術の壁」— 全コード実装に適用
```
fullstack-dev / frontend-dev のコード
  → tech-lead がレビュー
  観点: 「アーキテクチャの問題はないか」「セキュリティホールはないか」「スケールするか」
```

## 使用例

```
# 戦略提案書のクロスレビュー
/peer-review file=.claude/projects/2026-03-25_acme_strategy.md reviewer=kpi-analytics

# LP実装のクロスレビュー（レビュアー自動選定）
/peer-review file=src/app/lp/page.tsx

# 全プロジェクトファイルのバッチレビュー
/peer-review file=.claude/projects/ reviewer=brand-guardian
```
