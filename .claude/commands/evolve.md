# /evolve — スキル進化サイクル実行

引数: $ARGUMENTS（対象スキル名 or エージェント名 or "all"）

以下のスキル進化サイクルを実行してください:

## 実行手順

### Phase 1: 診断（現状把握）
1. `.claude/memory/` から直近の評価カード・evolution-logを収集
2. 対象スキル/エージェントの品質スコア推移を集計
3. スコアC判定以下のタスクを抽出し、パターン分類

### Phase 2: 原因分析
1. `.claude/skills/agent-evaluation.md` のフィードバックループエンジンに従い分析
2. `.claude/skills/debug-methodology.md` の反証ベースで原因仮説を3つ立てる
3. 各仮説を検証し、根本原因を特定

### Phase 3: 改善案生成
1. `.claude/skills/skill-evolution.md` の自動進化パターンライブラリを参照
2. 改善案を具体的なスキルファイル修正として記述
3. 変更Levelを判定（L1: 即時 / L2: A/Bテスト / L3: 承認要）

### Phase 4: 実行
- **L1の場合**: スキルファイルを直接修正 → バージョン履歴を更新
- **L2の場合**: A/Bテスト計画を作成 → テスト実行指示を出力
- **L3の場合**: strategy-leadへの提案書を作成

### Phase 5: 記録
1. `evolution-log.jsonl` に1行追記（既存行の変更禁止）。JSON形式で `{id, date, target, type, level, title, changes, rationale, expected_effect, result}` を記録
2. 修正したスキルのバージョン履歴を更新
3. 次回測定タイミングを設定

## 出力フォーマット

```
# 進化レポート: [対象名]

## 診断結果
- 期間: YYYY-MM-DD 〜 YYYY-MM-DD
- タスク数: XX件
- 平均スコア: XX.X/25
- トレンド: [↑/→/↓]

## 発見された課題
1. [課題1]: [データ根拠]
2. [課題2]: [データ根拠]

## 根本原因
[特定された原因]

## 改善アクション
| # | 変更内容 | 対象ファイル | Level | 期待効果 |
|---|---|---|---|---|
| 1 | [内容] | [パス] | L[X] | スコア+X.X |

## 実行結果
[実行した変更の詳細]

## 次のステップ
[効果測定のタイミング・方法]
```

## 引数なしの場合
全スキル・全エージェントの概要診断を実行し、最も改善インパクトが大きいものから順に3件のレポートを出力する。
