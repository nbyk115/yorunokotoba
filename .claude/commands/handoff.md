# /handoff — エージェント間ハンドオフ生成コマンド

エージェントが自分のフェーズを完了した時に呼び出す。
アウトプットを構造化し、次のエージェントが自律的に受け取れるファイルを生成する。

---

## 使い方

```
/handoff [to=エージェント名] [project=プロジェクトファイル名]
```

例:
```
/handoff to=proposal-writer project=2026-03-25_acme_saas-strategy
```

---

## 実行フロー

### Step 1: 送り出しチェック（自動）

以下を確認。ひとつでも未完了なら作業を中断してユーザーに報告:

```
□ インプットデータは具体的か（数値・ソース付き）
□ 期待アウトプットの形式・分量・期限が明確か
□ 参照スキル（§番号）を名指しで指定したか
□ 成功基準（チェックボックス）を定義したか
□ 反証チェック（Step 1〜3）を実施済みか
□ PLインパクトが関係する場合、数値で記載したか
□ 制約（禁止手法・禁止表現）を明示したか
```

### Step 2: ハンドオフファイル生成

`.claude/handoffs/SCHEMA.md` のフォーマットに従い、以下のパスにファイルを作成:

```
.claude/handoffs/active/[YYYYMMDD-HHMM]_[from]_[to]_[task-slug].md
```

`[YYYYMMDD-HHMM]` は実行時刻、`[task-slug]` はタスクの英語省略形（3〜5語）。

### Step 3: プロジェクトファイル更新

指定された `.claude/projects/[案件].md` の「ハンドオフログ」セクションに記録を追記:

```markdown
### [from] → [to] [日時]
タスク: [1文]
ファイル: .claude/handoffs/active/[ファイル名].md
```

### Step 4: 完了通知

```
✅ ハンドオフ生成完了
  From: [送り出しエージェント]
  To:   [受け取りエージェント]
  タスク: [1文]
  ファイル: .claude/handoffs/active/[ファイル名].md

次のアクション: [受け取りエージェント名] を起動してください。
  プロンプト例: 「[エージェント名]として、.claude/handoffs/active/[ファイル名].md を読んで作業を開始してください」
```

---

## 受け取り側エージェントの起動方法

ハンドオフファイルが生成されたら、次のようにエージェントを起動:

```
[エージェント名]として動いてください。
ハンドオフファイル: .claude/handoffs/active/[ファイル名].md
まずファイルを読み、受け取りチェックを実施してから作業を開始してください。
```

受け取りチェック（To側が着手前に確認）:
```
□ インプットは十分か（不足なら即差し戻し）
□ 成功基準を理解したか
□ 参照スキルを読んだか
□ 自分の担当範囲と次のハンドオフ先を把握したか
□ 前エージェントの反証チェック結果を確認したか
```

---

## ハンドオフ完了時の処理

作業完了後:

```bash
# activeからcompletedに移動
mv .claude/handoffs/active/[ファイル名].md .claude/handoffs/completed/

# プロジェクトファイルのステータスを更新
# → ハンドオフログの該当行に「✅ 完了」を追記
```

そして次のフェーズがある場合は `/handoff` を再度実行。
