# OKR テンプレート — Objectives & Key Results

> **作成者**: strategy-lead / product-manager / kpi-analytics
> **レビュー**: orchestrator（全社OKR）/ 各部門エージェント
> **用途**: 四半期・年次のOKR設計・レビュー・振り返りに使用

---

## OKR設計原則

```
良いOKRの条件:
✅ Objective: 野心的・感情に訴える・1文で言い切れる
✅ Key Result: 数値で測定可能・2値（達成/未達成）で判定できる
✅ KRは3〜5個（多すぎると焦点が散漫になる）
✅ 60〜70%達成で「良いOKR」（100%達成は目標が低すぎる）

悪いOKRの典型:
❌ 「事業を成長させる」（Objective が曖昧すぎる）
❌ 「ユーザーを増やす」（数値がない）
❌ 「品質を向上させる」（測定できない）
❌ KRが10個以上（全部できなくなる）
```

## OKRスコアリング

### 達成率の解釈
| 達成率 | 解釈 | アクション |
|---|---|---|
| 70〜100% | 良好（目標適切） | 継続・次期に引き上げ |
| 40〜69% | 要改善 | 原因分析・対策を立案 |
| 0〜39% | 問題あり | 根本原因特定・OKR自体の見直し |
| 100%超 | 目標が低すぎた | 次期は目標を引き上げ |

### 四半期OKRレビュー（月次チェックイン）
```
月次チェックインで確認する3点:
1. 進捗（数値）: 現在の達成率と予測着地点
2. 学び: この1ヶ月で仮説が更新されたことは何か
3. ブロッカー: OKR達成を妨げているものは何か（誰が解決するか）
```

## OKR設計ワークショップ手順

### Step 1: 前四半期振り返り（30分）
```
□ 前期OKRの最終達成率を確認
□ 「やり切れた KR」と「やり切れなかった KR」を分類
□ やり切れなかった理由を3つ挙げる（言い訳なし）
□ 次期に持ち越すべきテーマを確認
```

### Step 2: 優先課題の特定（30分）
```
□ 事業戦略から「今期最重要の課題」は何か → strategy-lead に確認
□ 顧客・市場から来るインサイト → competitive-analyst / market-researcher
□ プロダクト改善課題 → product-manager / feedback-synthesizer
□ 最大PLインパクトを持つ施策は何か → kpi-analytics に試算依頼
```

### Step 3: Objective の草案（15分）
```
条件チェック:
□ 読んだ人が「やりたい」と思うか（感情的な訴求力）
□ 1文で言い切れるか
□ 今期中に意味のある進捗が出るか
□ 戦略と整合しているか
```

### Step 4: Key Result の設計（45分）
```
各KRに問うべき質問:
□ この数値が達成されれば、Objectiveの達成を確信できるか？
□ 数値は操作されにくいか（不正な達成がしにくいか）？
□ 計測方法は明確で合意されているか？
□ 開始値（ベースライン）は計測済みか？

KR設計のアンチパターン:
❌ アクション目標（「XX施策を実施する」→ KRはアウトカムで書く）
❌ バイナリ目標（「達成/未達成」のみ→ 進捗が見えない）
❌ 複合目標（「AとBを達成する」→ 分割する）
```

### Step 5: 整合チェック（15分）
```
□ 全社OKR → 部門OKR → 個人OKR の紐づきが明確か
□ 部門間で依存しているKRの担当者間で合意されているか
□ リソース（人・予算・ツール）が確保されているか
```
