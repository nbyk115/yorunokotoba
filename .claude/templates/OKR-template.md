# OKR テンプレート — Objectives & Key Results

> **作成者**: strategy-lead / product-manager / kpi-analytics
> **必須レビュー**: orchestrator（全社OKR整合）/ kpi-analytics（PLインパクト接続）
> **用途**: 四半期・年次のOKR設計・月次チェックイン・四半期振り返り

---

## ⚠️ OKR導入前に読む：批判的視点

```
OKRは万能ではない。以下を理解した上で使う。

【OKRが機能しやすい環境】
✅ 探索フェーズ（PMF前後・新規事業）
✅ 知識労働中心のチーム（エンジニア・PM・デザイン）
✅ 「何をやるか」が曖昧で優先順位が必要な組織

【OKRが機能しにくい環境】
❌ 売上目標が明確なセールスチーム → KPIで管理する方が合理的
❌ オペレーション業務（SLA・品質・再現性が命）→ チェックリスト+KPIの方が適切
❌ 目標が週単位で変わる初期スタートアップ → 季節感がなく形骸化する

【佐藤裕介の視点】
「OKRは目標管理ツールではなく、優先順位の合意ツール。
 PLに落ちないOKRは趣味。KRを達成すると粗利がいくら増えるかを
 必ず計算してから設定する。OKRで管理できるのは行動であって、
 市場は管理できない。」

→ 本テンプレートはPLインパクトとOKRを強制接続する設計にしている。
```

## OKR設計原則

### 良いOKRの条件
```
Objective（O）:
  ✅ 読んだ人が「やりたい」と思う感情的な引力がある
  ✅ 1文で言い切れる（副文節なし）
  ✅ 今四半期中に意味のある進捗が出る
  ✅ 事業戦略に直結している

Key Result（KR）:
  ✅ 数値で測定可能（「改善する」ではなく「XX%改善」）
  ✅ 達成/未達成が誰が見ても明確に判定できる
  ✅ アウトカムで書く（「施策をやる」ではなく「結果がどうなる」）
  ✅ 3〜5個（1 Objective に対して）
  ✅ 60〜70%達成で「良いOKR」（100%達成は目標が低すぎた証拠）
```

### 悪いOKRの典型パターン（禁止）
| アンチパターン | 例 | 正しい書き方 |
|---|---|---|
| 曖昧なObjective | 「事業を成長させる」 | 「〇〇市場でカテゴリリーダーになる」 |
| アクション型KR | 「キャンペーンを3本実施する」 | 「オーガニックCV数を月300件に増やす」 |
| 数値なしKR | 「品質を向上させる」 | 「NPS を 20 → 40 に改善する」 |
| 達成確実なKR | 「現状維持する」 | 伸び代のある目標を設定する |
| KR多すぎ | 8個以上 | 3〜5個に絞る |

## OKR設計ワークショップ手順

### Step 1: 前四半期振り返り（30分）
```
□ 前期OKRの最終達成率を全員で確認（数値のみ・言い訳なし）
□ 「達成できたKR」と「未達のKR」を分類
□ 未達理由を分類する:
   ① 目標設定が高すぎた（野心的）→ OK（次期に持ち越し）
   ② 実行力・優先順位の問題 → 根本原因を特定
   ③ 外部環境の変化 → 次期の仮説に反映
□ 次期に持ち越すテーマを決定
```

### Step 2: 優先課題の特定（30分）
```
□ 今期最重要の事業課題は何か → strategy-lead に確認
□ 顧客・市場インサイト → competitive-analyst / market-researcher
□ プロダクト改善課題 → product-manager / feedback-synthesizer
□ 最大PLインパクトを持つ施策は何か → kpi-analytics に概算依頼
□ 「やらないこと」を先に決める（Not-to-do List）
```

### Step 3: Objective の草案（15分）
```
各Objectiveに問うべき質問:
□ 読んだ人がワクワクするか（感情的引力があるか）
□ 1文で言い切れるか（「かつ」「および」を使っていないか）
□ 今期中に意味のある進捗が測定できるか
□ 事業戦略の最重要課題と直結しているか
□ Objective同士が矛盾していないか
```

### Step 4: Key Result の設計（45分）
```
各KRに問うべき質問:
□ この数値が達成されれば、Objectiveの達成を確信できるか？
□ 操作・水増しが難しい指標か（「訪問数」より「有料転換数」）
□ 計測方法は明確で、全員が同じ数字を見られるか？
□ ベースライン（現状値）は計測済みか？（未計測なら今週中に計測）
□ PLインパクトを計算できるか？（できない場合は kpi-analytics に依頼）
```

### Step 5: 整合チェック（15分）
```
□ 全社O → 部門O → 個人O の紐づきが明確か
□ 部門間で依存しているKRの担当者間で直接合意されているか
□ リソース（人・予算・ツール）が確保されているか（OKRは願望ではなく計画）
□ OKRに紐づかない重要業務（BAU）は別途KPIで管理されているか
```

## 月次チェックイン（4週ごと）

**日時**: YYYY/MM/DD
**所要時間**: 60分

### アジェンダ
```
1. 数値確認（10分）
   - 各KRの現在達成率を共有
   - 期末着地予測（現ペースで達成できるか）

2. 学びの共有（20分）
   - 「仮説と違った点」を各チームが1〜2個報告
   - 良い学び（成功パターン）と悪い学び（失敗パターン）

3. ブロッカーと対策（20分）
   - OKR達成を妨げているものは何か
   - 誰が・いつまでに・どうやって解決するか

4. 中間調整の判断（10分）
   - OKRの修正が必要か（原則として変更しない・ただし外部環境変化時は例外）
   - 優先順位の変更が必要か
```

## OKRスコアリング基準

| 達成率 | 解釈 | 次期への反映 |
|---|---|---|
| 70〜100% | 良好（野心的で達成可能な目標設定） | 次期は目標を引き上げる |
| 40〜69% | 要改善 | 未達原因を分類（実行力 / 設定 / 環境） |
| 0〜39% | 問題あり | OKR設計・実行力・優先順位を根本見直し |
| 100%超 | 目標が低すぎた | 次期は20〜30%引き上げ |
