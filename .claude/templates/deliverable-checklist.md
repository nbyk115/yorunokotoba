## 共通チェック（全成果物）

### 品質
```
□ 反証チェック（トリプルチェック）が完了しているか
□ 数値に根拠・ソースが付いているか
□ 「様子を見る」「大幅に改善」等の抽象表現が含まれていないか
□ PLインパクトが数値で示されているか（コンサル案件）
□ 結論→根拠→具体アクションの順序で書かれているか
```

### ブランド（brand-guidelines §1 準拠）
```
□ トーンが一貫しているか（プロフェッショナル / 数値ベース）
□ 禁止表現が含まれていないか
□ 日本語品質（ppt-presentation §1）が守られているか
□ クライアント名の表記が統一されているか
```

### 構造
```
□ エグゼクティブサマリーが冒頭にあるか

□ 図表にキャプション・出典が付いているか
□ 重要な数値・結論が視覚的に目立つか
```

## 成果物別チェック

```

```

### 市場調査レポート
```
□ 情報グレード（A/B/C/D）が全データに付いているか
□ 参入判断（Go/No-Go）と根拠が明示されているか
□ 競合マップ・ポジショニングが含まれているか
□ D評価（未検証）データが最終版に含まれていないか
```

### LP / クリエイティブ
```
□ ターゲット（状況ベース）が定義されているか
□ PageSpeed Insights 90以上を確認済みか
□ SP / PC両方で表示確認済みか
□ CTAのリンクが正しく設定されているか
□ OGP / meta descriptionが設定されているか
```

### コード納品
```
□ README.mdが更新されているか
□ 環境変数のサンプル（.env.example）が含まれているか
□ テスト（ユニット / E2E）がパスしているか
□ console.logが削除されているか
□ セキュリティスキャン（/security-scan）が済んでいるか
```
