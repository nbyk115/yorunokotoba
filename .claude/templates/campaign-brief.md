## 1. 背景・目的・PLインパクト

### ビジネス背景（必ず事業KPIと接続する）
> なぜ今このキャンペーンが必要か。「売上が◯%落ちた」「新セグメント開拓フェーズ」等、数値で状況を説明する。

### キャンペーン目的（1文・SMART形式）
```
[ターゲット] に対して、[チャネル・手法] を通じて、[期間] 中に [数値目標] を達成する。
```

| 悲観 | ¥ | ¥ | % | ¥ |

**ブレイクイーブン**:（総予算 ÷ 粗利単価 = 必要CV数）= ___件

> PLインパクト試算は `kpi-analytics` に依頼。根拠なき数値は記入禁止。

### このキャンペーンをやらない場合のコスト（放置コスト）
> 「やらないリスク」を数値で明示する。機会損失・競合への流出・ブランド劣化等。

## 2. ターゲット定義

### プライマリターゲット
- **Who（誰）**: セグメント名・属性（年齢/職種/会社規模等）
- **When（文脈）**: どんな状況・タイミングで接触するか
- **Pain（今の痛み）**: 顧客が抱えている具体的な課題
- **Gain（求める成果）**: このキャンペーンで顧客が得たいもの
- **Barrier（行動を阻む障壁）**: 購買・行動を妨げているもの

### セカンダリターゲット
- **Who**:
- **文脈**:
- **優先度**: プライマリの何%のリソースを使うか

### ネガティブターゲット（意図的に除外するセグメント）
> 広告費の無駄を防ぐ。除外理由も明記する。

## 3. カスタマージャーニー × ファネル設計

```
[認知] → [興味・検討] → [比較・評価] → [購買・行動] → [継続・紹介]

このキャンペーンが狙うファネルステージ:
□ 認知（まだ課題に気づいていない層へのリーチ）
□ 検討（課題を認識し、解決策を探している層）
□ 獲得（比較検討中・購買直前の層）
□ 維持（既存顧客のアップセル・継続）
□ 紹介（口コミ・紹介インセンティブ）
```

### ファネル別チャネル役割

|---|---|---|---|
| 認知 | | インプレッション / リーチ | |
| 検討 | | CTR / 滞在時間 | |
| 獲得 | | CVR / CPA | |
| 維持/紹介 | | LTV / NPS | |

## 5. メッセージ設計

### コアメッセージ（1文）
> ターゲットの「Before（今の痛み）→ After（得られる未来）」を結ぶ1文。A4用紙1行に収まる長さ。

### サポートメッセージ（3つまで）
1. （機能・証拠）
2. （感情・共感）
3. （社会的証明・権威）

| 社会的証明（誰が使っているか） | 高/中/低 | |
| 価格・ROI訴求 | 高/中/低 | |
| 緊急性・限定性 | 高/中/低 | |

### トーン&マナー

## 9. リスクと対策

| リスク | 可能性 | 影響度 | 事前対策 | 発生時対応 |
|---|---|---|---|---|
| KPI大幅未達（CPA 150%超） | 中 | 高 | A/Bテストで早期検証 | 入稿停止→クリエイティブ刷新 |
| 予算枯渇（期中） | 低 | 高 | 週次ペーシングチェック | チャネル優先度に基づき再配分 |
| ブランド毀損（炎上） | 低 | 最高 | 事前legal・brand-guardian確認 | 即時停止→pr-communications起動 |
| 景表法違反リスク | 低 | 最高 | compliance-playbook §2確認 | 即時削除・法務相談 |
| 競合の同時施策 | 中 | 中 | 競合モニタリング体制 | 差別化訴求へシフト |
