# Agent Evaluation — エージェントアウトプット品質評価スキル

## 概要

ConsultingOSの全エージェントアウトプットを標準化されたスコアリングフレームワークで評価する。
orchestratorが統合フェーズで必ず実行。品質スコアはアーカイブに自動記録される。

---

## §1 5軸評価フレームワーク

各エージェントのアウトプットを5軸で評価（各10点満点 → 合計50点）:

### 軸1: 証拠品質（Evidence Quality）/ 10点
| スコア | 基準 |
|---|---|
| 9-10 | 一次情報（政府統計・公式IR・査読論文）で数値が裏付けられている |
| 7-8 | 主要メディア・シンクタンク（グレードB）ソース付き |
| 5-6 | 業界メディア（グレードC）。数値はあるがソースが弱い |
| 3-4 | ソースが曖昧または古い（2年以上前） |
| 1-2 | 根拠なし・推測のみ |

### 軸2: ロジック完全性（Logic Completeness）/ 10点
| スコア | 基準 |
|---|---|
| 9-10 | A→B→Cの全ステップに根拠。反論を先取りして潰している |
| 7-8 | 主要ステップに根拠あり。小さな飛躍が1-2箇所 |
| 5-6 | 結論は妥当だが中間ステップが不明瞭 |
| 3-4 | 結論の半分以上が根拠なしで飛躍している |
| 1-2 | ロジックの骨格がない |

### 軸3: 実用可能性（Actionability）/ 10点
| スコア | 基準 |
|---|---|
| 9-10 | 「誰が・何を・いつまでに・どれくらいの予算で」が全て明示 |
| 7-8 | アクションが具体的。一部の詳細が欠ける |
| 5-6 | 方向性は示されているが実行ステップが曖昧 |
| 3-4 | 「検討する」「様子を見る」レベル |
| 1-2 | 抽象論のみ |

### 軸4: 数値・PLインパクト（Quantified Impact）/ 10点
| スコア | 基準 |
|---|---|
| 9-10 | 売上/コスト/粗利への影響が楽観/標準/悲観3シナリオで数値化 |
| 7-8 | 主要指標が数値化。シナリオは1-2個 |
| 5-6 | 一部数値あり。PLへの落とし込みが不完全 |
| 3-4 | 定性説明のみ。「大幅改善」等の曖昧表現 |
| 1-2 | 数値なし |

### 軸5: 反証チェック実施（Falsification）/ 10点
| スコア | 基準 |
|---|---|
| 9-10 | Deep（深度3）を完全実施。残存リスクも明記 |
| 7-8 | Standard（深度2）実施。主要反論は潰されている |
| 5-6 | Light（深度1）のみ |
| 3-4 | 反証チェックの記録があるが形式的 |
| 1-2 | 反証チェック未実施（ドラフト扱い） |

---

## §2 総合グレード判定

| 合計スコア | グレード | 意味 |
|---|---|---|
| 46-50 | S | 納品可能・クライアント提示レベル |
| 40-45 | A | 軽微な修正で納品可 |
| 33-39 | B | 要改善。1-2軸を修正してから次フェーズへ |
| 25-32 | C | 大幅修正必要。orchestratorが差し戻し |
| 0-24 | D | 作業やり直し。起点エージェントを再起動 |

**C以下は次フェーズへ進まない（品質ゲート）。**

---

## §3 orchestratorによる評価プロトコル

複数エージェントの並列アウトプットを統合する前に必ず実施:

```
【Agent Evaluation】
日時: YYYY-MM-DD
案件: [プロジェクト名]

エージェント: [名前]
  軸1 証拠品質:     [X/10] — [1行コメント]
  軸2 ロジック:     [X/10] — [1行コメント]
  軸3 実用可能性:   [X/10] — [1行コメント]
  軸4 PLインパクト: [X/10] — [1行コメント]
  軸5 反証:         [X/10] — [1行コメント]
  合計: [X/50] グレード: [S/A/B/C/D]
  判定: [PASS → 統合フェーズへ / REQUIRE_REVISION → 差し戻し / REJECT → 再起動]
```

### 差し戻し基準
- **C（25-32）**: 最低スコア軸のエラーコード（ERR-01〜06）を指定して差し戻し
- **D（0-24）**: エージェント再起動。インプットの再設計も検討

### PASSの閾値
- **統合フェーズへの移行**: 全エージェントBグレード以上
- **ユーザー提示（中間報告）**: Aグレード以上
- **最終納品**: Sグレード（少なくとも主要アウトプットはS）

## §4 評価ログの自動記録

orchestratorは評価完了後、以下のパスにログを記録:

```bash
root=$(git rev-parse --show-toplevel)
log="$root/.claude/memory/archival/quality_scores.log"
echo "[$(date '+%Y-%m-%d %H:%M')] [案件名] [エージェント名] [スコア/50] [グレード]" >> "$log"
```

蓄積されたログは `/maintenance` 実行時にトレンド分析に活用する。

---

## §5 セルフ評価（非orchestratorエージェント用）

orchestrator以外のエージェントも、アウトプット末尾に自己評価を付与:

```
【自己評価】
証拠品質: [X/10] | ロジック: [X/10] | 実用性: [X/10] | PLインパクト: [X/10] | 反証: [X/10]
合計: [X/50] / 推定グレード: [S/A/B/C/D]
改善余地: [最も低いスコア軸と改善方法を1文]
```

これにより orchestratorが受け取り判断を即座に行える。
