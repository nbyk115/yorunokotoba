# Agent Evaluation: 自己評価チェックリスト（軽量版）

> 旧版（259行・25点満点 × 5項目・A/B テスト前提）は 5ヶ月稼働ゼロで形骸化したため廃止。
> **現行は変更時セルフレビューのみ**。週次・月次の自動評価は無し（先回り最適化禁止）。

## いつ使うか
1. エージェント / スキルファイルを変更したとき
2. 反証モードで致命的問題が発見されたとき
3. 同じカテゴリの修正が3回続いたとき（構造を疑うシグナル）

## チェックリスト（5問）

### Q1. 役割境界
- このエージェント / スキルの責務は **1行で説明できる**か？
- 隣接エージェント / スキルとの境界は **動詞レベルで分離**しているか？

### Q2. 反証通過
- 出力末尾に【反証チェック結果】Step 1-3 が付いているか？
- 残存リスクは明示されているか？

### Q3. PL接続（コンサル系）/ 検証可能性（開発系）
- 数値（粗利・行数・テスト件数等）に落ちているか？
- 「大幅に」「適切に」等の抽象表現は排除されているか？

### Q4. 形骸化シグナル
- 直近3案件で実際に呼ばれたか？呼ばれていないなら**削除候補**
- 思想的基盤の人物名列挙が CLAUDE.md と重複していないか？

### Q5. 外科的変更
- 1コミット = 1目的か？
- 依頼の範囲を超えた「ついで」変更が混入していないか？

## 進化ログへの記録

変更を実施した場合のみ `evolution-log.md` に追記:

```
## YYYY-MM-DD
- 対象: [agent/skill 名]
- 変更: [Before → After 1行]
- 根拠: [反証で発見した問題 / 案件で顕在化した痛み]
- 反証結果: ✅ Step 1-3 / 🔺 残存リスク
```

**4週間更新ゼロなら archive 検討**。書かない選択を許容する（書くことが目的化しないように）。

## 廃止された機能（参考）

- A/B テスト 5回比較 → 単一案件で判断するノイズが大きい / 比較設計コストが回収できない
- 25点満点スコアリング → 数値化が形骸化、定性判断のほうが速い
- `/evolve` コマンド → 自動診断は問題顕在化を待つ規律と矛盾

> **判断**: 「自己進化」は機能ではなく、**変更時の規律**。スキルを増やすのではなく、規律を実行する。
