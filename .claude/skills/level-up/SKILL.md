# /level-up: ConsultingOS 5 質問レバレッジ発見スキル

## 起動方法
`/level-up` または「次のレバレッジ点を見つけて」と自然言語で指示。

## 目的
直近 1 週間の運用を振り返り、5 質問でユーザーの「次のレバレッジ点」を発見する。/audit の構造的ギャップとは別に、運用上の具体施策候補を抽出。

## 5 質問（佐藤裕介流に再構築）

ユーザーに以下 5 質問を順次提示し、回答を整理して施策候補に変換する。

### Q1: 反復作業の発見
直近 1 週間で 3 回以上同じ作業を繰り返したものはあるか?
→ 自動化候補（Skills 化 / SDK Phase 拡張 / GitHub Actions 化）

### Q2: 営業様式の検証
「ご提案させてください → 検討します → 失注」型の営業に時間を使っていないか?
→ 「成果物プレビュー営業」（consulting-playbook）への転換余地

### Q3: PL 思考の徹底度
PL に落とせない施策（粗利インパクト / ブレイクイーブン / ユニットエコノミクスが不明）に時間を使っていないか?
→ 佐藤裕介流 PL 思考の再徹底、施策中止判断

### Q4: アセット帰属診断
既存案件パイプライン（Hotice / N.Y.CRAFT）の知見を再利用できる新案件はあるか?
→ ストック資産化、商品ライン候補（8 商品）の実需顕在化検証

### Q5: 形骸化検出
30 エージェント / 19 スキルの中で「使われていない」リソースはないか?
→ ruthlessly edit（追加は削除と 1 セット）の発動、CLAUDE.md ハードルール 13 適用

## 出力フォーマット

各回答に対して以下を生成:
```
【Q1 への回答分析】
発見: <反復作業の特定>
施策候補: <Skills 化 / SDK Phase / 自動化 のいずれか>
優先度: 高 / 中 / 低（PL 直結性 + 実需顕在化で判定）
反証チェック: <Step 1-3 + 残存リスク>

（同様に Q2-Q5）

【総合: 次の 1 週間で着手すべき 1 施策】
施策名: <最優先 1 件>
理由: <PL 直結性 + アセット帰属性で選定>
所要時間: <見積もり>
成功基準: <測定可能な定義>
```

## 運用頻度
- 週次（毎週金曜日）: /audit 実施後に /level-up で具体施策化
- 月次（毎月第 1 金曜日）: 詳細 level-up（5 質問 + 商品ラインナップ候補の実需検証）
- 結果は `evolution-log.md` の「level-up 記録」セクションに append

## 重要原則
- ユーザーの回答に対して即施策化せず、まず反証 Step 1-3 を実施（コンセンサス追従回避）
- 「全部やる」ではなく「1 施策に絞る」（佐藤裕介流 ruthlessly edit）
- 5 質問すべてに「施策不要」と回答可能（形骸化施策の追加禁止）

## 関連
- `/audit`: 5 軸自己評価（本スキルの前段）
- `consulting-playbook.md`: 「成果物プレビュー営業」「PL 思考」「アセット優先原則」
- `CLAUDE.md` ハードルール 13: 形骸化ルール禁止
