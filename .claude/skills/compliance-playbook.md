## §1 法的リスク評価フレームワーク

### リスク分類（4象限）
```
                 影響度 HIGH
                      │
    【要即対応】        │  【戦略的対応】
    - 法令違反リスク    │  - 規制変更への対応
    - 刑事罰・行政処分  │  - 新市場の法的調査
    - 大規模損害賠償    │  - M&A・海外展開の法務
                      │
確率LOW ──────────────┼────────────── 確率HIGH
                      │
    【モニタリング】    │  【予防対応】
    - 潜在的グレーゾーン │  - 日常的な契約リスク
    - 将来の規制変更    │  - 軽微な利用規約違反
                      │
                 影響度 LOW
```

### 即対応が必要な警告シグナル
- 個人情報の無断収集・第三者提供
- 景表法違反（優良誤認・有利誤認）
- 資金決済法・金融商品取引法に触れる可能性
- GDPR/CCPA等の海外データ保護法違反
- 下請法・独占禁止法の抵触

## §2 契約レビューチェックリスト

### 必須確認項目（全契約共通）
```
【当事者】
□ 契約当事者の特定（法人格・代表権の確認）
□ 署名権限の確認（代表者 vs 代理権付与）
□ 連帯保証・保証条項の有無

【業務範囲・対価】
□ 業務範囲の明確性（「その他」条項の射程）
□ 対価・支払条件（遅延損害金・消費税）
□ 追加作業の費用負担ルール

【知的財産】
□ 成果物の著作権帰属（業務委託 vs 請負）
□ 既存IP・バックグラウンドIPの扱い
□ 顧客データの利用権限範囲

【機密情報・個人情報】
□ 秘密保持義務の範囲・期間
□ 個人情報の取り扱い（個情法準拠）
□ データ漏洩時の通知・賠償条項

【責任制限・免責】
□ 損害賠償の上限額（月額報酬×X倍など）
□ 間接損害・逸失利益の免責
□ 不可抗力条項の定義

【解除・終了】
□ 解除事由（期限の利益喪失条項）
□ 解除後の処理（データ返還・破棄）
□ 残存条項（秘密保持・競業禁止）

【準拠法・管轄】
□ 準拠法（日本法 / 海外法）
□ 管轄裁判所（合意管轄）
□ 仲裁条項の有無
```

### 危険な契約文言（要注意フラグ）
| 危険な表現 | リスク | 対処法 |
|---|---|---|
| 「甲の指示に従い、その他必要な業務を行う」 | 業務範囲無限定 | 業務範囲を具体的に列挙し、「に限る」と明記 |
| 「知的財産権は甲に帰属する」 | 既存IP・ノウハウも取られる | 「本契約に基づき新たに創出されたものに限り」と限定 |
| 「乙は甲の事前承諾なく第三者に開示できない」（期間無制限） | 永久秘密保持義務 | 期間を明示（例: 契約終了後3年） |
| 損害賠償上限なし | 無制限の賠償リスク | 月額報酬の○ヶ月分を上限として明記 |
| 「いつでも解除できる」（即時解除） | 突然の契約打ち切り | 事前通知期間（30日以上）を設定 |

## §3 個人情報保護・プライバシー法

### 日本（個人情報保護法）
```
個人情報の定義: 生存する個人を識別できる情報
（氏名・生年月日・住所・メールアドレス等）

取得時の義務:
□ 利用目的の特定・公表（プライバシーポリシー）
□ 要配慮個人情報（病歴・障害・犯罪歴）は明示的同意が必要

利用・提供時の義務:
□ 取得目的の範囲内での利用
□ 第三者提供は原則として本人同意が必要
□ 委託先への監督義務（業務委託先管理）

保管・廃棄:
□ 安全管理措置（技術的・組織的）
□ 不要データの適切な廃棄

重大インシデント時（漏洩・不正アクセス）:
□ 個人情報保護委員会への報告（速やかに）
□ 本人への通知（必要に応じて）
```

### 海外（GDPR/CCPA主要チェック）
```
GDPR（EU）:
□ 適法根拠の特定（同意/契約/正当利益/法的義務）
□ データ主体の権利対応（削除権・アクセス権・ポータビリティ）
□ データ保護責任者（DPO）の要否
□ 標準契約条項（SCC）または十分性認定国への移転
□ プライバシーバイデザインの実装

CCPA/CPRA（カリフォルニア州）:
□ 個人情報の「販売」禁止/オプトアウト対応
□ 年次収益$25M超またはデータ量基準の適用確認
□ プライバシーポリシーへのCCPA対応記載
```

## §4 景品表示法（景表法）チェック

### 優良誤認・有利誤認の判断基準
```
優良誤認（品質・効能の誇大表示）:
❌ 「業界No.1」（根拠なし）
❌ 「必ず効果が出ます」（保証できない）
❌ 「〇〇に認定された」（事実と異なる）
✅ 「X社比 Y%コスト削減（2024年自社調査）」（根拠明示）

有利誤認（価格・条件の誇大表示）:
❌ 「通常価格100万円のところ→50%OFF」（通常価格の実態がない）
❌ 「今だけ！残りわずか」（常時行っているセール）
✅ 「定価XX円（当社通常販売価格）」（実際の定価との比較）

SaaS特有の注意点:
□ 無料トライアル終了後の自動課金の明示
□ 解約条件・違約金の明確な表示
□ 機能制限（フリー vs 有料）の誤解を招く表現
```

## §5 SaaS/テック企業の重要法令

### 資金決済法（決済・ポイント）
```
前払式支払手段に該当する可能性:
- ポイント・クレジット・バウチャーの発行
- ウォレット残高の保持

該当する場合の義務:
□ 内閣総理大臣への届出（発行済み額 1000万円超）
□ 供託義務（前払残高の1/2以上）
□ 払い戻し対応（原則不可・例外あり）
```

### 下請法（SaaS開発委託）
```
適用基準（資本金）:
- 発注側: 資本金1億円超 / 受注側: 資本金1千万円以下 → 適用

主な規制:
□ 書面（3条書面）の交付義務
□ 支払期日は受領後60日以内
□ 不当な値引き・やり直し要求の禁止
□ 受領拒否・返品の禁止

SaaSでよくある問題:
□ 月次の「仕様変更」による追加作業の無償強要
□ 「検収」を長期間引き延ばす行為
```

## §6 コンプライアンス体制構築

### 最小限のコンプライアンス体制（スタートアップ向け）
```
Phase 1（創業〜30名）:
□ 就業規則・ハラスメント防止規程
□ 個人情報保護方針（プライバシーポリシー）
□ 反社チェック体制（取引先・採用候補者）
□ 契約書テンプレートの整備

Phase 2（30〜100名）:
□ 内部通報制度の整備
□ 情報セキュリティ基本方針（ISMS or 簡易版）
□ コンプライアンス研修（年1回以上）
□ 資金決済法・景表法の社内チェック体制

Phase 3（100名以上・上場準備）:
□ 内部監査体制
□ リスク管理委員会
□ 上場審査対応（反社・インサイダー取引防止）
□ 外部弁護士顧問契約
```

### インシデント対応フロー
```
発生 → 即時報告（24時間以内）
       ↓
初期評価（法的影響・規制当局報告要否・メディアリスク）
       ↓
封じ込め（影響拡大の防止）
       ↓
規制当局・当事者への通知（期限内）
       ↓
原因究明・再発防止策
       ↓
事後レポート（内部・外部）
```

## 🔺 反証モード（トリプルチェック必須）

本スキルを適用する全アウトプットは、CLAUDE.md「反証モード」セクションに定義されたトリプルチェックを必ず実行すること。法務・コンプライアンスは**Deep（深度3）**で実施。

1. **自己反証**: 「この判断が誤りだった場合、どんな証拠が出るか」を想定
2. **構造反証**: 法令条文・判例との整合性確認。「規制当局はこの解釈に同意するか」
3. **実用反証**: 「クライアントがこの判断に基づいて行動した場合、何が起きるか」

> ⚠️ 本スキルは参考情報。最終的な法的判断は必ず弁護士に確認すること。
