## §1 日本法務の基本チェックリスト

### 事業開始前の法的確認事項
```
□ 事業の許認可要件（業種によって異なる）
  - 金融業: 金融庁の登録・免許
  - 医療・薬事: 薬機法・医療法
  - 人材紹介: 職業安定法（厚生労働省許可）
  - 飲食業: 食品衛生法・消防法
  - 不動産: 宅地建物取引業法

□ 会社法・商法
  - 登記義務
  - 資本要件
  - 取締役・監査役の義務

□ 労働法
  - 雇用契約書の必須記載事項（労働条件通知書）
  - 就業規則（10名以上で必須）
  - 固定残業代の適法設計（最高裁判例準拠）
  - 業務委託 vs 雇用の判断基準（偽装請負リスク）
```

## §2 景品表示法（景表法）チェック

### 優良誤認・有利誤認の判定基準
```
優良誤認（不当表示）:
  「業界No.1」「最高品質」「最先端」
  → 客観的根拠なしに使用 → 景表法違反

  ✅ 合法的な使用条件:
  - 「〇〇調査によるシェアNo.1（2024年）」（調査名・年・対象明示）
  - 「〇〇賞受賞」（受賞事実の証明書類あり）

有利誤認（価格表示）:
  「通常価格10万円 → 特価5万円」
  → 通常価格が実際より高く設定 → 景表法違反

  ✅ 合法的な価格表示:
  - 二重価格表示は「過去8週間のうち4週間以上の実際の販売価格」が条件
  - 「希望小売価格」は製造業者の設定価格に限る
```

### 広告・LP チェックリスト
```
□ 「No.1」「最高」「最安値」等の根拠資料の保管
□ 体験談・口コミの「個人の感想です」表示
□ 比較広告の客観的根拠の保管（比較の公正性）
□ キャンペーン条件の明示（期間・対象・除外事項）
□ 定期購入商品の解約条件の分かりやすい表示（特商法）
□ 健康食品・サプリの効能効果の適切な表示制限
```

## §3 個人情報保護法（PIPA）対応

### プライバシーポリシー必須記載事項（2022年改正対応）
```
1. 事業者の名称・代表者・住所・問い合わせ先
2. 利用目的（具体的かつ明確に）
3. 第三者提供の有無・提供先
4. 共同利用する場合の情報
5. 委託先への監督責任
6. 保有個人データの開示・訂正・削除請求への対応
7. Cookie・トラッキング技術の使用（改正後の要求事項）
8. 越境移転する場合の移転先国情報
9. 保存期間（または決定基準）
10. 問い合わせ窓口
```

### 要配慮個人情報（センシティブデータ）
```
以下は取得に「明示的な同意」が必須:
  - 人種・民族・信条
  - 社会的身分
  - 病歴・健康診断結果・障害
  - 犯罪歴・被疑者情報
  - 性生活・性的指向

違反リスク: 個人情報保護委員会による勧告・命令・公表
```

### Cookie同意管理（Cookieバナー設計）
```
必須要素:
  □ 利用するCookieの種類説明（必須/分析/広告）
  □ 拒否オプションの明示
  □ 同意の記録・保存
  □ 事後撤回の仕組み

オプトアウト必要なケース:
  - GA4等の分析ツール → 同意マネジメントプラットフォーム（CMP）推奨
  - 行動ターゲティング広告
  - リターゲティングピクセル
```

## §4 AI規制・AI利用の法的リスク

### 日本のAI規制現状（2024年時点）
```
現状: 日本は横断的なAI規制法なし（EU AI Actとは異なる）
方針: 「信頼できるAI」に向けた自主的ガイドライン中心

【適用される既存法】
- 個人情報保護法: AIによる個人情報処理に適用
- 製造物責任法: AIシステムが「製造物」に当たる場合
- 著作権法: AI生成コンテンツの著作権帰属（2024年時点で解釈が流動的）
- 独占禁止法: AIによる価格協調・差別的取り扱い

【注意が必要なAI活用ケース】
- 採用選考へのAI活用 → 差別禁止・均等法リスク
- 与信・融資判断へのAI活用 → 説明可能性要求の高まり
- 医療診断補助 → 薬機法・医療法との整合
- 個人評価・スコアリング → PIPA開示請求対象の可能性
```

### AI生成コンテンツの著作権
```
【日本著作権法の現状解釈（2024年8月時点）】
- AI生成コンテンツは原則「著作権なし」（人間の創作性がない場合）
- ただし、人間が創作的関与をした場合は著作権が発生する可能性
- 「学習用データとしての著作物の利用」は著作権法30条の4で許容（但し目的制限あり）

【実務上の対応】
□ 生成AIツールの利用規約を確認（著作権の帰属条件が異なる）
□ 商業利用禁止のモデルを使用していないか確認
□ 学習データに著作権侵害コンテンツが含まれていないか（→ ベンダーに確認）
□ クライアント向け成果物にAI生成物を含む場合は開示を検討
```

## §5 契約書チェック（SaaS・業務委託・NDA）

### SaaSサービス利用規約 必須条項
```
□ サービスの定義・範囲（何ができて何ができないか）
□ 利用料金・支払い条件・自動更新の明示
□ 禁止行為（不正利用・リバースエンジニアリング等）
□ 知的財産権の帰属（特にユーザーが作成したコンテンツ）
□ データの取り扱い（保存・削除・移転）
□ サービス水準（SLA・メンテナンス免責）
□ 免責事項・損害賠償上限（通常は月額料金相当）
□ 解除条件（規約違反時の即時解除条件）
□ 準拠法・合意管轄（日本法、東京地裁を推奨）
□ 反社会的勢力排除条項
```

### 業務委託契約 注意点
```
【偽装請負リスク防止】
以下が揃うと「雇用」とみなされるリスク:
  - 業務遂行の具体的指示（指揮命令）
  - 勤務時間・場所の指定
  - 他社への業務禁止（専属性）
  - 機材・ツールの提供（会社側）

【著作権・成果物の帰属】
  - デフォルト: 委託先（受注者）に著作権が帰属
  - 「著作権を含む全ての権利を発注者に帰属させる」条項を必ず入れる
  - ソースコード・設計図・データも明示的に移転条項を入れる

【秘密保持（NDA）】
  - 受領した秘密情報の定義（明確化必須）
  - 開示禁止・目的外使用禁止
  - 有効期間（契約終了後も3〜5年の存続条項）
  - 例外規定（公知情報・独自開発情報）
```

## §6 海外展開の法的リスク早見表

| 地域/国 | 主要法規制 | リスクレベル | 対応 |
|---|---|---|---|
| EU | GDPR・EU AI Act・DSA/DMA | 🔴 最高 | DPO選任・データ処理協定・AIシステム適合性評価 |
| 米国 | CCPA/CPRA（CA）・州法乱立 | 🟠 高 | 州ごとに対応・SOC2取得を検討 |
| 中国 | PIPL・データセキュリティ法・ネットワーク安全法 | 🔴 最高 | データの中国国内保存要求・越境移転規制 |
| 東南アジア | PDPA（タイ/シンガポール等）・業種別規制 | 🟡 中 | 国別にPDPA準拠確認 |
| 日本（SaaS輸出側） | 外為法・安全保障貿易管理 | 🟠 高 | 該非判定・需要者確認 |

## §7 コンプライアンス品質チェック（成果物出力前）

```
□ 「業界No.1」「最高品質」等の根拠資料が存在するか
□ 個人情報の取り扱いにプライバシーポリシーが準拠しているか
□ 広告表現に景表法違反のリスクがないか
□ 契約書に著作権帰属・損害賠償上限・秘密保持条項があるか
□ AI利用規約に生成コンテンツの権利帰属・禁止事項が明示されているか
□ 海外展開時にGDPR/CCPA等の適用可能性を確認したか
□ 法的判断が必要な事項について「弁護士確認を推奨」と明示したか
```

## §ASI時代の法務・コンプライアンス

> AGI到達後、法務の「書類作成・契約レビュー・条文調査」はAIが担う。人間の法務専門家の価値は「倫理判断・責任の帰属設計・前例のない問いへの対応」に集約される。

### AI・AGI時代の新法務リスク

```
【新カテゴリーの法的問題】
1. AI生成コンテンツの著作権:
   - AIが生成したコンテンツの著作権は誰に帰属するか（日本法未確定）
   - 学習データの著作権侵害リスク（生成AIベンダーのToS確認必須）
   - 開示義務: 「AIが生成しました」という表示義務の有無（国・メディアにより異なる）

2. AIによる意思決定の説明責任:
   - AIが採用・融資・医療判断に使われる場合の差別・偏見リスク
   - EU AI Act（2024年〜）: リスクカテゴリ別の義務・禁止事項
   - 日本: AI事業者ガイドライン（経済産業省 2024年）

3. LLM利用の情報管理:
   - 秘密情報をLLMに入力することによる情報漏洩リスク
   - ChatGPT等外部LLM利用時のデータ保持ポリシー確認必須
   - 社内AI利用規程の整備（「LLMに入れてはいけない情報リスト」の明確化）

4. AGIエージェントの契約行為:
   - AIエージェントが自律的に締結した契約の法的有効性
   - 2027年以降: 「AIの行為の法的主体性」に関する国際議論が加速
```

### EU AI Act 対応チェックリスト（2024年〜）

```
リスクカテゴリの確認:
□ Prohibited（禁止）: ソーシャルスコアリング・無差別顔認識 → 絶対禁止
□ High Risk（高リスク）: 採用・信用評価・医療診断 → 適合性評価・ログ保存・人間監督
□ Limited Risk（限定リスク）: チャットボット → 開示義務（AIと明示）
□ Minimal Risk（最小リスク）: スパムフィルター等 → 規制なし

High Riskに該当する場合の必須対応:
□ 技術文書の整備（アルゴリズムの説明可能性）
□ ログ・監査証跡の保存（最低10年）
□ 人間による最終監督プロセスの設計
□ バイアステストと差別禁止の確認
□ EU市場向けにはCEマークに相当するコンフォーミティ宣言
```

### 日本のAI関連法務（2024-2026年版）

```
□ AI事業者ガイドライン（経済産業省・総務省 2024年）の確認
□ 個人情報保護法: AIによるプロファイリングへの本人通知義務
□ 不正競争防止法: 競合のAI学習データへのアクセス・スクレイピング制限
□ 景表法: AI生成コンテンツへの「AI生成」表示の要否（現時点では任意だが注意）
□ 労働法: AIによる労務管理・評価の透明性確保
□ 医療機器法: 医療AIプロダクトへの薬事承認の要否（SaMD分類）
```

### ASI時代に法務専門家が担う不可侵の価値

```
ASI耐性の高い法務業務:
✅ 前例のない法的問題への判断（AGI関連の新問題は全て前例なし）
✅ 倫理的グレーゾーンの判断（技術的に可能でも「すべきか」の判断）
✅ 規制当局との交渉・ロビー活動（人間の信頼関係が必要）
✅ 説明責任の設計（「誰が責任を取るか」の構造設計）
✅ クライアント・社内への信頼構築（緊急時・危機対応）

AIが代替する法務業務:
❌ 標準契約書のレビュー・リスクフラグ付け
❌ 判例・条文の検索・要約
❌ コンプライアンスチェックリストの適用
❌ デューデリジェンス資料のレビュー
```
