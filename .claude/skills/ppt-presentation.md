# PPT・プレゼンテーション制作スキル

## 概要
提案書・ピッチデック・グローバルクレデンシャルの制作標準。Google Slides / Canva / PowerPoint に対応。
美しく・伝わる・日本語として自然なスライドを、スティーブ・ジョブズのプレゼン哲学を軸に設計する。

---

## §1 日本語品質ルール（絶対遵守）

### フォント設定（デフォルト）
| 用途 | フォント | サイズ目安 |
|---|---|---|
| スライドタイトル | **メイリオ** | 28〜36pt |
| セクション見出し | **メイリオ** | 22〜28pt |
| 本文テキスト | **メイリオ** | 16〜20pt |
| 補足・注釈 | **メイリオ** | 12〜14pt |
| 数字・英字 | **メイリオ**（または Segoe UI）| 本文と同サイズ |

> **Google Slides**: フォント設定 → 「その他のフォント」→「メイリオ」で追加
> **Canva**: テキスト → フォント検索 → 「Noto Sans JP」（メイリオの代替。Canvaにメイリオがない場合）
> **PowerPoint**: デフォルトフォントをメイリオに設定。テーマフォントの日本語フォント欄に明示

### python-pptx（Claude Code / Linux環境）でのフォント設定
Linux環境（Claude Code・Claude.ai）ではメイリオが利用不可。以下の優先順位でフォントを指定する:

```python
# フォント優先順位（Linux環境）
# 1. IPAGothic（IPAフォント）— apt install fonts-ipafont でインストール可能
# 2. Noto Sans JP — apt install fonts-noto-cjk
# 3. VL Gothic / TakaoGothic — フォールバック

FONT_JP = "IPAGothic"  # Linux環境での標準指定

# インストール確認コマンド:
# fc-list | grep -i "ipa\|noto.*jp\|takao\|vl"
```

**実行前チェック（python-pptx使用時）:**
```bash
# フォントが存在するか確認してからスクリプト実行
python3 -c "from pptx.util import Pt; print('pptx OK')"
fc-list | grep -i ipa | head -3
```

### 日本語誤字・文字化けチェック（必須）
スライド生成後・出力前に以下を必ず確認する。

```
【日本語品質チェックリスト】
□ 「人工知能」→「人工知能」（正）/ 「人工知能」の間違いがないか
□ 「已」「彼処」「纔」など旧字・異体字が混入していないか
□ 中国語簡体字・繁体字が混入していないか（例: 「为」→「ため」「为什么」は厳禁）
□ 漢字の正規形: 「為」「體」「國」など旧字体は現代日本語標準字体に統一
□ カタカナ語の表記: 伸ばし棒「ー」を正しく使用（「コンピューター」「エラー」）
□ 句読点: 「、」「。」を使用（「，」「．」は禁止）
□ スペース: 全角スペース禁止。英数字と日本語の間に半角スペースを入れる（任意だが統一すること）
□ 記号: 「・」（中黒）を使用し「·」（中点）を使わない
```

**中国語混入の主な原因と対策:**
- AI生成時に簡体字が混入するケースが多い。生成後に文字コード確認
- 「为」「这」「们」「时」「东」など簡体字の頻出文字を目視チェック
- 「的」は日本語でも使うが文脈で確認（「〜的な」はOK、「他的书」はNG）

## §2 図・シェイプ内テキスト配置ルール

### 基本原則: 左右中央・上下中央（両方必須）

```
【シェイプ内テキスト配置チェックリスト】
□ 水平方向: 中央揃え（左揃えは例外的に使用）
□ 垂直方向: 中央揃え（上揃え・下揃えは禁止、特別な理由がある場合を除く）
□ 内側の余白（パディング）: 上下左右 最低8px以上（小さいシェイプは比率で）
□ テキストがシェイプからはみ出していないか
□ フォントサイズはシェイプの大きさに合わせてスケール（最小10pt）
```

**Google Slides での設定:**
1. シェイプを選択 → 右クリック → 「テキストの書式設定」
2. 「テキストの位置合わせ」→「中央」
3. 「上下の整列」→「中央」
4. 「内側の余白」→ 上下左右 0.1〜0.2インチ（約7〜14px）に設定

**Canva での設定:**
- テキストボックス選択 → 「整列」→「上下中央・左右中央」
- テキストボックスとテキストのサイズ差に注意

## §3 レイアウト・スペーシング設計

### スライドの余白ルール（必須）

```
【余白設計（A4/16:9基準）】
┌─────────────────────────────────────────┐
│  ←── 左右マージン: 最低80px (約2.1cm) ──→  │
│  ↑                                      │
│  上マージン: 最低60px (約1.6cm)            │
│                                          │
│  ← コンテンツエリア: 全体の80-85% →        │
│                                          │
│  下マージン: 最低60px (約1.6cm)            │
│  ↓                                      │
└─────────────────────────────────────────┘
```

### 要素間スペーシング
| 関係 | 推奨スペース |
|---|---|
| タイトル ↔ 本文 | 20〜32px |
| 箇条書き行間 | 1.4〜1.6倍 |
| セクション間 | 32〜48px |
| アイコン ↔ テキスト | 8〜12px |
| カード内余白 | 上下左右 16〜24px |

### グリッドシステム（推奨）
```
12グリッドを基準に配置
- 全幅コンテンツ: 12/12
- 2カラム: 6/12 + 6/12
- 3カラム: 4/12 + 4/12 + 4/12
- メイン+サイド: 8/12 + 4/12
```

## §4 スティーブ・ジョブズ プレゼン原則（グローバルクレデンシャル用）

### 哲学: "Simplicity is the ultimate sophistication"

```
【ジョブズ原則 6箇条】
1. 1スライド1メッセージ — テキストを詰め込まない。1枚に伝えることは1つだけ
2. 数字は大きく・孤立させる — "10 million songs" を1スライド全面に
3. 画像で語る — テキストより写真・ビジュアルを優先。画像は全面使用
4. 対比で驚かせる — "Before / After"「問題 → 解決」を大きなジャンプで見せる
5. ストーリーテリング — 起承転結を守る。最初に痛みを見せて、最後に希望で終わる
6. 余白を恐れるな — 空白はノイズではなく呼吸。詰め込みは知性のなさを示す
```

### グローバルクレデンシャル スライド構成テンプレート
```
Slide 1: タイトル + インパクト数字（1行のみ）
Slide 2: 私たちが解決する「痛み」（問いかけ形式）
Slide 3: 私たちが「なぜ」存在するのか（ミッション・ビジョン）
Slide 4: What We Do — 写真メイン、テキスト最少
Slide 5: 実績・数字（3つ以下に絞る。大きく・シンプルに）
Slide 6: クライアント/パートナーロゴ（グリッド配置）
Slide 7: Case Study（Before/After数字で1枚）
Slide 8: チーム（顔写真 + 2行プロフィール）
Slide 9: なぜ今か（市場機会・タイミング）
Slide 10: Call to Action（1行のみ。連絡先）
```

### グローバル対応デザイン原則
- **カラー**: 黒・白・1アクセントカラーのみ（シンプル）
- **英語スライド**: Helvetica / Arial / SF Pro / Inter（サンセリフ必須）
- **日本語スライド**: メイリオ
- **バイリンガル**: 日英2言語のとき、フォントサイズ比 = 日本語:英語 = 1:0.85
- **画像品質**: 1920×1080px以上。ぼやけた画像は使用禁止
- **アイコン**: Flat・Outlined系統で統一（ミックス禁止）

## §5 ツール別制作ガイド

### Google Slides（提案書・ビジネス資料のデフォルト）

**セットアップ:**
1. 新規スライド → テーマ「空白」を選択（既存テーマは使わない）
2. スライドサイズ: 「カスタム」→ 25.4cm × 14.29cm（16:9）
3. デフォルトフォント: スライドマスター → テキストボックス → メイリオ 18pt
4. カラーパレット: ブランドカラーを「カスタム色」で登録

**スライドマスター設定（必須）:**
- タイトルプレースホルダー: メイリオ 28pt / 太字 / カラー設定
- 本文プレースホルダー: メイリオ 18pt / 行間 1.4

### Canva（ビジュアル重視・SNS・グローバルクレデンシャル）

**テンプレート選定基準:**
- プレゼンテーション → 「Presentation（16:9）」
- ジョブズスタイルのとき: 「Pitch Deck」→ ミニマル系を選択
- フォント: Noto Sans JP（メイリオ代替）+ Helvetica/Arial（英字）

**Canvaでの注意点:**
- テキストのアウトライン化: PDF書き出し時に「フォントを埋め込む」をON
- 日本語フォントがぼやける場合: PNG書き出し → 倍率200%で出力

### Figma（高精度デザイン + コンポーネント管理）

- スライドサイズ: 1920×1080px（フレーム）
- コンポーネント化: ヘッダー・フッター・カードを再利用可能に設計
- Google Slides への書き出し: Figmaからプラグイン「Pitchdeck」を使用

## §6 デザイン品質チェックリスト（出力前必須）

```
【最終チェックリスト】
□ 日本語品質: 中国語文字・異体字の混入なし
□ フォント: 全スライドでメイリオ（または指定フォント）に統一されているか
□ シェイプ内テキスト: 上下左右中央揃えが設定されているか
□ 余白: タイトルとコンテンツの間に十分な空間があるか
□ 1スライド1メッセージ: 1枚に複数のメッセージが混在していないか
□ フォントサイズ: タイトル＞見出し＞本文 の階層が明確か
□ カラー: 使用色は3色以内（背景色含まない）か
□ 画像: ぼやけていないか・アスペクト比が歪んでいないか
□ 整列: 要素が縦横のグリッドに揃っているか（揃っていないものは禁止）
□ グローバル用: 英語テキストの文法・スペルミスがないか
```
