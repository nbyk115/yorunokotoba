## 1. なぜ従来 KPI で AI ROI が測れないか

従来のマーケティング KPI（CVR / CPA / ROAS）は「人 1 人が 1 単位のアウトプットを生む」前提で設計されている。AI 導入による効果は「同じ人員がより多くのアウトプットを出す」「同じコストでより高品質なアウトプットが出る」という形で現れるため、アウトプット総量を固定した従来フレームでは差分が見えない構造的問題がある。

具体的に測定が困難な理由:
- AI 効果は「コスト削減」より「コスト回避」として発生する（採用を抑制したコストは PL に現れない）
- 速度向上は CVR に直結しないが、意思決定サイクルを早める間接効果を持つ
- アウトプット品質の改善は定性的で、CVR / CPA 指標では定量化されない
- AI ツール費用（SaaS 月額）は小さく見えるが、人件費代替効果は複数桁大きい

INFERENCE: Callan Consulting "State of AI in Technology Marketing 2026"（2026 年 4 月）によれば、AI 特有の KPI を追跡しているマーケターは調査回答者の一部にとどまるとされ、AI ROI の数値化が業界全体の課題として明示されている。McKinsey 調査でも上位企業の AI ROI 定量化が困難であることが報告されている。

## 2. 4 軸測定フレーム

AI 投資の効果を以下 4 軸で分解して計測する。軸ごとに独立した KPI ツリーを持ち、軸間の混在計測は禁止。

| 軸 | 測定対象 | 代表的 KPI | ビジネス換算 |
|---|---|---|---|
| 速度 (Speed) | タスク完了時間の短縮 | 制作リードタイム / 初稿完成時間 | 時間単価 x 削減時間 |
| アウトプット量 (Volume) | 単位時間あたりの生産物数 | コンテンツ本数 / レポート件数 | 増分アウトプット x 単価 |
| コスト回避 (Cost Avoidance) | 採用・外注・残業の抑制 | 回避外注費 / 回避採用コスト | 実際に発生しなかったコスト |
| 品質 (Quality) | アウトプットの正確性・一貫性 | 修正回数 / ブランド準拠率 / 初稿承認率 | 手直しコスト削減 |

**IMPORTANT**: 4 軸を合算して「AI 総合スコア」に丸めることを禁止。軸ごとの独立評価を維持する（小野寺メソッド: 目的別 KPI 分岐の原則）。

## 3. 各軸の測定指標と算出式

### 3-1. 速度軸

| KPI | 算出式 | 計測方法 |
|---|---|---|
| 制作リードタイム削減率 | (AI 導入前平均時間 - AI 導入後平均時間) / AI 導入前平均時間 x 100 | タスク管理ツールのタイムスタンプ |
| 初稿完成時間 | ブリーフ受領から初稿提出までの実測時間（分） | Slack / Jira / Notion タイムスタンプ |
| 承認サイクル回数 | 1 コンテンツあたりの差し戻し回数 | CMS / ワークフローログ |
| 意思決定サイクル | レポート依頼から意思決定まで（時間） | カレンダー + Slack ログ |

### 3-2. アウトプット量軸

| KPI | 算出式 | 計測方法 |
|---|---|---|
| 月次コンテンツ生産量 | 月ごとの公開コンテンツ総数（ブログ / SNS / 広告バナー） | CMS ダッシュボード |
| 1 人あたりアウトプット量 | 月次総アウトプット数 / マーケターヘッドカウント | CMS + HR システム |
| A/B テスト実施頻度 | 月次 A/B テスト本数 | 実験管理ツール / GA4 |
| 多言語バリエーション展開数 | 1 ソースコンテンツから派生した言語バリエーション数 | 翻訳 / CMS ログ |

### 3-3. コスト回避軸

| KPI | 算出式 | 計測方法 |
|---|---|---|
| 回避外注費 | (外注していた場合の推定単価 x AI 処理件数) - AI ツール費 | 過去外注単価 x 処理件数 |
| 回避採用コスト | 増員が不要になった FTE 数 x 採用単価 | HR 見積もり比較 |
| 残業抑制コスト | 削減された残業時間 x 残業単価 | 勤怠システム比較 |
| SaaS 代替コスト | 廃止・縮小できた外部ツール費用の合計 | 契約管理システム |

INFERENCE: 「コスト回避」は PL 上に現れないため、経営報告では「回避コスト = 潜在コスト - 実支出」として明示的に追記する。会計上の節約とは別項目で管理することを推奨する。

### 3-4. 品質軸

| KPI | 算出式 | 計測方法 |
|---|---|---|
| 初稿承認率 | 初稿で承認された件数 / 全提出件数 x 100 | ワークフローシステム |
| ブランド準拠率 | ブランドガイドライン違反指摘件数 / 全レビュー件数 | レビューログ |
| 事実誤認率 | ファクトチェックで修正が発生した件数 / 全コンテンツ件数 | QA ログ |
| 再利用率 | 一度生成されたアセットが再利用された回数 | アセット管理ツール |

## 4. ROI 金額換算ロジック

人員削減なき生産性向上（Productivity Gain without Headcount Reduction）を現金化する考え方。

### 4-1. 基本換算式

```
AI 生産性 ROI（年次） =
  ( 速度効果 + アウトプット量効果 + コスト回避効果 + 品質効果 )
  - ( AI ツール費用年額 + 導入・学習コスト )
```

### 4-2. 速度効果の現金化

```
速度効果（円） =
  削減された総時間（時間） x 時間単価（円/時間）

例:
  制作リードタイム 40% 削減 x 月間制作時間 200h x 時間単価 5,000 円
  = 月次 40,000 円 / 年次 480,000 円
```

### 4-3. アウトプット量効果の現金化

```
アウトプット量効果（円） =
  増分アウトプット数 x 1 件あたり外注相当単価

例:
  月 10 本増 x 外注単価 30,000 円 = 月次 300,000 円
```

### 4-4. ブレイクイーブン計算

```
ブレイクイーブン到達月 =
  ( 導入コスト + 学習コスト ) /
  ( 月次速度効果 + 月次アウトプット量効果 + 月次コスト回避効果 )
```

IMPORTANT: 金額換算値は「推定値」として明示し、楽観/標準/悲観の 3 シナリオを必ず示す。単一シナリオでの ROI 提示は禁止（kpi-analytics 規律準拠）。

## 5. AI 特有 KPI ダッシュボード設計

marketing-analyst が Looker Studio / Tableau / Metabase で構築する際のテンプレート。

```
【ダッシュボード名】AI 生産性 KPI ダッシュボード
【対象ユーザー】CMO / マーケ部門長 / 実務担当者
【更新頻度】週次（速度・量軸）/ 月次（コスト回避・品質軸）
【データソース】タスク管理ツール / CMS / 勤怠 / 会計

【セクション 1: サマリー（北極星指標）】
  - AI 生産性倍率: (現在の 1 人あたりアウトプット量) / (AI 導入前ベースライン)
  - 月次コスト回避額（推定）: 速度 + 量 + 回避の合算
  - AI ツール費用対効果（ROI 倍率）: 月次効果 / 月次ツール費

【セクション 2: 4 軸トレンド】
  - 速度軸: 制作リードタイム推移（月次折れ線）
  - 量軸: 1 人あたり月次アウトプット数（月次棒グラフ）
  - コスト回避軸: 累積コスト回避額（月次積み上げ）
  - 品質軸: 初稿承認率 / 事実誤認率（月次折れ線）

【セクション 3: 人別・チーム別分解】
  - 個人の AI 活用度スコア（AI ツール使用頻度 x アウトプット量）
  - チーム別 4 軸レーダーチャート

【アラート条件】
  - 事実誤認率が前月比 2 倍超 → QA プロセス見直しトリガー
  - 制作リードタイムが基準値を 20% 超過 → AI ワークフロー診断トリガー
  - AI ツール費用が月次コスト回避額を超過 → ツール選定再評価トリガー
```

Critical Few KPI 規律（コーシック）: ダッシュボードの北極星指標は 3 つ以内に絞る。4 軸全ての詳細指標は「深掘りサブダッシュボード」に分離する。

## 7. 形骸化検知

KPI 数値だけを追跡して実態が伴わない「数値ゲーミング」を検出するチェックリスト。

### 7-1. 形骸化パターンと検出方法

| 形骸化パターン | 検出シグナル | 対処 |
|---|---|---|
| アウトプット量は増えたが品質が劣化 | 初稿承認率の低下 / 事実誤認率の上昇 | 量軸目標を一時停止し品質軸を優先 |
| 速度向上したが意思決定が遅い | 制作リードタイムは短縮 / 承認サイクルは変化なし | ボトルネックが AI ではなく承認プロセスと特定 |
| コスト回避額の過剰見積もり | 回避額 > 実際の外注削減額 | 実外注請求書と照合して推定値を補正 |
| AI ツール費が効果を上回る | ROI 倍率 < 1.0 が 3 ヶ月継続 | ツール廃止 or 代替ツール評価 |
| 一部メンバーのみが活用 | チーム別 AI 活用度スコアの分散が大 | 活用度低いメンバーへの研修・ワークフロー再設計 |

### 7-2. 四半期形骸化レビュー（kpi-analytics 規律準拠）

毎四半期、以下の Pre-mortem チェックを実施する:

- 「この 4 軸 KPI を 1 年追って失敗するシナリオを 5 つ挙げよ」
- 「数値は改善しているが現場感覚と乖離している箇所はどこか」
- 「AI ツールへの依存度が上がりすぎてスキル劣化していないか」

## 8. 関連エージェント連携

| エージェント | 役割 | AI KPI との接点 |
|---|---|---|
| marketing-analyst | 主担当: AI KPI ダッシュボード構築・運用 | GA4 / Looker Studio への AI 指標追加、4 軸トレンドの可視化 |
| kpi-analytics | 副担当: ROI 金額換算・ブレイクイーブン計算 | PL 連携・楽観/標準/悲観シナリオ設計 |
| marketing-director | 戦略承認: AI KPI 目標値の設定 | 北極星指標の承認・予算配分への反映 |
| strategy-lead | 事業 KPI との整合: AI 効果が事業 KPI に接続されているか確認 | AI 生産性倍率を事業成長率 KPI ツリーに統合 |
| performance-marketer | 広告 AI 活用の ROI 測定 | AI 生成クリエイティブの ROAS 比較・A/B テスト設計 |

### 責任分界

- marketing-analyst が計測設計・ダッシュボード構築を主担当
- kpi-analytics が金額換算ロジック・ブレイクイーブン計算・シナリオ設計を担当
- 両エージェントの連携プロトコル: marketing-analyst がデータを収集・可視化し、kpi-analytics が意思決定用の数値解釈を付与する

## 9. 運用ルール

- 本フレームの初回適用時: 「AI 導入前ベースライン」を必ず計測・記録してから導入を開始する。ベースラインなしの ROI 測定は無効とする
- 計測ツール選定: marketing-analyst に優先相談。GA4 / タスク管理ツール API / CMS API の統合設計が必要
- 更新サイクル: 4 週間ごとに evolution-log.md の「再評価カレンダー」で形骸化チェックを実施
- スコアリング: 4 軸のうち 3 軸以上で目標達成 = 「AI 投資成功」と判定（単一軸達成は不十分）
