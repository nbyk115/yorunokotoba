# Cybersecurity Playbook 補助: 脆弱性パッチ管理（パッチの波対応）

> 親ファイル: `.claude/skills/cybersecurity-playbook.md`。AI による脆弱性発見の爆発的増加（「パッチの波」）への、パッチ管理の標準手順。2026-05-17 統合（出典: 平和博「AI『ミトス』で増す攻撃力」Yahoo! ニュース エキスパート、2026-05-18 付）。

## 1. 背景: なぜ今パッチ管理か

AI による脆弱性探索の自動化で、発見・公表される脆弱性の件数が桁違いに増加する局面（「パッチの波」）が予測されている。攻撃側は AI で脆弱性発見と悪用を高速化するため、防御側のパッチ適用速度が追いつかないと露出時間が拡大する。

- FACT（記事報道）: Anthropic が 2026-04-10 公式ブログで対応推奨事項を公表。英 NCSC が 2026-05-01「脆弱性パッチの波に備える」を公表
- FACT（記事報道）: 日本でも 2026-05-12 に首相がサイバー安全保障担当中心の対策を指示、2026-05-14 に金融庁が官民共同作業部会を開催
- INFERENCE: 「攻撃側は組織規模を区別しない」ため、中小規模の ConsultingOS 案件（自社プロダクト / クライアント）も対象になりうる

## 2. Anthropic 推奨事項（2026-04-10 公式ブログ、記事報道）

1. 米 CISA「既知悪用脆弱性（KEV）」リストのパッチは すべて適用する
2. パッチの優先順位付けに FIRST 運営の「EPSS（エクスプロイト予測スコアリングシステム）」を使う。EPSS は CVE 脆弱性が 30 日以内に悪用される確率を予測する標準指標
3. インターネット接続システムのパッチ適用時間を短縮する。脆弱性発見から 24 時間以内が推奨、その他の脆弱性も数日以内
4. システム停止リスクが許容範囲内なら、パッチの展開と再起動を自動化する
5. 脆弱性の発見件数が桁違いに増えることを前提に計画する。受け入れ / トリアージ（優先順位付け）/ 修復状況の追跡の各プロセスも、件数増加に合わせて迅速化する

## 3. 英 NCSC 推奨事項（2026-05-01「脆弱性パッチの波に備える」、記事報道）

1. 「パッチの波」到来に今から備える
2. 攻撃対象領域（attack surface）の優先順: まずインターネット接続および外部露出システムを速やかに特定・対処し、次にクラウド / オンプレミス内部環境をカバーする
3. 自動更新が利用可能なら有効化する
4. 自動更新が使えない場合は、米 CISA の「SSVC（ステークホルダー別脆弱性分類）」のようなリスク優先順位付けの仕組みで、どのパッチを先に当てるか判断する
5. YOU MUST: パッチ適用だけでは不十分。サポート終了 / サポート対象外のシステムは、新システムに置き換えるか、サポート対象に戻す

## 4. ConsultingOS 適用

### 4.1 ConsultingOS 自体のリポジトリ（consulting-os / 案件リポ）

- 依存パッケージのパッチは §8.5 サプライチェーンセキュリティの Dependabot 設定で自動 PR 化。低リスク更新は自動マージ、メジャー更新は手動レビュー
- GitHub の Dependabot security alert / Secret Scanning は有効化必須（cybersecurity-playbook §3 既定）
- 使い捨てコンテナ実行基盤のため OS レイヤのパッチ管理は基盤側責務。ConsultingOS 側の責務は依存ライブラリと GitHub Actions のバージョン固定 + 更新

### 4.2 クライアント助言時（infra-devops / legal-compliance-checker）

クライアントのセキュリティ案件・DD で本セクションを参照し、以下を助言の骨子とする:

- KEV 全件パッチ + EPSS による優先順位付けを推奨
- 外部露出資産から内部へ、の順で attack surface を棚卸し
- 自動更新の有効化、不可なら SSVC でトリアージ
- サポート終了システムの置換計画を「今やれること」として明示

### 4.3 今やれること（チェックリスト）

- [ ] 配布済みの脆弱性パッチを確実に適用済みか確認
- [ ] 「パッチの波」を想定したトリアージ / 修復追跡プロセスの計画
- [ ] 「サポート終了」「サポート対象外」システムの洗い出しと対応方針決定
- [ ] インターネット接続資産の特定と優先対処

## 5. 関連フレームワーク（公式リソース）

| 略称 | 正式名 | 運営 | 用途 |
|---|---|---|---|
| KEV | Known Exploited Vulnerabilities | 米 CISA | 実際に悪用が確認された脆弱性のカタログ。全件パッチ対象 |
| EPSS | Exploit Prediction Scoring System | FIRST | CVE が 30 日以内に悪用される確率予測。優先順位付け |
| SSVC | Stakeholder-Specific Vulnerability Categorization | 米 CISA | ステークホルダー別のリスク優先順位付けフレームワーク |
