## 1. 核心テーゼ: 和製 FDE = PoC 地獄構造

ユーザー発言: 「おそらく和製 FDE は基本 AI ベンチャーの PoC 地獄になるのではと言うのが当面の見通し」

= 日本市場の AI ベンチャーは構造的に以下のパターンに陥る:

1. PoC 案件は短期売上に上がる
2. PoC は売上として「積み上がらない」（一過性、ストック化しない）
3. システム運用保守 = 長期的に美味しい収益は他社（既存 SI / 大手）に持っていかれる
4. AI ベンチャーは PoC 提供で疲弊、労働集約から脱却できない

ConsultingOS 整合（既存 SaaS 死テーゼ PR #133 + ハーネスエンジニアリング PR #145 との関係）:
- Anthropic 垂直 OS 「SaaS 死」= 表層、ただし「PoC + 運用保守」二層構造は依然存続
- Sierra のハーネスエンジニアリング = 運用保守領域の moat 化、PoC からの脱出経路

= ConsultingOS は「PoC 地獄構造」を honest に認識した上で脱出戦略を選択する必要あり。

## 2. PoC 地獄脱出 4 戦略

ユーザー提示 4 戦略 (FACT、ConsultingOS 事業当事者分析):

### 2.1 戦略 1: 営業 / 案件増やす

- 短期: 売上拡大 (例: 関根さん N&Y Craft OEM イニシャル獲得済)
- 中期: 案件数増加 → 工数膨張 → 1 人アクセンチュア状態（労働集約）
- 限界: 個人 orchestrator のキャパ上限、Sierra ハーネス moat に到達しづらい

ConsultingOS 整合: マスク式原則 4 アルゴリズム実行 5 ステップ「サイクル加速」と整合、ただし単独だと限界あり。

### 2.2 戦略 2: アップセル / クロスセル

- 既存顧客内での課題を見つけて仕事を増やす
- LTV/CAC 改善、ストック化への第一歩
- Legora 流原則 1 チェンジマネジメント（PR #DL）と連動

ConsultingOS 整合: 関根さん N&Y Craft OEM Phase 1 構築後の Phase 2-3 拡張パス（在庫 / 配送 / 顧客対応の業務 OS 化）。

### 2.3 戦略 3: 運用保守領域獲得

- PoC 単発提供から運用保守の長期契約獲得へ
- Sierra ハーネスエンジニアリング (PR #145) と同型 = 本番稼働技術蓄積が moat
- 既存 SI / 大手から運用保守を奪う必要、構造的に困難

ConsultingOS 整合: ハードルール 17 体系 + 21 hook + 反証チェック = 運用保守の moat 化、ConsultingOS 規律体系が「Sierra ハーネス」の個人版。

### 2.4 戦略 4: 自社 SaaS 化（ユーザー推奨方向）

ユーザー発言: 「自社 SaaS の何が良いかって、運用保守も丸っと自前でパッケージングされていて、月額モデルで積み上がるところ。そして労働集約から脱却できるので疲れない。バリエーションも上がる。」

= 4 戦略のうち構造的に最も優位:
- 運用保守 packaging 済 = 単発でなく月額ストック
- 月額モデル = 売上積み上がり、Anthropic CFO 指数関数思考 (PR #144) との整合
- 労働集約脱却 = アルトマン式原則 1 AI 1 人チーム + 原則 8 アイデアマンの逆襲
- バリエーション向上 = a16z Defensibility「Action exhaust」+ vertical AI-OS の moat 化

ConsultingOS 整合:
- 関根さん N&Y Craft OEM = 自社 SaaS パッケージング戦略の実証（イニシャル + 月次 + レベシェア = 月額モデル）
- 水野さん v4 投資テーゼ = 自社 SaaS への投資、vertical AI-OS verticalization trend への early bet
- 「1 人アクセンチュア状態に収斂」(ユーザー発言) = 戦略 1 単独の限界、戦略 4 への収斂を構造的に推奨
