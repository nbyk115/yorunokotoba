# マスク式インセンティブ設計原則（2026-05-08 統合）

`strategy-lead` + `client-success` + `marketing-director` 連携で、ConsultingOS 自身および ICP のマネジメント / 組織設計 / KPI 設計に適用する横断原則。`consulting-playbook.md` §11「致命傷を避ける 2 原則」が「立ち上げ / 撤退」の上流判断であるのに対し、本節は「日常運用での大胆挑戦」の継続条件を担う。

## 原則: リスクとリターンのバランスを大胆行動側に傾ける

イーロン・マスク発言（2026 年公開対談、INFERENCE: 文脈は SpaceX / Tesla / xAI 組織運営、一次出典 URL 未確認）: 「失敗を過度に罰すれば人は漸進的にしか動かない。誰も罰を恐れて大胆に挑戦しない。リスクとリターンを均衡させ、大胆行動を有利にしなければイノベーションは起きない」。

**YOU MUST**: 罰則系仕組み（KPI 未達ペナルティ / 監査ログ / 違反記録 / 検証ゲート）を導入する全提案で「報酬系仕組み」との対称配置を必須化。罰則のみの片肺設計は漸進バイアスを生み、Goodhart 化（数値達成のための形式逃避）を加速する。

## 適用ガイド 3 軸

「軸 1: 罰則 / 報酬の対称配置」: 違反ログを作るなら大胆挑戦ログも作る、未達 KPI を採点するなら高 ROI 失敗を称賛する、検証ゲートを増やすなら実験許可枠も増やす。罰則のみの片肺設計を禁止。

「軸 2: 失敗コストの上限管理」: 大胆挑戦の失敗コストに上限を設定し（例: 月次予算 X% 以内 / 6 ヶ月以内に撤退ライン）、上限内なら失敗を罰しない。`consulting-playbook.md` §11「致命傷を避ける 2 原則」の撤退判断ゲートと統合運用。

「軸 3: 漸進 vs 大胆の比率宣言」: 提案 / PR / 施策に「漸進改善」「大胆挑戦」ラベル付与、四半期 / 月次で比率を可視化。漸進 100% は組織が萎縮しているシグナル。`marketing-director` 60/40 ルール（既存 60% / 新規 40%）の組織版として運用。

## 原則 2: 評判資本主義（2026-05-08 アプデ統合）

イーロン・マスク発言（2026 年公開対談、INFERENCE: Aakash Gupta 経由 X ポスト、一次出典 URL 未確認）: 「現金は資産として不十分、真の資産は評判と知識」。

「市場が評判 / 知識を資本へ自動変換する事象」が原則 1 と並ぶ柱。背景観察:

- 資本の転換: サム・アルトマン解雇直後に無制限リソース約束、ミラ・ムラティ製品ゼロで $2B 調達 = 創業者の評判そのものが資本として機能（FACT: メディア報道、独立検証要）
- 信用の指標: Adam Neumann（WeWork 崩壊）/ Travis Kalanick（Uber 追放）でも数億ドル即調達 = 過去失敗より事業スケール経験が評価される
- 最適化エラー排除: イーロンは全財産失っても再富豪化可能 / SBF（FTX）は信用破壊で完全排除 = 真の資産は銀行残高でなく他者の信用ネットワーク

YOU MUST: KPI / 報酬 / 投資判断で「現金 / 形式達成度（PR 数 / hook 数 / score）= 遅行指標」と「評判 / 知識 / 実証 ROI / クライアント推薦 = 真の資産」を区別。後者の最大化を優先する設計を強制。

## 適用ガイド 4 軸（原則 2 追加）

「軸 4: 評判 / 知識資産化」: 罰則 / 報酬の対称配置（軸 1）の報酬単位を「現金 / ボーナス」から「評判 / 知識 / 推薦ネットワーク」に拡張。具体例:
- KPI: 売上 / 利益 → 加えて「業界推薦数」「メディア露出」「採用候補者の流入」「クライアント NPS」を主指標化
- 評価: ボーナス → 加えて「カンファレンス登壇権」「業界レポート寄稿権」「研究予算」を報酬化
- 投資判断: 現金 ROI → 加えて「評判フライホイール起動コスト」「知識複利の起点になるか」を判定軸化

軸 1 ~ 3 は `consulting-playbook.md` 本体 §マスク式インセンティブ設計参照。
