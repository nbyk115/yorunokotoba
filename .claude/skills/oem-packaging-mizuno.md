## 1. 概要: OEM パッケージの位置づけ

ConsultingOS は「個人版 Anthropic JV + Sequoia "サービスがソフトウェア" の物理達成形態」として設計された AI 駆動の経営統合 OS である（出典: ICP.md §5 / evolution-log.md 2026-05-04 Phase 5 設計記録）。

水野氏向け OEM パッケージは以下の思想で設計する:

- ConsultingOS の金融業界プレイブック（finance.md §8 Claude 公式金融エージェントテンプレ統合を含む）を、水野氏固有の投資家ユースケースに特化した「カスタムラッパー」として提供する
- 属人営業ではなく「構造と再現性で売る」原則（佐藤裕介メソッド）を適用。個人の関係値に依存しない仕組みとして設計
- フロー（案件単位の成果物）とストック（毎月更新されるプレイブック・テンプレ資産）を統合して提供（小野寺信行メソッド）

### 痛み（Pain）

- 投資先企業の財務評価・企業価値分析を都度外部委託 → コスト高・スピード遅延
- ピッチブック・企業価値評価レポートの品質が案件ごとにばらつく
- KYC・デューデリジェンスのワークフローが属人化・標準化不足
- 投資先のモニタリング（月次 KPI / 業績推移）の集計・可視化に工数がかかる

### ジョブ（Job to be Done）

- M&A・エンジェル投資の案件評価を「再現可能な品質」で高速化する
- 投資先企業の月次モニタリングを自動化・定形レポート化する
- KYC / デューデリジェンスを法令準拠しつつ効率化する
- ピッチ資料・クレジット・メモを ConsultingOS エージェントが協調して自動生成する

| クレジット・メモ | 融資 / メザニン案件の信用分析メモ作成 |

### 3.2 ConsultingOS hook 統合

水野氏向けパッケージには ConsultingOS の以下のメカニズムが標準搭載される:

- 反証チェック（CLAUDE.md ハードルール 1 物理化版）: 全アウトプット末尾に Step 1-3 + 残存リスクを付与
- FACT / INFERENCE / SPECULATION 3 ラベル: 金融分析での数値断言を構造的に防止
- brand-guardian チェック: 日本語字形 / 単位整合性 / 「断定的判断の提供禁止」（金商法準拠）
- legal-compliance-checker 事前ゲート: 投資助言業登録要否・AML/CFT 規制への該非判定

### 4.1 提供パターン

| 形態 | 内容 | 対象ユースケース |

| ハイブリッド | プラグインパック + 月次サポート + 大型案件時のスポット支援 | 推奨モデル |

### 4.2 SLA 設計

- 月次レポート納品: 毎月第 X 営業日（日程は契約時に確定）
- 大型案件対応（ピッチブック / 企業価値評価）: 依頼受領後 24 時間以内に初稿
- 緊急対応（KYC / クレジット・メモ）: 4 時間以内の初稿（平日 9:00-18:00 を標準とする、INFERENCE: 運用体制に依存するため契約時に確定）
- プレイブック月次更新: Anthropic 公式仕様変更 / 規制改正から 5 営業日以内に反映

## 5. 商品化価格設計（PL に落ちる粗利 + ブレイクイーブン）

> SPECULATION: 以下の PL 試算は仮説段階。有償パイロット 1 社の実測値取得が最優先（evolution-log.md 2026-05-04 Phase 5 設計記録より）。実際の工数・外部データソースコスト・Anthropic API コストは案件実施後に実測値で差し替える。

### 5.1 コスト構造仮説

| コスト項目 | 月次見積 | 備考 |
|---|---|---|
| Anthropic API（Claude Sonnet 4）| 2-10 万円（SPECULATION）| 案件規模・呼び出し回数に依存 |
| 外部データソース（FactSet / 帝国データバンク等）| 10-30 万円（SPECULATION）| ライセンス形態・利用頻度に依存。ライセンス条件は各社直接確認必須 |
| 運用工数（エージェント設定・品質確認）| 3-8 万円（SPECULATION）| 月次定常業務として計算 |
| 合計コスト（下限-上限）| 15-48 万円（SPECULATION）| |

### 5.2 粗利試算（ハイブリッドモデル想定）

- 売上: 月次 20-60 万円（INFERENCE）
- コスト: 15-48 万円（SPECULATION）
- 粗利: 5-12 万円 / 月（SPECULATION）
- 粗利率: 25-50%（SPECULATION）

### 5.3 ブレイクイーブン設計

初期設定コスト（プラグインパック構築 + 教育 + カスタマイズ）を仮に 50-100 万円（SPECULATION）と想定した場合:

- ブレイクイーブン: 月次粗利 5 万円のケースで 10-20 ヶ月（SPECULATION）
- ブレイクイーブン: 月次粗利 10 万円のケースで 5-10 ヶ月（SPECULATION）

「大幅に改善」ではなく「N ヶ月でブレイクイーブン」という数値化を有償パイロット後に実測値で更新する（evolution-log.md 2026-11-04 再評価カレンダー参照）。

### 5.4 PL 判断軸

- 粗利率 30% 未満は価格再設計または提供スコープ縮小が必要
- 外部データソースコストが変動大のため、初月は固定費型ライセンス交渉を優先
- API コストは案件量によりスケールするため、月次上限を SLA に明記して上振れリスクをヘッジ

## 5-B. 株式取引プロンプト 7 種（水野氏向け活用シナリオ）

> INFERENCE: 本セクションは `.claude/skills/prompt-templates/12-finance-trading.md` に収録する 7 プロンプト（T01-T07）を水野氏の投資家業務に具体的に当てはめたものである。投資助言業登録要否の判断は `legal-compliance-checker` に事前確認すること（金商法 28 条以下）。

### 活用シナリオ A: 投資先デューデリジェンス（案件評価フェーズ）

水野氏が新規投資案件を評価する際、以下の順序で 7 プロンプトを組み合わせて使用する:

| ステップ | 使用プロンプト | 目的 |
|---|---|---|
| 1 | T03 ニュース to トレード変換器 | 対象企業に関するプレス / 決算リリースを短期 / 長期インパクトに変換 |
| 2 | T02 自動化テクニカルアナリスト | 上場企業の場合は株価トレンド・モメンタムを定量評価 |
| 3 | T01 トレード・アイデア・ジェネレーター | ポジション取得のエントリー価格帯・ストップロス水準を仮設定 |
| 4 | kpi-analytics 連携 | T01-T03 の出力を EV / EBITDA マルチプルと突合して投資判断を構造化 |

出力: 投資評価メモ（proposal-writer が SCQA + Governing Thought 形式に整形）

### 活用シナリオ B: 既存ポートフォリオモニタリング（月次定例）

保有投資先の月次モニタリングに T05 / T06 / T07 を組み合わせる:

| タイミング | 使用プロンプト | 目的 |
|---|---|---|
| 月次第 1 営業日 | T05 ポートフォリオ・リスク・マネージャー | 全保有ポジションの集中度 / 弱ポジション / ストレステストを月次更新 |
| 週次（毎週月曜）| T02 自動化テクニカルアナリスト | ウォッチリスト銘柄のテクニカル状況を週次でスキャン |
| 日次（取引日）| T07 完全自動化取引計画 | 上場株の保有調整日は 4 フェーズ計画を生成してアクション規律を確保 |
| 日次（引け後）| T06 トレーディング・ジャーナル・アナライザー | 取引記録を蓄積し月次で行動バイアスをレビュー |

出力: 月次投資家レポート（kpi-analytics が月次パフォーマンス統計を集計 → proposal-writer が整形）

### 活用シナリオ C: リスクマネジメント（ポートフォリオ全体）

大規模な市場変動（VIX 急上昇 / 地政学イベント等）発生時の緊急対応フロー:

| フェーズ | 使用プロンプト | 目的 |
|---|---|---|
| 即時評価 | T05 ポートフォリオ・リスク・マネージャー | 20% 下落耐性ストレステストを緊急実施。損失想定額を即算出 |
| ニュース評価 | T03 ニュース to トレード変換器 | トリガーとなったニュースの長期影響を構造化 |
| 行動計画 | T07 完全自動化取引計画 | 翌営業日の対応方針を 4 フェーズで事前設計（感情的判断の排除）|
| 検証 | T04 戦略バックテスター | 「市場急落後に保有継続」vs「一部売却」の過去実績を検証（INFERENCE）|

IMPORTANT: 市場急変時はサーキットブレーカー / 流動性危機への対応が最優先。プロンプト出力はあくまで判断補助材料とし、実際のポジション操作は本人の意思決定に基づくこと。
