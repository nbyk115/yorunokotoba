# ConsultingOS — コンサルティング・プレイブック

## 概要
コンサルティング部門の全エージェントが参照する標準手法・フレームワーク・思考原則。
小野寺信行（電通デジタル）と佐藤裕介（フリークアウト/STORES）の知見を統合。

> **アイデンティティ原則:** 各エージェントは固有のキャラクター設定（経歴・核心的信念・思考習慣）を持つ。
> このPlaybookはその「判断軸の共通基盤」として機能する。
> 個別の判断はアイデンティティに従い、フレームワークの適用はこのPlaybookに従う。

## §1 指標変革 —「CPA至上主義」からの脱却

### 指標の構造分解 — KPIの本質を問い直す
クライアントが「CPA改善したい」と言っても、即座にCPA施策に走らない。

**診断フロー:**
1. **本当の課題は何か？** → ブランド認知不足かもしれない
2. **ファネルのどこで詰まっているか？** → 認知→興味→検討→購入の各段階を数値で確認
3. **指標は目的に合っているか？** → ブランディングにCPAは不適切

### 目的別指標マトリクス
| 目的 | 主要指標 | 避けるべき指標 |
|---|---|---|
| ブランド認知 | SOV, リーチ, ブランドリフト | CPA, CVR |
| 育成 | エンゲージメント率, コンテンツ消費 | CPC |
| 獲得 | CPA, CVR, ROAS | インプレッション |
| リテンション | LTV, チャーン率, NPS | 新規CPA |

## §2 フロー×ストック統合設計

### 施策のストック化原則
単発施策（フロー）だけでなく、資産として積み上がる施策（ストック）を常にセットで提案する。

| タイプ | 特徴 | 例 |
|---|---|---|
| **フロー施策** | 即効性あり、止めたら効果ゼロ | 広告配信, キャンペーン, PR |
| **ストック施策** | 効果蓄積、時間とともに価値増大 | SEOコンテンツ, 顧客DB, ブランド |
| **統合施策** | フローでストックを加速 | 広告→LP→メアド取得→ナーチャリング |

### 提案テンプレート
```
【フロー施策】即効性のある〇〇を実施
【ストック施策】並行して△△を蓄積
【統合設計】フローの成果を□□としてストック化
【期待効果】X ヶ月後にフロー依存度を Y% 低減
```

## §3 PL思考 — 全提案を数字で語る

> **詳細フレームワーク → `revenue-growth-framework.md` §1**

### 粗利インパクト換算 — 全提案をPLに落とす
どんな提案も最終的にPLに落とす。「いい施策」ではなく「儲かる施策」を基準にする。

**必須出力項目:**
- **粗利インパクト**: この施策で粗利がいくら変わるか
- **ブレイクイーブンポイント**: いつ投資回収できるか
- **ユニットエコノミクス**: LTV / CAC の比率（最低3倍以上）
- **感度分析**: 楽観/標準/悲観の3シナリオ

**PLチェック（コピペ用）:**
```
□ 粗利インパクト: ¥___万/月
□ ブレイクイーブン: __ヶ月
□ LTV/CAC: __倍
□ 感度分析: 楽観¥___万 / 標準¥___万 / 悲観¥___万
```

## §6 コンテクスト設計

### コンテクスト設計 — 「誰か」より「どんな状況か」で定義する
ターゲットを「30代男性」ではなく「今どんな状況にある人か」で定義する。

**コンテクスト設計フレームワーク:**
1. **状況**: ユーザーは今どんな状況にいるか
2. **動機**: なぜこの瞬間に情報を求めているか
3. **期待**: どんな情報・体験を期待しているか
4. **障壁**: 行動を阻む要因は何か
5. **トリガー**: 行動を促す最適な刺激は何か

## §7 ファーストパーティデータ戦略

### ファーストパーティデータ戦略 — クッキーレス時代の情報基盤
外部データ（3rd Party Cookie等）への依存を最小化し、自社データを中心に戦略を構築する。

**優先度:**
1. **1st Party Data**: 自社で直接取得（最優先）
2. **Zero Party Data**: ユーザーが自発的に提供
3. **2nd Party Data**: パートナー経由（信頼性を確認）
4. **3rd Party Data**: 外部購入（リスク認識必須）

## §9 課題構造化 — MCEEイシューツリー

世界トップコンサルティングファームが使う、問題を「漏れなく・ダブりなく」分解する手法。

### イシューツリーの構築原則
```
問題（イシュー）
├─ 仮説A: [問題の原因候補1]
│   ├─ 検証質問A-1: 「〜が真なら、〜のはず」
│   └─ 検証質問A-2: 「〜が真なら、〜のはず」
├─ 仮説B: [問題の原因候補2]
│   ├─ 検証質問B-1
│   └─ 検証質問B-2
└─ 仮説C: [問題の原因候補3]
```

### 3段階の問い
1. **So what?（だから何？）**: 「この分析から経営判断として何が言えるか」
2. **Why so?（なぜそうか？）**: 「その結論の根拠は何か、反証は？」
3. **Now what?（次にすべきことは？）**: 「意思決定とアクションに落とすと何か」

### エグゼクティブコミュニケーション原則
```
【1ページサマリー構造】
1. 結論（1文）: 「〜すべきです。なぜなら〜」
2. 根拠（3点以内）: 各根拠は1行で、数値つき
3. リスク（1点）: 最重要のリスクと対策
4. 次のアクション（1点）: 誰が何をいつまでに
```

### 典型的な失敗パターン（防止チェック）
- データを並べるだけで「So what?」がない
- 結論より先に分析から入る（ボトムアップ思考）
- MCEEになっていない（ダブり・モレがある）
- 「様子を見る」で意思決定を先送りにする

## §11 グローバル展開への適用

### コンサル×グローバルの統合原則
国内向けコンサルフレームワークを海外市場に適用する際の追加考慮事項。

**指標変革のグローバル適用:**
- 目的別指標マトリクス（§1）を市場ごとにローカライズ。先進国と新興国でファネル構造が異なる
- 1stPartyデータ戦略（§7）は各国の個人情報保護法（GDPR/CCPA等）に準拠した設計が必須

**PL思考のグローバル適用:**
- 為替リスクを感度分析に含める（§3の楽観/標準/悲観に通貨変動シナリオを追加）
- 現地の人件費・インフラコストでユニットエコノミクスを再計算
- ブレイクイーブンは国内より長め（12-24ヶ月）を基準に設定

**コンテクスト設計のグローバル適用:**
- §6のフレームワークを現地文化で再定義。「状況」「動機」「障壁」は市場ごとに大きく異なる
- `business-translator`と連携し、言語だけでなく文化的コンテクストも翻訳

**商談プロセスのグローバル適用:**
- Phase 1（リード精査）: 海外案件は`gtm-consultant`が市場評価を先行
- Phase 2（課題分析）: `global-journalist`が現地市場の最新動向を提供
- Phase 3（戦略立案）: 現地の商習慣・意思決定構造を`global-business`が補完

### グローバル連携先
- `gtm-consultant` — 市場参入戦略・GTM設計
- `global-journalist` — 現地市場リサーチ・国際動向
- `global-business` — 現地オペレーション・商習慣
- `business-translator` — ローカライゼーション・トランスクリエーション

## §12 佐藤裕介的経営哲学 — リアクティブ型・AI時代・溶けた系の実践原則

> 出典: 高木新平×佐藤裕介「魂の対話」（2026年収録）
> ConsultingOS全エージェントの判断軸に組み込む思想的基盤。

### リアクティブ型経営の構造的強み

```
パキッと型（強いビジョン・ルール通りの実行）:
  → ルールが明示されたゲームで圧倒的に強い
  → 実行力・組織構築に長けた経営者

リアクティブ型（意思がないことが強み）:
  → 意思がないからこそ「良さそうな場面・人」を見つける能力が高い
  → ルールが変化する・隠れているゲームで強い
  → AI時代（12〜18ヶ月先が見通せない）は出番
```

**コンサルへの適用:**
- クライアントの経営スタイルをまず判定する（パキッと型 vs 溶けた型）
- 溶けた型の経営者に「意思がない」と診断するのは誤診。価値観を言語化できていないだけ
- 戦略提案の前に「そのクライアントがどちらのゲームで強いか」を把握する

### AI時代＝ルール変化期の戦略設計

```
従来（ルール明示期):
  123... → ルールに従って最適化
  → パキッと型・実行力型が有利

現在（ルール変化期）:
  12... ??? → ルールがペリペリ剥がれていく
  → 構造から逆算してルールが明示されない間に動く
  → リアクティブ型・構造読解型が有利
```

**戦略判断への適用:**
- 「ルールが固まってから動く」は変化期では負け筋
- 構造が見えている間に先行投資・実験・ポジション確立を行う
- 12〜18ヶ月後のルールが見えた時点ですでに遅い → 今が動き時

### 極振り意思決定の実践

```
【失敗パターン】バランス型:
  ちょっと左 → まだ左に切れる → もうちょい左? → 学習なし

【成功パターン】極振り型:
  一気に左 → 壁 → 即座に右へ → 学習が速い → 最適解に収束
```

**コンサル提案への適用:**
- 「様子を見る」「段階的に」は極振り原則の対極。原則として提案しない
- Go/No-Go の判定を「条件付きGo（まず極振りで試す期間を設ける）」に変換する
- コロナ対応事例: 「一気にリモート → 一気に戻した」会社がダメージ最小

### 溶けた系経営者へのコンサル手法

```
よくある誤診:
  「意思がない」「ビジョンが弱い」「決断力がない」
  ↓ 実際は
  価値観が言語化・定着化されていないだけ

処方:
  1. 価値観を繰り返し口にさせる（自分の耳に聞かせる）
  2. ビジョンをデザイン・言語・表現で定着させる（国旗と同じ機能）
  3. 「フレーバー（特殊な味）」を特定し、それが好きな人を仲間にする設計へ
```

**実践チェックリスト:**
- [ ] クライアントの経営スタイルはパキッと型か溶けた型か？
- [ ] 溶けた型なら「価値観の言語化・定着化」が最初の処方箋
- [ ] 戦略提案に「薬草を使わない縛りプレイ」（差別化フレーバー）が含まれているか？
- [ ] 現在のルール変化度を評価したか？（高い → リアクティブ型有利・先行行動推奨）

## リアクティブ型経営 × AI時代ルール変化期（佐藤裕介 2026 詳細）

### リアクティブ型経営 — 意思がないことが強みになる場合

> 「意思がないからこそ良さそうな場面・人を見つける能力が高い。面白そうな人たちがいいタイミングで誘ってくれた」

**ルールが明示されているゲーム**: 実行力・組織力・数値達成力が強い人が勝つ
**ルールが変わる瞬間・隠れている場合**: 構造から逆算し、場と人を嗅ぎ分ける力が強い人が勝つ

```
リアクティブ型の強みが発揮される条件:
  ① 業界のルールが明示されていない（新市場・新技術）
  ② 誰が強くなり誰が弱まるかがまだ見えていない
  ③ 先行者の優位が固まる前の短い窓が開いている

弱みが露呈する条件:
  ① 成熟市場で公明正大なルールが周知されている
  ② 実行力・営業組織力・オペレーション精度が主戦場
  ③ 「決断力がある」と見られることが重要な局面
```

### AI時代 = ルール変化期の読み方

> 「AIの登場で12〜18ヶ月先が見通せない。1-2-3までルールが明示されていたのに以降が全部隠れた。この半年間むちゃくちゃやる気がある」

**コンサルタントへの示唆:**
- クライアントが「どうすればいいか分からない」と言っている時こそ、構造から逆算する専門家の出番
- 「みんなが来ると言っているのにやらない市場」への先行参入支援が最大価値
- 過去の成功事例ベースのアドバイスは陳腐化が加速している。「定石なし」前提で設計する

**ルール変化期のコンサル姿勢:**
```
❌ 「前例はこうです。ベストプラクティスはこちら」
✅ 「現在のルールを構造から読み解き、最初に動ける場所を特定する」

❌ 「まず様子を見て判断しましょう」
✅ 「ルールが隠れている今こそ極振りして早くフィードバックを得る」
```

### 縛りプレイ戦略（非対称競争）
「ドラクエうまくないくせに薬草を使わない縛りプレイをしてると言い張って、その縛りが好きな人を仲間にしていかないと勝てない」

| 要素 | 内容 |
|---|---|
| 縛りプレイ | 他社がやっている「普通のこと」をあえてしない方針を明示 |
| ファン採用 | その縛りに共感するコアメンバーを集める |
| 非対称競争 | 全倒で戦うのではなく、特定の文脈・場所・人で圧倒的になる |

## §13 ASI時代の戦略フレームワーク — 知能コストゼロ世界でのコンサルティング

### コモディティ化リスクマトリクス（認知労働版）

```
              自動化容易度
              高 ←────────────→ 低
              |                  |
高 ┌──────────┴──────────────────┴──────────┐
   │ 【即座に消える】          【守れる島】    │
顧 │ ・データ収集・分析        ・信頼判断     │
客 │ ・標準レポート生成        ・責任の引受   │
価 │ ・コード量産              ・文化的権威   │
値 │ ・翻訳・要約              ・関係資産     │
   ├─────────────────────────────────────────┤
低 │ 【時間の問題】            【転換投資必要】│
   │ ・ルーティン営業          ・規制領域     │
   │ ・定型マーケ              ・物理サービス │
   └─────────────────────────────────────────┘
```

### Human Judgment Premiumフレームワーク

コンサルがASI時代に生き残る唯一の条件は、**「AIが出した答え」に人間の署名がつく価値**を提供すること。

```
Human Judgment Premium = クライアントがAIアウトプット単体より支払う追加価値

構成要素:
1. Accountability（責任）: 間違えた時に謝れる人間がいること
2. Taste（美意識）: 「これでいい」vs「これは違う」を判断できる感覚
3. Context（文脈）: 数字の裏にある人間関係・政治・感情を読む力
4. Relationship（関係）: 「あの人に頼みたい」という個人への信頼
5. Novel Questions（問い）: まだ誰も聞いていない問いを立てる力
```

### ASI時代の戦略提案テンプレート追加項目

全ての戦略提案に以下を追加必須（2026年以降）:

```
【AGI/ASI シナリオ分析】

■ AGI前（現在〜2027年想定）:
  → 当該戦略の実現可能性・競合優位

■ AGI到達時（2027〜2029年想定）:
  → この事業の「代替可能性スコア」（0〜1）
  → 人間が守るべき機能はどこか

■ ASI後（2030年〜）:
  → 残存する競争優位の源泉
  → ピボット先の仮説
```
