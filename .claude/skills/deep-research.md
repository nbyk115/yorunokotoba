# Deep Research — 多ソース深掘り調査スキル

## 概要
15〜30のユニークソースを組み合わせた、証拠ベースの調査レポートを生成する。
単一検索ではなく、複数角度からの情報収集と徹底的なクロスリファレンスが核心。
出典: affaan-m/everything-claude-code (deep-research)

---

## §1 起動トリガー

以下のいずれかに該当する場合に本スキルを適用する:
- 「詳しく調べたい」「深掘りしたい」「現状を把握したい」
- 競合・市場・技術の in-depth 分析要求
- デューデリジェンス・投資判断材料の収集
- 「〜の最新動向は？」「〜の実態は？」

---

## §2 調査プロセス

### Step 1: 意図の明確化
```
- 目的を特定: 学習 / 意思決定 / コンテンツ作成
- ゴールを1文で定義してから調査開始
```

### Step 2: 問いの分解
```
トピックを3〜5つのサブ問いに分解する
例「SaaS市場の競合分析」→
  1. 主要プレーヤーと市場シェアは？
  2. 価格・機能の差別化ポイントは？
  3. 直近12ヶ月の動向・ニュースは？
  4. 参入障壁と撤退事例は？
  5. 顧客の声・レビューはどうか？
```

### Step 3: 多ソース検索
```
各サブ問いに対して:
- キーワードの2〜3バリエーションで検索
- 合計15〜30ユニークソースを目標
- 情報源の種類を混在させる:
  公式サイト / ニュース / 業界レポート / 学術論文 / フォーラム / レビューサイト
```

### Step 4: 深読み
```
3〜5つの主要ソースを完全に精読する（スキャンではなく全文読み）
→ 見出し・数値・引用の文脈を正確に把握
```

### Step 5: 統合・レポート生成
```
テーマ別に整理:
- 事実は全てソース帰属を付ける
- 複数ソースでクロスリファレンスする
- 最新情報（直近12ヶ月）を優先する
```

---

## §3 品質基準

| 基準 | 内容 |
|---|---|
| 全主張にソース帰属 | 「〜によると」「〜（出典: URL）」を必ず付ける |
| クロスリファレンス | 重要な事実は複数ソースで確認する |
| 最新性 | 直近12ヶ月の情報を優先、古い情報は日付を明記 |
| 未検証の明示 | 「未確認」「推定」「〜と言われているが未検証」を明示 |
| 事実と予測の分離 | 「現状〜だ」と「〜と予測される」を明確に区別 |
| ソースなし主張禁止 | 根拠なき断言は禁止 |

## §4 レポートフォーマット

```markdown
## エグゼクティブサマリー
[3〜5文。最重要な発見のみ]

## [テーマ1]
[分析内容] (出典: [ソース名/URL])

## [テーマ2]
...

## 主要タケアウェイ
1. [最重要な洞察]
2. [2番目の洞察]
3. [3番目の洞察]

## 不確実性・データギャップ
- [確認できなかった情報]

## ソース一覧（概要付き）
1. [ソース名] - [URL] - [概要1文]
...

## 調査方法論
[検索キーワード・使用ツール・調査期間]
```
