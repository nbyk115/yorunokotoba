# Planning with Files — ファイルベース計画管理スキル

## 概要
コンテキストウィンドウをRAM（揮発性・有限）、ファイルシステムをDisk（永続性・無限）として扱う、
Manusスタイルのファイルベース計画システム。複雑な多ステップタスクの文脈を永続化する。
出典: B0Qi/skill-planning-with-files, OthmanAdi/planning-with-files

---

## §1 コアコンセプト

```
Context Window = RAM（揮発性・限定）
Filesystem     = Disk（永続性・無限）

→ 重要な情報は必ずファイルシステムに書き出す
→ セッションが切れても作業を再開できる状態を保つ
```

### 使用条件
| 適用する | 適用しない |
|---|---|
| 3ステップ以上の複合タスク | 単純な質問への回答 |
| リサーチプロジェクト | 1ファイルの編集のみ |
| 大規模な開発作業 | 単発のコマンド実行 |
| 多くのツール呼び出しが必要なタスク | |

## §2 ディレクトリ構造

```
.claude-plans/
├── index.md                    ← 中央タスクレジストリ（現在フォーカスのトラッカー）
└── YYYY-MM-DD-<task-slug>/
    ├── plan.md                 ← タスク計画・目標・アプローチ
    ├── findings.md             ← 調査結果・重要な発見
    └── progress.md             ← 進捗ログ・完了ステップ・ブロッカー
```

### index.md フォーマット
```markdown
# タスクインデックス

## 現在のフォーカス
- タスク: [アクティブなタスク名]
- ディレクトリ: .claude-plans/YYYY-MM-DD-[slug]/
- ステータス: 進行中

## タスク履歴
| タスク | ディレクトリ | ステータス | 完了日 |
|---|---|---|---|
| [タスク名] | .claude-plans/... | 完了 | YYYY-MM-DD |
```

## §3 クリティカルルール

### 2-アクションルール
```
Web検索・ファイル読み込み・ブラウザ操作を2回行うたびに
→ 直ちに findings.md に重要な発見を保存する
（記憶に頼らない。コンテキストは揮発する）
```

### 意思決定前の再読み込み
```
重大な決定をする前に必ず plan.md を再読する
→ 目標からの逸脱を防ぐ
```

### エラーログの義務
```
失敗したアクションは progress.md に記録する
→ 同じ失敗を繰り返さない
```

### 3ストライクプロトコル
```
同じアクションを3回試みて失敗した場合:
→ 同じアプローチで再試行しない
→ 根本原因を特定し、代替アプローチを計画する
```

---

## §4 セッション開始・継続プロセス

### 新規タスク開始時
```
1. .claude-plans/ ディレクトリを確認
2. index.md にタスクを登録
3. YYYY-MM-DD-<slug>/ ディレクトリを作成
4. plan.md に目標・アプローチ・成功基準を記載
5. 作業開始
```

### セッション再開時（/clear後・翌日等）
```
1. .claude-plans/index.md を読む
2. アクティブタスクのディレクトリを特定
3. plan.md → findings.md → progress.md の順に読む
4. 最後の完了ステップから再開
```

---

## §5 ファイル書き込みルール

### findings.md への記録タイミング
- 2回のツール操作後
- 重要な事実・数値・URLを発見したとき
- 仮説が証明/棄却されたとき

### progress.md への記録タイミング
- ステップ完了時
- ブロッカーに遭遇したとき
- 方針転換したとき

### plan.md の更新タイミング
- スコープが変わったとき
- 前提が覆ったとき
- 新しい制約が判明したとき
