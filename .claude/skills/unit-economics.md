## 1. 基本ユニットエコノミクス計算

### LTV（顧客生涯価値）

```
LTV = ARPU × 粗利率 × 平均継続期間

より精密な計算:
LTV = Σ(月次粗利 × 生存率) [全期間]

簡易版（SaaS）:
LTV = ARPU × 粗利率 / チャーンレート

例:
  ARPU: ¥15,000/月
  粗利率: 70%
  月次チャーンレート: 3%
  → LTV = ¥15,000 × 70% / 3% = ¥350,000
```

### CAC（顧客獲得コスト）

```
CAC = (マーケ費用 + 営業費用) / 新規獲得顧客数

チャネル別CAC（必ず分解する）:
  広告CAC = 広告費 / 広告経由の獲得数
  オーガニックCAC = コンテンツ制作費 / オーガニック獲得数
  営業CAC = 営業人件費 / 営業経由の獲得数
  紹介CAC = 紹介プログラム費 / 紹介獲得数

例:
  月次広告費: ¥3,000,000
  月次広告経由獲得: 100件
  → 広告CAC = ¥30,000

  月次営業人件費: ¥5,000,000（5名×100万）
  月次営業経由獲得: 20件
  → 営業CAC = ¥250,000
```

### LTV/CAC 判定基準

```
LTV/CAC ≥ 3x    → 健全（事業継続・投資加速）
LTV/CAC = 2-3x  → 要改善（CAC削減またはLTV向上が必要）
LTV/CAC < 2x    → 危険（抜本的見直し必須）
LTV/CAC ≥ 5x    → 投資加速ゾーン（最速で獲得を増やす）

AI Agent Company基準:
LTV/CAC ≥ 5x    → 標準（AI自動化でCAC構造的に低いはず）
LTV/CAC < 3x    → 人件費モデルと変わらない（AI導入の意味がない）
```

### Payback Period（投資回収期間）

```
Payback Period = CAC / (ARPU × 粗利率)

判定基準:
Payback ≤ 12ヶ月  → 健全
Payback 12-18ヶ月 → 要改善
Payback > 18ヶ月  → 危険（資金繰りリスク高）

例:
  CAC: ¥30,000
  ARPU: ¥15,000/月
  粗利率: 70%
  → Payback = ¥30,000 / (¥15,000 × 70%) = 2.9ヶ月 ✅
```

## 2. SaaS特化ユニットエコノミクス

### MRR分解ツリー

```
MRR
├─ New MRR（新規獲得）
│   = 新規顧客数 × 平均ARPU
├─ Expansion MRR（アップセル・クロスセル）
│   = 既存顧客数 × アップセル率 × 追加ARPU
├─ Contraction MRR（ダウングレード）
│   = 既存顧客数 × ダウングレード率 × 削減ARPU（マイナス）
└─ Churned MRR（解約）
    = 先月顧客数 × チャーンレート × 平均ARPU（マイナス）

Net MRR Growth = New + Expansion - Contraction - Churned

Net Revenue Retention (NRR):
NRR = (先月MRR + Expansion - Contraction - Churned) / 先月MRR × 100%
NRR > 100%  → Expansion > Churn（プロダクト主導の成長）
NRR = 100%  → Churnを埋めているだけ
NRR < 100%  → 事業縮小
```

### コホート分析テンプレート

```
月次チャーンコホート分析:

| 獲得月 | M0 | M1 | M3 | M6 | M12 |
|---|---|---|---|---|---|
| 2025-01 | 100% | 90% | 75% | 60% | 45% |
| 2025-02 | 100% | 88% | 72% | | |
| 2025-03 | 100% | 91% | | | |

→ M12生存率が45%の場合:
  平均継続期間 = 1/月次チャーン率 = 1/0.0459 ≒ 21.8ヶ月
  LTV = ARPU × 粗利率 × 21.8ヶ月
```

## 3. AI Agent Company 固有指標

### エージェントROI計算

```
Agent ROI = (人件費節約額 - AI運用コスト) / AI運用コスト × 100%

人件費節約額 = 自動化された業務時間 × 人件費単価/時間

AI運用コスト:
  = API費用 + インフラ費用 + 保守人件費

例:
  自動化業務: リサーチ・レポート作成（月200時間）
  人件費単価: ¥5,000/時
  人件費節約: ¥1,000,000/月

  API費用: ¥50,000/月
  インフラ: ¥30,000/月
  保守: ¥100,000/月（1名×0.1FTE）
  AI運用コスト: ¥180,000/月

  Agent ROI = (¥1,000,000 - ¥180,000) / ¥180,000 × 100% = 456% ✅
```

### 自動化率トラッキング

```
自動化率 = AI処理タスク数 / 全タスク数 × 100%

目標ベンチマーク（AI Agent Company）:
  繰り返し業務: 80%以上自動化
  調査・分析: 60%以上自動化
  意思決定支援: 40%以上自動化
  クリエイティブ: 30%以上自動化

AI Cost per Action（アクションあたりAIコスト）:
  CPA_AI = 月次AI運用コスト / 月次自動処理タスク数

  例:
    月次AI運用コスト: ¥200,000
    月次処理タスク: 2,000件
    → CPA_AI = ¥100/件

  人間の場合:
    担当者1名（¥500,000/月）× 処理能力200件/月
    → 人件費per件 = ¥2,500/件

  AI優位性 = ¥2,500 / ¥100 = 25倍のコスト効率
```

|---|---|---|---|---|---|

| 紹介/バイラル | | | | | |
| パートナー | | | | | |

→ LTV/CAC最大かつスケール可能なチャネルに予算を集中
→ LTV/CAC < 2xのチャネルは即停止を検討
```

### マーケティング効率指標

```
ROAS（Return on Ad Spend）:
  ROAS = 広告経由売上 / 広告費
  目標: ROAS ≥ 3x（粗利率50%の場合の損益分岐点）

真のROAS（粗利ベース）:
  真のROAS = 広告経由粗利 / 広告費
  目標: 真のROAS ≥ 1.5x

獲得コスト効率指標:
  MER（Marketing Efficiency Ratio）= 総売上 / 総マーケ費用
  ブレンドCAC = 総マーケ費用 / 総新規獲得数
```

## 6. 成長ステージ別チェックポイント

### Stage 0: PMF探索（〜月次売上¥500万）
```
重点指標:
  □ NPS > 40（熱狂的ファンが存在するか）
  □ 解約理由の80%が「機能不足」以外
  □ 自然発生的な紹介が月1件以上
  □ LTV/CAC計測可能な状態に到達
```

### Stage 1: 初期スケール（月次売上¥500万〜¥3000万）
```
重点指標:
  □ LTV/CAC ≥ 3x（獲得コスト回収見込み）
  □ Payback ≤ 12ヶ月
  □ MoM成長率 ≥ 15%
  □ NRR ≥ 100%（解約以上のExpansion）
  □ 主力獲得チャネルが1つ以上確立
```

### Stage 2: スケールアップ（月次売上¥3000万〜¥1億）
```
重点指標:
  □ LTV/CAC ≥ 4x
  □ NRR ≥ 110%（既存顧客が自己成長）
  □ ブレンドCAC改善トレンド（規模効果）
  □ 2チャネル以上で再現性ある獲得
  □ 自動化率 ≥ 50%（AI Company の場合）
```

### Stage 3: 安定成長（月次売上¥1億〜）
```
重点指標:
  □ NRR ≥ 120%
  □ オーガニック・紹介が獲得の40%以上
  □ 新プロダクト/市場への投資余力（営業利益率20%+）
  □ プロダクトバリュー次世代化計画の実行中

## 7. 計算ショートカット（クイック判断用）

```
SaaS健全性クイックチェック（3分で判断）:

1. 粗利率は？
   < 50% → 変動費構造を見直す
   50-70% → 標準的
   > 70% → 優秀

2. NRRは？
   < 90% → 緊急（解約止めが最優先）
   90-100% → 要改善
   > 110% → 投資加速

3. CAC回収は12ヶ月以内か？
   Yes → 獲得を加速
   No → CAC削減 or ARPU向上の施策が先

4. LTV/CACは3倍以上か？
   Yes → 健全。スケール投資可能
   No → ユニットエコノミクス改善が先。スケールするほど赤字拡大

5. MoM成長率は？
   < 5% → 成長が止まっている（PMF再確認）
   5-15% → 標準
   > 20% → 高成長（NRRとCAC品質を維持しながらスケール）
```

## 8. AI Agent Company 固有の経営判断基準

佐藤裕介の知見をAI Agent Company文脈に適用:

```
意思決定フィルター（全施策・全投資に適用）:

1. 「これは資産になるか？」
   → コンテンツ・データ・ネットワーク・プロダクトが蓄積されるか
   → 止めたらゼロになる消耗施策は後回し

2. 「AIで自動化できるか？」
   → 人がやる必然性のある業務は何か（AIにできない部分に人を集中）
   → 6ヶ月後にAI自動化できる業務に今人を採用するな

3. 「プロダクトが売る構造か？」
   → 営業個人に依存した獲得は再現性がない
   → フリートライアル・PLG・ウイルス係数で自走する設計か

4. 「2年後に陳腐化するか？」
   → 現在の価値提案の賞味期限を必ず設定する
   → 次のバリュー（Phase N+1）への移行計画が存在するか

5. 「グローバルスケールできるか？」
   → 日本市場だけでユニットエコノミクスが成立するか確認
   → グローバル展開でどれだけLTVが伸び、CACが下がるか試算
```
