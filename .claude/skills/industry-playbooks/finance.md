# 金融業界プレイブック

> ⚠️ **ハルシネーション禁則 + 法的助言不可**: 本プレイブックは**法的助言ではない**。金融業界は規制が極めて厳しく、提案・分析時は必ず**法務・コンプライアンス部門 + 弁護士のレビュー**を経ること。数値・規制内容は**必ず一次ソース（金融庁・各業界団体）で最新版を確認**。

## 1. 業界構造

### 業種分類
- **銀行**: メガバンク / 地方銀行 / 信用金庫 / ネット銀行
- **証券**: 大手証券 / ネット証券 / 投信運用
- **保険**: 生命保険 / 損害保険 / 共済
- **資産運用**: 投資信託 / プライベートバンキング / VC / PE
- **決済・キャッシュレス**: クレジットカード / QR コード決済 / 電子マネー
- **暗号資産・Web3**: 暗号資産交換業者 / NFT
- **FinTech**: ネオバンク / 与信 SaaS / 会計 SaaS / マネーフォワード等

### 業界構造の特徴
- **規制業種**: 金融庁の許認可が必須。ほぼすべての業務に規制
- **顧客資産の受託**: 信託責任・受託者責任が重い
- **システム障害許容度ゼロ**: ATM・取引システム停止は社会問題化
- **AML/CFT**: マネロン・テロ資金供与防止が継続課題
- **DX 後発**: 規制とレガシーシステムで DX が他業界より遅れる傾向
- **FinTech との協業 / 競合**: 銀行 API 開放（電子決済等代行業）、BaaS 化進行

## 2. 業界固有 KPI

> ⚠️ 銀行・証券・保険で KPI 体系が大きく異なる。**断定数値の引用禁止**。各社 IR / 業界白書で最新値確認。

### 銀行
- **ROE / ROA**: 自己資本利益率・総資産利益率
- **NIM**（Net Interest Margin / 預貸利ざや）
- **OHR**（Overhead Ratio）: 経費率
- **不良債権比率（NPL Ratio）**
- **自己資本比率**: バーゼル III 規制（最低水準あり）
- **預貸率 / 預証率**: 資金運用バランス

### 証券
- **AUM**（Assets Under Management）: 預かり資産
- **取引高 / 手数料収入**
- **アクティブ口座数 / 新規口座開設**
- **ROE**

### 保険
- **コンバインドレシオ**（損保）: 損害率 + 経費率
- **ソルベンシーマージン比率**: 健全性指標
- **新契約価値（NBV）**（生保）
- **解約率**

### FinTech / 決済
- **MAU / DAU**
- **決済取扱高（GMV / Volume）**
- **アクティブユーザー数**
- **テイクレート**（手数料率）
- **CAC / LTV**

## 3. 業界規制・コンプライアンス（極めて重要）

> ⚠️ 金融規制違反は**業務停止命令・行政処分・刑事罰**の対象。本プレイブックの内容は**最新の金融庁ガイドラインで必ず確認**。

### 共通規制
- **金融商品取引法（金商法）**: 証券業・投資助言・ファンド運用全般
- **銀行法**: 銀行業
- **保険業法**: 保険業
- **資金決済法**: 暗号資産・前払式支払手段・資金移動業
- **割賦販売法**: クレジットカード・包括信用購入あっせん
- **貸金業法**: 消費者金融・カードローン

### マネロン・テロ資金（AML/CFT）
- **犯罪収益移転防止法**: 本人確認・疑わしい取引届出
- **FATF 勧告**: 国際基準、日本は審査対象
- **金融庁 AML/CFT ガイドライン**

### 顧客本位の業務運営（Fiduciary Duty）
- **金融庁「顧客本位の業務運営に関する原則」**
- **FD（Fiduciary Duty）原則**

### システム・サイバーセキュリティ
- **FISC 安全対策基準**: 金融機関の情報システム安全対策
- **金融庁 サイバーセキュリティガイドライン**
- **データ保護**: 個人情報保護法 + 金融分野の特則

### 広告・募集
- **金商法 適合性原則**: 顧客属性に応じた商品提案
- **断定的判断の提供禁止**: 「絶対儲かる」等は禁止
- **誇大広告禁止**: 不確実な事項を確実と誤認させる表示禁止

### FinTech 特有
- **電子決済等代行業者登録**: 銀行 API 接続事業者
- **資金移動業ライセンス**: 送金事業者
- **暗号資産交換業者登録**: 暗号資産取引所
- **改正資金決済法**: ステーブルコイン規制

## 4. 商習慣・契約慣行

### 営業プロセス
- **適合性原則**: 顧客の知識・経験・財産・取引目的に応じた商品提案
- **重要事項説明**: 法定書面の交付・説明義務
- **クーリングオフ**: 一定期間の契約解除権
- **個別対応**: 大口顧客・機関投資家向けは個別契約

### システム導入
- **長期検証**: 半年〜2年の POC・並行運用
- **規制対応**: 金融庁ヒアリング・FISC 監査
- **ベンダー選定**: 入札 + 多重審査
- **支払サイト**: 月末締め翌月末払い等が標準

### コンプライアンス
- **三線防御**: 第1線（営業）/ 第2線（コンプライアンス・リスク管理）/ 第3線（内部監査）
- **Fit and Proper**: 役員の適格性審査（金融庁）
- **行動規範**: 利益相反管理 / インサイダー取引防止

## 5. 典型的な失敗パターン

### 失敗 1: 規制違反による業務停止命令
- 不適切販売（顧客本位違反）/ AML 体制不備 / システム障害放置
- → legal-compliance-checker と金融庁ガイドラインの定期確認

### 失敗 2: システム障害の社会問題化
- 大規模障害でメディア・国会で取り上げられる事態
- → ヴォーゲルス Frugal Architect / 障害前提設計 / 多重化

### 失敗 3: AML/CFT 不備による FATF 評価低下
- 国際送金・暗号資産で疑わしい取引を放置
- → AI による継続モニタリング・SAR（疑わしい取引届出）の体制整備

### 失敗 4: FinTech との競争で本業侵食
- 決済・送金・与信で FinTech に顧客流出
- → 銀行 API 開放・BaaS 化・FinTech 協業

### 失敗 5: 「絶対儲かる」表現での金商法違反
- 投資商品の広告・説明で断定的判断を提供
- → 全マーケ素材を法務・コンプライアンス審査

## 7. 提案・分析時のチェックリスト（金融特化）

### 提案前（重要）
- [ ] **金融庁監督指針** の最新版確認
- [ ] **業態ライセンス**確認（銀行業 / 金商業 / 保険業 / 資金移動業等）
- [ ] **法務・コンプライアンス部門**との事前調整
- [ ] AML/CFT 体制の確認

### 提案中
- [ ] **適合性原則**遵守の設計（顧客属性に応じた商品）
- [ ] **断定的判断の提供禁止**（「絶対儲かる」「100%安全」等の禁止表現チェック）
- [ ] **重要事項説明**の組み込み
- [ ] **個人情報保護**＋ **金融分野特則**遵守
- [ ] **FISC 安全対策基準**準拠（システム提案時）

### 提案後
- [ ] 行政指導・規制変更の継続ウォッチ
- [ ] 顧客本位の業務運営方針との整合性
- [ ] 内部監査・第2線レビューの実施
- [ ] AML/CFT モニタリングの継続

### 8.2 6 ワークフロー標準実装手順

> 各ワークフローは法務・コンプライアンスレビュー必須（金融業法 / 金商法 / 投資助言業登録要否は legal-compliance-checker で事前確認）。

#### 8.2.1 ピッチブック（Pitchbook）

1. `proposal-writer` が SCQA ブロック + Governing Thought を作成
2. `competitive-analyst` が M&A 対象企業の業界ポジション・競合比較表を生成
3. Claude 金融エージェントが財務データ（FactSet / S&P Global / LSEG）を自動取得し Excel 連携でモデル更新
4. `kpi-analytics` が EV / EBITDA 倍率・PER・PBR のマルチプル分析を実施
5. `sales-deck-designer` が PowerPoint に出力（1 スライド = 1 メッセージ原則）
6. `legal-compliance-checker` が「断定的判断の提供禁止」「適合性原則」準拠を確認

#### 8.2.2 企業価値評価（Valuation）

1. Claude 金融エージェントが DCF / LBO / Comparable Companies の 3 手法を並列実行
2. 財務データソース: FactSet（株価・財務情報）/ S&P Global（格付け・クレジット）/ Morningstar（ファンド・ETF）
3. `kpi-analytics` がブレイクイーブン・感応度分析（WACC ±1% / 成長率 ±0.5%）を試算
4. 出力: Excel モデル + PowerPoint サマリー（Word 版エグゼクティブサマリー付属）

#### 8.2.3 KYC（本人確認・顧客デューデリジェンス）

1. `legal-compliance-checker` が犯罪収益移転防止法・FATF 勧告の最新要件を確認
2. Claude 金融エージェントが顧客属性フォームを自動生成（帝国データバンク / 東京商工リサーチ連携で法人与信確認）
3. AML/CFT スコアリング: 疑わしい取引パターンを ML モデルで継続モニタリング
4. Outlook 連携で SAR（疑わしい取引届出）ドラフトを自動生成 → 担当者承認フローへ

#### 8.2.4 月次決算（Financial Close）

1. Claude 金融エージェントが Excel 財務モデルと連携し、前月比・前年同期比・予実差異を自動算出
2. `kpi-analytics` が ROE / NIM / OHR 等の業界 KPI を自動集計
3. Word テンプレートに月次レポートを生成（経営会議向けエグゼクティブサマリー形式）
4. `brand-guardian` が字形・トーン・数値単位の整合性を最終チェック

#### 8.2.5 アクチュアリー・レビュー（Actuarial Review）

1. Claude 金融エージェントがソルベンシーマージン比率・NBV 等の保険特有 KPI を算出
2. 死亡率テーブル / 解約率モデルをデータソース（S&P Global / Morningstar）から取得して更新
3. `legal-compliance-checker` が保険業法・ソルベンシー II 相当の規制要件を確認
4. レポートは Word + Excel 形式で保険庁提出フォーマットに準拠（INFERENCE: 最新監督指針で要確認）

#### 8.2.6 クレジット・メモ（Credit Memo）

1. `strategy-lead` が対象企業の事業リスク・業界環境を分析
2. Claude 金融エージェントが財務比率（D/E レシオ / ICR / FCF マージン）を FactSet から自動取得
3. `legal-compliance-checker` が貸金業法・銀行法・信用リスク管理ガイドライン準拠を確認
4. 出力: Word クレジット・メモ + Excel 財務サマリー + Outlook 承認フロー連携
