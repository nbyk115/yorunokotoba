# Strategic Compact — 戦略的コンテキスト圧縮スキル

## 概要
自動圧縮（任意の地点でタスクを中断する）に依存せず、タスクの論理的な区切り目で
意図的に `/compact` を実行するコンテキスト管理戦略。
長時間セッションでの文脈喪失リスクを最小化し、重要情報を確実に保持する。
出典: affaan-m/everything-claude-code (strategic-compact)

---

## §1 コアコンセプト

```
自動圧縮 = 任意のタイミングで中断 → 実装中に変数名・ファイルパスが消える
戦略的圧縮 = フェーズの区切り目で意図的に実行 → 必要な文脈を保持したまま圧縮
```

### 圧縮後に残るもの（永続）
- CLAUDE.md の指示
- TodoWrite のタスクリスト
- `.claude-plans/` のファイル（planning-with-filesと組み合わせて最強）
- `.claude/memory/` のメモリファイル
- git の状態・コミット履歴

### 圧縮後に消えるもの（揮発）
- 会話履歴・中間推論
- ファイル内容のキャッシュ
- 変数名・関数名の文脈
- デバッグ過程のログ

---

## §2 圧縮すべきタイミング

### ✅ 圧縮してよいタイミング
| タイミング | 理由 |
|---|---|
| リサーチ → 計画 の移行時 | 調査結果はファイルに保存済み。会話履歴は不要 |
| デバッグセッション完了後 | 解決済みのデバッグログは邪魔になる |
| 主要マイルストーン達成後 | Phase 1完了 → Phase 2開始前 |
| タスク切り替え時 | 全く異なるトピックに移る前 |
| 200K以上のトークン使用時 | 応答品質の低下が始まる前に |

### ❌ 圧縮してはいけないタイミング
| タイミング | 理由 |
|---|---|
| 実装の途中 | 変数名・ファイルパスが消えてコードが壊れる |
| 複数ファイルをまたぐリファクタリング中 | 依存関係の文脈が必要 |
| エラーのデバッグ中（原因未特定） | 調査過程が消えると振り出しに戻る |

---

## §3 圧縮前チェックリスト

```
圧縮実行前に必ず確認:
□ 重要な発見・数値は .claude-plans/findings.md に書き出したか
□ 現在のTODOは TodoWrite に記録済みか
□ 次フェーズに必要なファイルパス・関数名はメモしたか
□ 実装中のコードは全て保存・コミット済みか
□ 未解決のエラーや調査中の仮説はファイルに記録したか
```

## §4 コンテキスト最適化（予防策）

### 肥大化を防ぐ日常的な習慣
```
1. スキルのレイジーロード
   → 全スキルをCLAUDE.mdに列挙しない。必要なときだけ参照

2. 重複指示の排除
   → グローバルとローカルのCLAUDE.mdで同じことを繰り返さない

3. 50ツール呼び出しルール
   → 50回のツール呼び出し後に圧縮を検討する
   （settings.json の COMPACT_THRESHOLD で調整可）

4. planning-with-files との連携
   → 調査結果は会話内に持ち続けず、即座にファイルに書き出す
```

### ConsultingOSでのコンテキスト消費が多い操作
```
高消費:
- WebSearch × 10回以上の並列調査
- 大量のエージェント出力の統合
- 長大なコードベースの読み込み

対策:
- 調査系タスクは .claude-plans/findings.md に即書き出し
- エージェント出力はサマリーのみ保持
- コードはGlobで必要なファイルのみ読む
```

|---|---|
| orchestrator | 多フェーズ案件でフェーズ間に圧縮を挟む |
| competitive-analyst | リサーチ完了後・分析開始前に圧縮 |
| tech-lead | 設計フェーズ完了後・実装フェーズ開始前に圧縮 |
| fullstack-dev | 機能実装完了後・次機能開始前に圧縮 |

## §6 planning-with-filesとの統合パターン

```
【最強の組み合わせ】
planning-with-files（永続化） × strategic-compact（文脈管理）

1. タスク開始: .claude-plans/{task}/plan.md に計画を書く
2. 実行中: 2アクションごとに findings.md に書き出す（planning-with-files §3）
3. フェーズ完了: 圧縮前チェックリストを実行 → /compact
4. 再開時: .claude-plans/index.md を読んで文脈を復元
→ コンテキスト消費 30-50% 削減、文脈喪失ゼロ
```
