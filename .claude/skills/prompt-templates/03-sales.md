# 提案書・セールスプロンプト集（5本）

## P11. B2B 商談資格化（MEDDPICC 完全版）

```
あなたは proposal-writer エージェントです。
.claude/agents/consulting/proposal-writer.md「フレームワーク」に従ってください。

[商談情報]:
- クライアント: [社名・業界・規模]
- 接触経緯: [リード元 / 初回接触日]
- 接触担当: [窓口担当の役職]
- 想定案件規模: [金額レンジ]

以下を MEDDPICC で資格化:

1. Metrics（定量化された成果指標）
   - 「成功した時、何の数字がどう動いていれば OK?」を質問・確定

2. Economic Buyer（予算決裁の最終権限者）
   - 「最終的に予算を承認するのはどなた?」
   - 窓口担当 ≠ Economic Buyer の場合の接触戦略

3. Decision Criteria（意思決定の判断基準）
   - 「複数案を比較する時、何で評価しますか?」

4. Decision Process（意思決定プロセス）
   - 「契約までに何ステップ・誰の承認が必要?」

5. Identify Pain（痛みの数値化）
   - 「この課題で月いくら損失? 何時間ロス?」

6. Champion（社内推進者）
   - 「社内で誰が一番この課題に困ってる?」
   - Champion 行動評価 L0/L1/L2/L3 のどれか

7. Paper Process（契約・調達プロセス）
   - 法務 / 調達 / セキュリティレビュー / NDA / MSA

8. Competition（競合）
   - 他ベンダー / 内製 / 現状維持

最初の30分で確定する3つの数字:
- Pain の月次損失額
- 予算の桁感
- 意思決定までの月数

出力形式: 各項目を表で。確定 / 不明 / 要追加ヒアリングを明示。
受注確率（%）と次アクションを最後に提示。
```

## P12. セールスデッキ作成（hotice 18P 構成）

```
あなたは sales-deck-designer エージェントです。
.claude/agents/creative/sales-deck-designer.md と examples/proposal-template/structure.md
に従ってください。

[クライアント]: [社名・業界]
[ピッチの目的]: [初回提案 / コンペ / 二次提案 / etc.]
[時間制約]: [プレゼン時間]

18P 構成で各ページの中身を設計:

静（導入）
- P1 カバー: 「X を Y に変える」型タグライン
- P2 自己紹介: ロゴ + 一行紹介
- P3 業界構造の現状: 数値 + 出典 URL + 発行日

知（課題）
- P4 クライアントの現状仮説
- P5 真の課題（指標を疑う / 小野寺式）
- P6 リスク（PL インパクト数値化）

強（差別化）
- P7 ポジショニング（敵地で戦わない）
- P8 競合比較（自社が勝つ / 全社で差分が出る軸のみ）
- P9 独自の強み（解釈した強み / 領域特化）

信（実績）
- P10 ケース（公開可能・上場企業は匿名化）
- P11 数字実績（出典・条件明記）
- P12 顧客の声（許諾済みのみ）

実務（解決策）
- P13 解決策の全体像（フロー × ストック）
- P14-15 具体施策 1-2
- P16 体制・タイムライン

決（クロージング）
- P17 PL 試算（粗利・ブレイクイーブン・前提条件）
- P18 次のアクション

各ページのコピー:
- 二項動詞型タグライン
- 抽象語（気づき・体験・共感）禁止
- 業界略語展開（MCN → クリエイター事務所 等）
- 上から目線禁止
- 最上級の無根拠使用禁止

出力末尾に反証チェック結果。
```

## P13. 競合比較表（敵地で戦わない）

```
あなたは sales-deck-designer エージェントです。
examples/proposal-template/competitive-comparison.md に従ってください。

[自社の領域]: [業界・領域を限定]
[競合]: [社名 4-6社]

設計原則:

1. 載せる軸の選定
   - 自社が勝つ軸
   - 全社で差分が出る軸（自社のポジションが読み取れる）
   - 「非公表 / 要相談」が半数以上の行は削除

2. 商品名 < 解釈した強み
   - ❌ 「ツール A / 機能 B を保有」
   - ✅ 「[領域] での独自データ蓄積」「[領域] での運用力」

3. ポーター式 Focus 戦略
   - 競合カテゴリによって比較領域を限定
   - 例: 総合広告グループとは「インフルエンサー領域での」強みに限定

4. 自社弱点を自分から surface しない
   - 「留意点」「今後の課題」「改善中」行を自発的に作らない

5. ティール 独占性ゲート（B2B SaaS）
   - 10x 優位 / 独占の4要素（最低2つ）/ Last Mover / Secret thesis

出力形式:
| 軸 | 自社 | 競合A | 競合B | 競合C |
|---|---|---|---|---|
| ... | ◎ ... | △ ... | ○ ... | × ... |

凡例: ◎ 強み / ○ 同等 / △ やや弱い / × 弱い（自社の × 行は載せない）

出力末尾に反証チェック結果（公式 / IR / 主要紙ソース確認含む）。
```

## P14. 提案前ヒアリング設計

```
あなたは proposal-writer エージェントです。
クライアントとの初回ヒアリングを設計してください。

[クライアント]: [社名・業界]
[初回 MTG 時間]: [30分 / 60分]

ヒアリング設計原則:
- 最初の5分で「解決したい課題のトップ3」を聞く
- クライアントに8割話させる設計
- 予算・権限・時期は直接聞かず、状況から推測

質問テンプレ（最初の30分で確定する3つの数字）:

1. Pain の月次損失額
   - 「この課題で月いくら損失していますか? 何時間ロスしてますか?」

2. 予算の桁感
   - 「同類施策にこれまでどれくらい投資してきましたか?」（直接聞かず推測）

3. 意思決定までの月数
   - 「Why now? なぜ今このタイミングで動いている?」
   - 「半年前ではダメだった理由は? 半年後ではダメな理由は?」

Champion 育成の質問:
- 「この課題、社内で誰が一番困ってる?」
- 「もしこれを解決できたら、その人の評価はどう変わる?」
- 「Economic Buyer に説明する時、何があれば説得力が増す?」

NG 質問:
- ❌ 「ご予算は?」
- ❌ 「決裁権者は?」
- ❌ 自社が話しすぎる（7割超）

ヒアリング結果テンプレ:
- BANT 4軸の初期評価
- MEDDPICC 8項目の確定 / 不明
- Champion 行動評価 L0/L1/L2/L3
- 受注確率と次アクション

出力末尾に反証チェック結果。
```

## P15. クロージング資料作成

```
あなたは proposal-writer + client-success エージェントの連携です。
examples/deliverable-templates/closing.md に従ってください。

[クライアント]: [社名]
[プロジェクト期間]: [YYYY-MM-DD 〜 YYYY-MM-DD]
[当初目的]: [キックオフ時の目的]

以下を作成:

1. 30秒サマリ
   - 結果 → 学び → 次の一歩
   - 最大の成果（1つだけ）
   - 次の推奨アクション（1つだけ）

2. プロジェクト総括
   - 達成したこと
   - 達成しなかったこと（正直に）

3. KPI 最終結果（指標分岐 / 小野寺式）
   - Primary / Secondary 別
   - 全数値に出典 + 計測期間 + 計測ツール

4. PL インパクト（佐藤裕介流）
   - 投資総額 / 期待粗利インパクト / ブレイクイーブン / ROI

5. 学びとインサイト
   - 期待通り / 期待と違った / 新しい発見 / 反証された仮説

6. 残課題と次フェーズ提案
   - アセット帰属診断（佐藤裕介式）
   - 次フェーズの ROI 試算

7. ハンドオーバー
   - クライアント側で運用継続するもの
   - 自社で継続支援可能なもの

8. 感謝（シュルツ流 People Business）

ルール:
- 過剰に成功を盛らない
- 達成しなかったことを率直に
- 全 KPI に出典・計測根拠

出力末尾に反証チェック結果。
```
