# 戦略・分析プロンプト集（5本）

## P01. 新規事業参入判断（佐藤裕介 3変数交点 + ティール 独占性ゲート）

```
あなたは strategy-lead エージェントです。CLAUDE.md「佐藤裕介の知見」と
.claude/agents/consulting/strategy-lead.md「必須ゲート」に従って判断してください。

[新規事業の概要]:
[簡潔に記述]

[ターゲット市場]:
[市場名 + 黎明期/拡大期/成熟期 のラベルと根拠]

以下のフレームで参入判断を行ってください:

1. 市場構造分析（ポーター 5フォース）
   - 5フォース全てをスコアリング（強/中/弱）
   - 業界の構造的収益性上限を試算

2. 3変数交点ゲート（佐藤裕介）
   - 技術変化: 何が新しく可能になったか
   - ユーザー変化: 行動・期待・人口構成の変化
   - 市場構造変化: 規制・競合再編・コスト構造
   - 3つが重なる地点を提案しているか（1つでも欠ければ「良いプロダクトだが負ける」）

3. ティール独占性ゲート
   - 10x優位はあるか（漸進改善なら却下）
   - 独占の4要素のうち最低2つを満たすか
   - Last Mover ポジションになれるか
   - Secret thesis を1文で書けるか

4. PL試算（3シナリオ）
   - 保守/現実/楽観で投資総額・粗利・ブレイクイーブン

5. 競合分析（クリステンセン Disruption 判定）
   - ローエンド or 新市場から始まっているか
   - 既存リーダーが「儲からない」と無視する領域か

出力形式: 各セクションを表またはチェックリスト形式で。
出典 URL + 発行日が紐づかない数値は使用禁止（.claude/memory/data-sources.md 参照）。

最後に GO / 慎重に GO / NO GO の判定 + 確信度（高/中/低）を反証チェック結果と共に提示。
```

## P02. 競合分析（ポーター 5フォース + Activity Map）

```
あなたは competitive-analyst エージェントです。
.claude/agents/consulting/competitive-analyst.md「必須ゲート」に従って分析してください。

[分析対象]: [競合企業名 / URL]
[自社の領域]: [業界・領域を限定]

以下を実施:

1. ポーター式 業界構造分析
   - 5フォースをスコアリング
   - 業界の利益が誰に流れているか（買い手/売り手/競合/代替品）を金額で
   - stuck in the middle 検出
   - 競合の Activity Map 作成

2. ポジショニング分析
   - コアメッセージ・ターゲット・売っている感情
   - 価格設定・利益率
   - コンテンツ戦略・反応の良いフォーマット
   - 弱点（脆弱性・顧客不満・やっていないこと）

3. クリステンセン Disruption 判定
   - ローエンド/新市場起点か
   - 性能向上で主流市場に侵食するベクトル
   - 既存リーダーが収益構造で対応できない領域

4. 自社差別化（佐藤裕介式 領域限定）
   - 商品名ではなく「解釈した強み」で記述
   - 自社が勝つ軸 / 全社で差分が出る軸のみ採用
   - 自社が負ける軸は載せない

出典は公式 / IR / 主要紙のみ。クライアントヒアリング由来の非公開データは使用禁止。
出力末尾に反証チェック結果。
```

## P04. 戦略提案書 アウトライン（バーバラ・ミント SCQA）

```
あなたは proposal-writer エージェントです。
.claude/agents/consulting/proposal-writer.md「必須セクション」に従って構成してください。

[クライアント]: [社名・業界]
[提案の核心]: [1-2文]
[読了時間目標]: 30秒で骨子・10分で全体把握

以下の SCQA + Pyramid 構造で提案書アウトラインを作成:

1. SCQA ブロック（冒頭、必須）
   - Situation: 既知の前提（クライアントの現状）
   - Complication: 変化・問題（なぜ動かないといけないか）
   - Question: 問い
   - Answer: 結論（Governing Thought）

2. Governing Thought（30秒ピッチ）
   - 1文で記述
   - "If you can't put your message in 30 seconds, you don't have a message yet."

3. 支持論点（3-5項目、MECE）
   - 各論点の主張
   - 各論点を支えるデータ・事例
   - 出典 URL + 発行日

4. プロダクトバリューの外側分析（佐藤裕介）
   - 機能ではなく市場選定の妥当性
   - ポジションの持続性

5. PL試算（P03 のフォーマット）

6. 次のアクション（クライアントが行動できる粒度）

ルール:
- 1ページ 1メッセージ
- 1段落 3文以内
- フィラー / ヘッジ表現禁止
- 上から目線禁止（〜できます → 〜いただけます）
- 最上級の無根拠使用禁止（業界最大級・最も等）
- 全数値に出典 URL + 発行日

出力末尾に反証チェック結果（MECE 自己診断含む）。
```

## P05. 第一原理分解（クイック版）

```
あなたは strategy-lead エージェントです。
`.claude/skills/first-principles-breakdown.md` の5ステップを適用してください。

[分解対象]: [問題・主張・常識]

以下を実施:

1. 前提の列挙
   - 何が前提として置かれているか全て書き出す
   - その前提は誰がどこで言ったか（出典）

2. 前提の検証
   - 各前提は本当に正しいか
   - 反証可能性はあるか
   - 確証バイアスはないか

3. 構成要素への分解
   - 物理的・論理的に最小単位まで分解
   - 各要素の独立性を確認

4. 真理からの再構築
   - 検証された要素のみで論を組み直す
   - 飛躍・暗黙の前提を排除

5. 結論
   - 元の主張・常識との差分
   - コンセンサス疑念チェック（佐藤裕介）: 「みんなが良いという施策」を採用前に根拠を3レベル分解

出力形式: 各ステップを段階的に。
出力末尾に反証チェック結果。
```
