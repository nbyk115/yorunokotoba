# Frontend Design — プロダクショングレードUI設計スキル

## 概要
「AIスロップ」な汎用デザインを排除し、文脈固有の大胆なビジョンと
システマティックなUX/アクセシビリティルールを組み合わせた
本番品質フロントエンドの設計・実装スキル。
出典: Lyvyos42/frontend-design-skill

---

## §1 Phase 1 — デザイン思考

### 実装前に必ず明確にする4点

```
1. Purpose（目的）: このインターフェースは何の問題を解くか？誰が使うか？
2. Tone（トーン）: 美的方向性を1つ選び切る（下記リストから）
3. Constraints（制約）: フレームワーク・パフォーマンス・アクセシビリティ要件
4. Differentiation（差別化）: 「何を1つ、見た人が覚えているか」
```

### トーン選択肢（必ず1つに絞る）
| トーン | 特徴 |
|---|---|
| 徹底的ミニマル | 余白・タイポグラフィのみで勝負 |
| マキシマリスト・カオス | 複雑・高密度・刺激的 |
| レトロフューチャー | 70-80年代SFの再解釈 |
| オーガニック/ナチュラル | 自然素材・流体・有機的曲線 |
| ラグジュアリー/洗練 | 余白・繊細・プレミアム感 |
| ブルータリスト/ロウ | 生のHTML美学・直接性 |
| アールデコ/ジオメトリック | 幾何学的・対称・装飾的 |
| ターミナル/データ密集 | モノスペース・グリッド・情報密度最大 |
| エディトリアル/マガジン | タイポグラフィ主役・非対称レイアウト |

**CRITICAL**: トーンを決めて、意図を持って実行する。強いミニマリズムと洗練されたマキシマリズム、どちらも正解—鍵は「意図性」。

## §2 Phase 2 — 美的ガイドライン

### タイポグラフィ
- **禁止フォント**: Arial, Roboto, Inter, System fonts（汎用すぎる）
- **推奨**: 文脈固有の選択。表示用フォント + 本文フォントのペアリング
- **データ密集UI**: JetBrains Mono / Berkeley Mono / IBM Plex Mono（数字の区別が明確なもの）
- **重要**: ページをまたいで同じフォントに収束しない。毎回異なる選択をする

### カラー & テーマ
- CSS変数/セマンティックトークンで一貫性を確保
- ダークモード: 彩度をやや下げる（純黒背景は避ける）
- 均等配色より「支配色 + シャープアクセント」の方が強い
- ライト/ダーク両バリアントを最初から設計する

### モーション
- Reactの場合: Motion Library推奨
- ページロード: スタガードリビール1つを丁寧に（散漫なマイクロインタラクションより効果的）
- `prefers-reduced-motion` を必ずサポートする
- CSSのみで解決できる場合はCSSのみ使う

### 空間構成
- 非対称・オーバーラップ・対角フロー・グリッド外れ要素
- 寛大なネガティブスペースまたは制御された高密度
- 予測可能なレイアウトを避ける

### 背景・視覚的詳細
- グラデーションメッシュ・ノイズテクスチャ・幾何学パターン・透明レイヤー
- ダークモードUI: box-shadowではなくlightnessで高度を表現
- カスタムカーソル・グレインオーバーレイ・装飾ボーダーを積極検討

## §3 Phase 3 — 設計ルール（優先順位順）

### Priority 1: アクセシビリティ（CRITICAL）

**コントラスト:**
```
通常テキスト: 4.5:1以上（WCAG AA）
大テキスト: 3:1以上
エラー/成功カラー: 4.5:1以上
データの線/バー: 背景に対して3:1以上
```

**フォーカス & ナビゲーション:**
```
- 全インタラクティブ要素に視覚的フォーカスリング（2-4px）
- タブ順序は視覚的順序と一致
- エラー後: 最初のエラーフィールドにオートフォーカス
```

**ラベルとテキスト:**
```
- 全フォーム入力に <label for=""> 必須
- アイコンのみのボタン: aria-label 必須
- 見出し階層: h1-h6 を飛ばさない
```

**スクリーンリーダー:**
```
- フォームエラー: aria-live region または role="alert"
- トースト: aria-live="polite"、フォーカス奪取禁止
- チャート: テキストサマリーまたは aria-label（インサイトを説明）
- スキップリンク: キーボードユーザー用に必須
```

### Priority 2: タッチ & インタラクション（CRITICAL）

```
タッチターゲット最小: 44×44pt（Apple HIG）/ 48×48dp（Material）
ターゲット間隔: 最小8px
100ms以内の視覚フィードバック（リップル・ハイライト・scale 0.95-1.05）
全クリッカブル要素: cursor: pointer
非同期操作中: ボタン無効化 + スピナー表示
ホバーのみに依存しない（タップでも主要操作が可能）
```

### Priority 3: パフォーマンス（HIGH）

```
画像: WebP/AVIF + srcset + lazy load（below-fold）
width/height または aspect-ratio を宣言（CLS < 0.1）
フォント: font-display: swap、クリティカルフォントのpreload
重要なCSSはインラインまたは早期ロード
```

### Priority 4: フォームデザイン（HIGH）

```
- リアルタイムバリデーション（ブラー時、送信前ではない）
- エラーメッセージはフィールド直下に表示
- 関連フィールドをグループ化
- 成功状態を明示的に示す
- オートコンプリート属性を設定（name/email等）
```

### Priority 5: ナビゲーション（MEDIUM）

```
- 現在位置のアクティブ状態を常に示す
- パンくずリスト（3階層以上）
- モバイル: ハンバーガーは最終手段
- 60% rule: スクロールなしで最重要コンテンツが見える
```

---

## §4 実装禁止事項

**絶対禁止（"AIスロップ"の特徴）:**
- 汎用フォントファミリー（Inter, Roboto, Arial, Space Grotesk等）
- 定番カラースキーム（白背景に紫グラデーション等）
- 予測可能なレイアウト
- 文脈なしの汎用デザイン
- 世代をまたいで同じフォント・色に収束すること
