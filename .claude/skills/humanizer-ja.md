# Humanizer-JA — 日本語AI臭除去スキル

## 概要
AIが生成した日本語テキストの「AI臭」を検出・除去し、人間が書いたような自然な文章に変換する。
6カテゴリ・30パターンの検出ルールと書き換えガイドを提供する。
出典: kgraph57/humanizer-ja

---

## §1 AI臭の6カテゴリと検出パターン

### カテゴリ1: 視覚記号の過剰使用
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| Markdown残留 | 「**重要**です」「## 概要」 | 記号を外し文体で強調 |
| 全角ダッシュ乱用 | 「コスト削減——つまり」 | 読点か接続詞に置換 |
| カッコの多用 | 「施策（3つ）を（検討）した」 | 不要なカッコを除去 |
| スラッシュ列挙 | 「戦略/施策/方針」 | 「戦略・施策・方針」または文章化 |

### カテゴリ2: リズムの単調さ
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| 語尾の反復 | 「〜です。〜ます。〜です。」が3文連続 | 語尾を意図的にずらす |
| 接続詞の過剰 | 「そして、また、さらに、また」 | 接続詞を削るか文を結合 |
| 均一なトーン | 全文が同じ熱量・同じ距離感 | 強弱・メリハリをつける |

### カテゴリ3: マニュアル調の展開
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| 長い前置き | 「本レポートでは、以下の観点から…」 | 冒頭を結論から始める |
| 構造の冗長性 | 「第1に…第2に…第3に…以上が…」 | 必要な順序のみ残す |
| 定型の締め | 「ご不明な点はお問い合わせください」 | 文脈に合う終わり方に変換 |

### カテゴリ4: 曖昧な立場・逃げ表現
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| 過剰な限定詞 | 「一般的に」「場合によっては」が多用 | 立場を明確にするか削除 |
| 偽中立 | 「AもBも重要です」（判断を避ける） | どちらかに踏み込む |
| ケースバイケース逃げ | 「状況によって異なります」で終わる | 具体的な判断基準を示す |

### カテゴリ5: 抽象語彙の濫用
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| 概念の膨張 | 「シームレスな体験」「ホリスティックな」 | 具体的な動詞・名詞に置換 |
| 根拠なし評価語 | 「革新的」「最先端」「画期的」 | 数値・事実に置換か削除 |
| AI定番フレーズ | 「〜を実現します」「〜に向けて」 | 直接的な表現に変換 |

### カテゴリ6: 不自然な比喩
| パターン | AI臭の例 | 修正方針 |
|---|---|---|
| ツール・機械比喩 | 「チームを駆動する」「歯車として」 | 人間的な表現に変換 |
| 身体比喩 | 「組織の骨格」「血液となるデータ」 | 直接表現に変換 |
| 調理比喩 | 「戦略を煮詰める」「アイデアを調理」 | ビジネス文脈の語彙に変換 |

## §2 書き換えプロセス

```
Step 1: スキャン
  → テキストを全文読み、30パターンのAI臭をマークする
  → 深刻度を「高（意味が薄れる）」「中（違和感がある）」「低（気になる程度）」で分類

Step 2: 優先除去（高・中から着手）
  → カテゴリ4・5（曖昧表現・抽象語）は必ず除去
  → カテゴリ1（記号）は文体に合わせて判断

Step 3: リズム調整
  → 語尾パターンを確認: 同じ語尾が3文以上続いたら変える
  → 文の長さに変化をつける（短文・長文を交互に）

Step 4: 立場の明確化
  → 「場合によっては」→ 具体的条件に
  → 「重要です」→ なぜ重要か1文付加

Step 5: 最終確認
  → 事実・数値・意味は保持されているか
  → ブランドトーンと整合しているか
```

## §3 出力ルール（禁止・必須）

### 禁止
- Markdownフォーマット（見出し・太字・箇条書き記号）を出力テキストに含めない
- 「〜を実現します」「〜に向けて取り組みます」等のAI定番フレーズ
- 意味のない安全クッション語（「なお」「ちなみに」「また」の連発）
- 弱否定（「〜でないとは言えない」「〜ではないかもしれません」）

### 必須
- 事実・数値・固有名詞はそのまま保持
- 元の意図・論旨を変えない（言い方だけを変える）
- コンテキストに応じたトーン（ビジネス正式・カジュアル・報告書）を維持
