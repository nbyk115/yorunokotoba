# Claude Health — Claude Code設定診断スキル

## 概要
Claude Codeのプロジェクト設定を6層フレームワークで診断し、Critical/Structural/Incrementalの3段階で問題を報告する。
設定の健全性を維持し、品質劣化・セキュリティリスクを早期検出する。
出典: tw93/claude-health

---

## §1 6層診断フレームワーク

```
Layer 1: CLAUDE.md          → 指示の品質・明確性・矛盾チェック
Layer 2: Rules              → ルール配置の適切性・重複・矛盾
Layer 3: Skills             → スキル参照の正確性・インジェクションリスク
Layer 4: Hooks              → フック設定のカバレッジ・漏れ
Layer 5: Subagents          → サブエージェント定義の品質・権限設定
Layer 6: Verifiers          → 品質ゲートの存在・有効性
```

---

## §2 プロジェクトティア判定

```
Simple（単純）:
  → ファイル数少・単一貢献者・CI不在
  → ローカル分析のみ実施

Standard（標準）:
  → 中規模・複数貢献者・基本CI
  → 並列サブエージェントで詳細分析

Complex（複雑）:
  → 大規模・チーム・本格CI/CD
  → 全層フル診断 + セキュリティスキャン強化
```

---

## §3 診断項目と重大度

### Critical（今すぐ修正）
| 項目 | 検出内容 |
|---|---|
| ルール違反 | CLAUDE.mdで定義したルールが実際に守られていない |
| セキュリティ問題 | スキルへのプロンプトインジェクション・データ漏洩リスク |
| 矛盾する指示 | 複数のCLAUDE.mdが相互矛盾する内容を含む |
| 危険な権限 | allowedToolsに過剰な権限が付与されている |

### Structural（近々修正）
| 項目 | 検出内容 |
|---|---|
| フック漏れ | 重要なイベント（PostToolUse等）がカバーされていない |
| 品質ギャップ | コードレビュー・テスト自動化が欠如 |
| スキル設定ミス | 参照スキルファイルが存在しないまたは不正形式 |
| MCPサーバー問題 | 未使用・冗長・設定不備のMCP |

### Incremental（随時改善）
| 項目 | 検出内容 |
|---|---|
| コンテキスト肥大 | CLAUDE.mdが肥大化し読み込みコストが増大 |
| スキル最適化 | 重複スキルの統合機会 |
| プロンプトキャッシュ | キャッシュ設定の最適化余地 |
| 3層防御 | 多層防御の欠如 |

## §4 診断実行プロセス

```
Step 1: データ収集（1バッシュブロックで実行）
  - CLAUDE.md内容（全層）
  - .claude/settings.json
  - hooks設定
  - skills一覧とファイルサイズ
  - MCPサーバー一覧
  - 直近の会話履歴サンプル

Step 2: セキュリティ処理
  - APIキー・トークン・シークレットをサブエージェントに渡す前に秘匿化
  - グレーディング: [REDACTED_KEY_xxx]形式で置換

Step 3: ティア判定 → 診断深度を決定

Step 4: 層別診断（ティアに応じて並列/直列）

Step 5: レポート生成
```

### レポートフォーマット
```
## Claude Code Health Report

**プロジェクト**: [名前]  **ティア**: [Simple/Standard/Complex]
**診断日時**: YYYY-MM-DD HH:MM

### 🔴 Critical（今すぐ修正）
- [問題1]: [具体的な内容と修正方針]

### 🟡 Structural（近々修正）
- [問題1]: [具体的な内容と修正方針]

### 🟢 Incremental（随時改善）
- [問題1]: [具体的な内容と修正方針]

---
修正しますか？層ごとに対応できます:
[ ] global CLAUDE.md  [ ] local CLAUDE.md  [ ] hooks  [ ] skills
```

## §5 データ駆動原則

- **ツール欠如を設定ミスとしない**: jqやpython3が使えない環境では対応する診断項目をスキップ
- **単純プロジェクトに複雑な指摘をしない**: ティアに応じた診断深度
- **自動修正しない**: 問題を提示し、ユーザーの判断を仰ぐ
- **出力言語**: CLAUDE.mdのCommunicationルール → 最近のメッセージ言語 → 日本語デフォルト
