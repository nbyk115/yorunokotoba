# Global Expansion Playbook — グローバル展開プレイブック

## 概要
Global部門の全エージェントが参照する海外展開の標準手法・評価フレームワーク・品質基準。
市場評価から参入実行・ローカライズ・現地オペレーションまでの一貫した判断基準を提供する。

---

## §1 市場評価フレームワーク — 「どこから攻めるか」を構造で決める

### 原則
「英語圏だから米国」「人口が多いからインド」は禁止。市場構造・自社アセット適合度・PLインパクトの3軸で判断する。

### 市場評価スコアカード
| 評価軸 | 重み | スコア基準（1-5） | 判定 |
|---|---|---|---|
| 市場規模（TAM→SAM→SOM） | 25% | 1:10億未満 / 3:50-200億 / 5:500億超 | |
| 成長率（CAGR） | 15% | 1:0%以下 / 3:5-15% / 5:20%超 | |
| 競合強度 | 15% | 1:独占 / 3:寡占 / 5:分散（参入余地大） | |
| 参入障壁（法規制・言語・文化） | 15% | 1:極めて高い / 3:中程度 / 5:低い | |
| 自社アセット適合度 | 20% | 1:全く活かせない / 3:部分的 / 5:高い適合 | |
| ペイバック期間 | 10% | 1:36ヶ月超 / 3:18-24ヶ月 / 5:12ヶ月以内 | |

**判定基準:**
- **4.0以上**: Go（優先参入）
- **3.0-3.9**: 条件付きGo（特定リスクを解消後に参入）
- **3.0未満**: No-Go（現時点では見送り）

### 市場クラスタリング
地域を「攻めやすさ × 市場規模」でクラスタリングし、参入順序を決める。

| クラスタ | 特徴 | 参入優先度 | 例 |
|---|---|---|---|
| A: 高適合×大市場 | 自社の強みが活き、規模も大きい | 最優先 | — |
| B: 高適合×小市場 | 低リスクで検証可能 | テスト市場として優先 | — |
| C: 低適合×大市場 | 魅力的だが参入コスト高 | 中期計画 | — |
| D: 低適合×小市場 | 参入メリット薄い | 見送り | — |

> **佐藤裕介の知見を適用**: 「参入できる力があるのに挑戦しなかったことが最大の失敗」（ダイヤモンド2020）。参入判断はPLシミュレーションと撤退基準を先に設定してから行う。

## §2 参入モデル設計 — 「どう入るか」を構造で決める

### 参入モデル比較マトリクス
| モデル | 初期投資 | 現地コントロール | スピード | リスク | 適合条件 |
|---|---|---|---|---|---|
| PLG（Product-Led） | 低 | 高 | 速 | 低 | セルフサーブ可能なSaaS |
| 販売代理店 | 低〜中 | 低 | 速 | 中（品質管理） | 現地ネットワーク不足時 |
| リモートチーム | 低〜中 | 中 | 速 | 中 | 技術系サービス |
| JV（合弁） | 中 | 中 | 中 | 中（パートナー依存） | 規制が厳しい市場 |
| 現地法人設立 | 高 | 高 | 遅 | 高 | 長期コミット市場 |
| M&A | 非常に高 | 高 | 速 | 高（PMI） | 即座に市場シェアが必要 |

### 参入モデル選択フロー
```
プロダクトはセルフサーブで完結する？
├─ Yes → PLGモデル（Freemium/Free Trial）
│         └─ 現地言語対応は必要？ → business-translator に依頼
└─ No → 現地での対面接点が必要？
    ├─ No → リモートチーム構築
    └─ Yes → 現地パートナー候補がいる？
        ├─ Yes → 販売代理店 or JV
        └─ No → 現地法人設立 or M&A検討
```

## §3 ローカライズ品質基準 — 翻訳を超える5層適合

### ローカライズの5層モデル
| 層 | 対象 | 品質基準 | 担当 |
|---|---|---|---|
| L1: 言語 | テキスト・UI・ドキュメント | 自然な現地語。直訳不可 | business-translator |
| L2: 文化 | 色・アイコン・メタファー・画像 | 文化的タブー回避。現地の感覚に適合 | ux-designer + brand-guardian |
| L3: 商習慣 | 決済・契約・営業プロセス | 現地の商慣行に準拠 | global-business |
| L4: 法規制 | プライバシー・広告規制・ライセンス | 各国法令に完全準拠 | legal-compliance-checker |
| L5: 価格 | 通貨・価格帯・課金モデル | 購買力平価+競合価格+心理的価格帯 | gtm-consultant + kpi-analytics |

### ローカライズ品質チェック（コピペ用）
```
□ L1 言語: ネイティブが読んで違和感がないか
□ L2 文化: 色・画像・表現に文化的問題はないか
□ L3 商習慣: 決済手段は現地で一般的か（例: 東南アジアのCOD）
□ L4 法規制: 個人情報保護法（GDPR/CCPA/PIPL等）に準拠しているか
□ L5 価格: 現地購買力に合った価格帯か
□ 全層統合: プロダクト体験として一貫しているか
```

### 地域別ローカライズ重点項目
| 地域 | L1重点 | L2重点 | L3重点 | L4重点 | L5重点 |
|---|---|---|---|---|---|
| 北米 | 米語/加英語の差 | 多様性表現 | クレカ中心 | CCPA/SOX | ドル建て |
| EU | 24公用語 | 国ごとの文化差 | VAT対応 | GDPR（最厳格） | €+購買力差 |
| 東南アジア | 多言語（英+現地語） | 宗教的配慮 | COD/eWallet | 国ごとに異なる | PPP調整必須 |
| 中国 | 簡体字+独自表現 | 赤=吉、白=喪 | Alipay/WeChat Pay | PIPL/サイバーセキュリティ法 | 元建て |
| 中東 | アラビア語RTL対応 | イスラム文化配慮 | 金曜休業 | シャリア準拠 | 国ごとに通貨異なる |
| 中南米 | ブラジルPT≠ポルトガルPT | カラフルさ好む傾向 | Boleto/PIX(BR) | LGPD(BR) | 通貨変動大 |

## §4 グローバルPL設計 — 海外事業の損益を正確に読む

### 原則
国内PLフレームをそのまま適用しない。海外特有のコスト構造と収益構造を正確に反映する。

> **詳細PLフレーム → `revenue-growth-framework.md` §1, §7**

### 海外PL追加コスト項目
| カテゴリ | 項目 | 概算比率 | 備考 |
|---|---|---|---|
| 初期投資 | 法人設立・登記 | ¥100-500万 | 国により大幅に異なる |
| 初期投資 | ローカライズ（L1-L5） | ¥200-1,000万 | 対象市場数×層数で変動 |
| 初期投資 | 現地法務・会計顧問 | ¥100-300万/年 | 必須。削減不可 |
| 変動費 | 為替ヘッジコスト | 取引額の1-3% | 通貨ペアにより変動 |
| 変動費 | 国際決済手数料 | 売上の2-5% | Komoju/現地決済の選択 |
| 変動費 | 移転価格税制対応 | 粗利の5-15%がリスク | 事前確認（APA）推奨 |
| 固定費 | 現地人材（採用・給与） | 日本比0.3-3x | 地域により大幅差 |
| 固定費 | オフィス・インフラ | 日本比0.2-2x | リモートなら最小化可 |

## §5 クロスボーダーリスク管理 — 見落としがちなリスクを構造化

### リスクカテゴリと対策
| リスク | 影響度 | 検知方法 | 対策 | 担当 |
|---|---|---|---|---|
| 為替変動 | 高 | 月次モニタリング | 先物/オプション/自然ヘッジ | kpi-analytics |
| 規制変更 | 高 | global-journalist監視 | 事前シナリオ策定 | legal-compliance-checker |
| 地政学リスク | 中〜高 | global-journalist監視 | 分散投資・撤退基準明確化 | strategy-lead |
| 文化摩擦 | 中 | 現地NPS/フィードバック | 異文化研修・現地人材登用 | global-business |
| データ越境規制 | 高 | 法改正モニタリング | データローカライゼーション | infra-devops |
| 知財侵害 | 中 | 商標・特許ウォッチ | 現地商標登録・特許出願 | legal-compliance-checker |
| パートナーリスク | 中 | 四半期レビュー | 契約条件・KPI・代替候補 | global-business |

### 撤退基準（事前に定義しておく）
```
以下のいずれかに該当した場合、撤退を検討:
□ 18ヶ月連続で月次売上目標の50%未達
□ LTV/CACが12ヶ月連続で1.5倍未満
□ 規制変更により事業モデルが成立しなくなった
□ 地政学リスクにより安全な事業継続が困難
□ 累積赤字が事前設定の上限（¥___万）を超過
```

## §6 グローバルGTMプレイブック — 市場別の攻め方

### フェーズ設計
| フェーズ | 期間 | 目的 | 主要KPI | 投資水準 |
|---|---|---|---|---|
| Phase 0: リサーチ | 1-2ヶ月 | 市場理解・仮説構築 | 一次情報収集件数 | 最小（リサーチ費のみ） |
| Phase 1: テスト | 3-6ヶ月 | PMF検証・初期ユーザー獲得 | アクティベーション率 | 低（¥500万以下） |
| Phase 2: 拡大 | 6-18ヶ月 | グロース・チャネル確立 | MRR成長率・LTV/CAC | 中（PL計画に基づく） |
| Phase 3: 定着 | 18ヶ月〜 | 収益化・オペレーション最適化 | 営業利益率・NPS | 適正（黒字化ライン） |

### Phase 0: リサーチチェックリスト
```

□ 競合マップ（現地+グローバルプレイヤー）
□ 規制環境の一次調査（法令原文の確認）
□ 顧客インタビュー（最低5社/5名）
□ 価格感度テスト（簡易サーベイ or LPテスト）
□ 参入モデルの候補絞り込み
□ 3年PLシミュレーション（楽観/標準/悲観）
□ Go/No-Go判断
```

## §7 情報収集の品質基準 — 一次情報ファースト

### ソース優先度（全Global部門共通）
| 優先度 | ソースタイプ | 例 | 信頼度 |
|---|---|---|---|
| 1 | 政府統計・法令原文 | 各国統計局、官報、法律データベース | A |
| 2 | 企業IR・公式発表 | 年次報告書、プレスリリース | A |
| 3 | 主要メディア・学術論文 | Reuters, FT, Nature, 専門ジャーナル | B |
| 4 | 業界レポート・アナリスト | Gartner, McKinsey, CB Insights | B |
| 5 | 業界メディア・ブログ | TechCrunch, 現地業界サイト | C |
| 6 | SNS・未検証情報 | X/Twitter, Reddit, 掲示板 | D |

### クロスリファレンス原則
- 同一事実は**最低2ソース**で確認
- 信頼度Dのソースのみに基づく結論は禁止
- 情報のバイアス（政府系/企業系/独立系）を明示
- 収集日時を記録（海外情報は鮮度が命）

> **小野寺メソッド適用**: 報道されている数字の定義・前提を確認。海外統計は国ごとに集計基準が異なる。

### ハンドオフ品質基準（双方向チェック）

#### ✅ 送り出し側（From）— Global部門から他部門へ渡す前に確認
```
□ 対象市場・参入フェーズを明示したか
□ 収集済み一次情報にソースグレード（A〜D）を付けたか
□ 現地の制約条件（法規制/文化/言語）を具体的に記載したか
□ PLシミュレーションの前提条件（為替・価格・チャネルコスト）を数値で渡したか
□ 撤退基準（事前合意した閾値）を定義したか
□ 参照スキルを「いつ・どのセクションを」明示して渡したか
□ 反証モード（Deep/Standard/Light深度）を適用済みか
□ 「ソースなし・未検証情報のみ」は差し戻し対象と明示したか
```

#### ✅ 受け取り側（To）— Global部門へのタスク引き継ぎ前に確認
```
□ 市場スコアリング基準（定量指標）が提示されているか
□ 調査対象の地域・言語・競合の範囲が明確か
□ 期待アウトプット形式（レポート/スコアカード/PL表）が指定されているか
□ ローカライズ5層（L1〜L5）のどの層まで対応が必要か明確か
□ 前工程の反証チェック結果を引き継いでいるか
```

#### 🚫 受け取り拒否基準（Global部門）
以下のいずれかに該当する場合は即座に差し戻す:
1. **数値根拠なしの参入依頼**: 「東南アジアに展開したい」のみ
2. **未検証・ソースなしの海外情報**: グレードD相当の情報のみで判断を求める
3. **反証未実施のアウトプット**: 前工程でトリプルチェックが省略されている

## §9 グローバルクレデンシャル プレゼンテーション設計

> **海外ピッチ・国際パートナー向けのプレゼンは、スティーブ・ジョブズのプレゼン哲学を基軸にする。**
> 詳細な制作ルールは `ppt-presentation.md` §4 を参照。

### グローバルピッチの鉄則

```
【Global Credential Deck 設計原則】
1. ビジュアル主体 — テキストは1スライド3行以内。代わりに写真・数字・図で語る
2. 数字は孤立させる — "42カ国展開" を1スライド全面に。箇条書きの中に埋めない
3. 英日バイリンガル — 英語メイン + 日本語サブタイトル（フォント: 英字 Helvetica/Inter、日本語 メイリオ）
4. 対比で驚かせる — 「日本のスタンダード → グローバルでの価値」を大きなジャンプで見せる
5. ストーリーテリング — Why（なぜ存在するか）→ What（何ができるか）→ Proof（実績・数字）
6. 余白を活かす — 詰め込みは信頼を失う。1スライドの情報量は視聴者の5秒で理解できる量まで
```

## §11 グローバルプライシング戦略

### 購買力平価（PPP）調整係数（参考値・必ず現地調査で検証）
| 市場 | PPP係数（日本=1.0） | WTP調整目安 |
|---|---|---|
| 米国 | 0.85 | 日本価格 ×1.2〜1.5 |
| 欧州（独・仏） | 0.90 | 日本価格 ×1.1〜1.3 |
| 東南アジア（SG） | 0.95 | 日本価格 ×1.0〜1.2 |
| 東南アジア（ID・VN） | 2.0〜2.5 | 日本価格 ×0.3〜0.5 |
| 中国 | 1.2 | 日本価格 ×0.6〜0.9 |
| インド | 3.0〜4.0 | 日本価格 ×0.2〜0.4 |

> ⚠️ 上記は参考値。実際のWTPは必ず現地PSM調査（`market-researcher`）で検証する。

### グローバルプライシング設計原則
1. **アンカリング**: グローバル標準価格を先に設定し、そこから現地購買力に合わせて調整
2. **為替リスク管理**: 長期契約はUSD/EUR建てを検討（円建てだと為替リスクが顧客に転嫁）
3. **グレー市場対策**: 市場間価格差が30%超になると並行輸入・転売リスクが発生
4. **心理的価格帯**: 現地での「プレミアム」「標準」「アクセシブル」の価格感覚を調査
5. **決済手段の現地化**: Komojuで日本・東南アジアの主要決済（コンビニ/LINE Pay/QR）に対応

## §12 ローカライズ品質基準チェックリスト（層別合格基準）

> ローカライズの5層（L1〜L5）それぞれに「合格/不合格」を判定する具体的基準を定義する。

### L3: 商習慣対応（Business Practice）
- [ ] 現地決済方法が3種類以上対応（例: 日本=クレカ+コンビニ+PayPay）
- [ ] 契約書・利用規約が現地語かつ現地法準拠（法務チェック済み）

### L4: 法規制対応（Legal/Regulatory）
- [ ] `legal-compliance-checker`による現地法規制チェック完了
- [ ] 個人情報保護法（GDPR/PIPL/APPI等）への準拠確認
- [ ] 必要ライセンス・登録の取得確認（金融/医療/教育等の規制業種）

- [ ] 競合3社以上の現地価格を調査・比較済み

- [ ] 推奨価格でのPLシミュレーション（ブレイクイーブン確認済み）

## §13 為替感度分析テンプレート

> 海外展開PLは「為替が動くと全て狂う」。3通貨ポジションを管理し、感度分析を事前に実施する。

### 為替ポジション管理の3軸

| ポジション | 説明 | リスク管理 |
|---|---|---|
| 売上（現地通貨） | 現地での販売収入（例: USD, SGD） | 円高になると円換算収益が減少 |
| 直接コスト（現地通貨） | 現地スタッフ・オフィス・マーケ費用 | 現地通貨のまま = 売上とヘッジ関係 |
| 本社連結（円） | 日本本社への送金・親会社への計上 | 為替変動を最終的に受ける |

**自然ヘッジの設計**: 売上と費用を同一通貨にすることで為替リスクを圧縮する。

### 為替感度分析テンプレート

```
対象市場: [国名]  為替ペア: JPY/[通貨]  基準レート: [X円/1通貨]

【売上（現地通貨建て）】
現地売上: [X通貨/月]
円換算（基準）:     X円
円換算（+10%円安）: +X円（+XX%）
円換算（-10%円高）: -X円（-XX%）
円換算（-20%円高）: -X円（-XX%）

【PLへの最終インパクト（売上−現地コスト）】
基準シナリオ:  粗利 X円
+20%円安シナリオ: 粗利 X円（+XX%）
-20%円高シナリオ: 粗利 X円（-XX%）

【ブレイクイーブン為替レート】
粗利ゼロになる為替レート: [X円/1通貨]
（現在レートからXX%の円高で損益分岐点）
```

### ヘッジ手法の選択基準

| 取引規模 | 推奨ヘッジ方法 | コスト目安 |
|---|---|---|
| 月1,000万円未満 | ナチュラルヘッジ（現地費用を現地通貨で調達） | 0% |
| 月1,000万〜1億円 | 為替予約（3〜6ヶ月先物） | 0.5〜1.5%/年 |
| 月1億円以上 | オプション + 予約の組み合わせ | 1〜3%/年 |

**注意**: 為替ヘッジコストをPLシミュレーションに必ず含めること。含めない場合はERR-01として差し戻す。

## §14 ASI時代のグローバル展開戦略

> AGI到達後、「情報収集・翻訳・市場調査」はほぼ自動化される。人間のグローバル担当者が担う価値が根本的に変わる。

### グローバル職能のASI耐性マトリクス

| グローバル機能 | 代替可能性(0-1) | 残存理由 |
|---|---|---|
| 翻訳・ローカライズ（基本） | 0.90 | AIが瞬時に高品質翻訳 |
| 海外市場データ収集 | 0.85 | AGIが全言語のデータを即時分析 |
| 競合情報収集・整理 | 0.80 | AIエージェントが自律的に調査 |
| **文化的コンテクスト解釈** | **0.30** | **現地生活体験なしでは誤訳・誤解が生じる** |
| **現地パートナーシップ構築** | **0.20** | **信頼関係は人間の顔と時間が必要** |
| **規制当局との関係** | **0.10** | **人間の責任者が不可欠** |

### ASI時代の参入判断フレームワーク

```
従来の参入判断:「市場規模 × 成長率 × 競合密度」で判断

ASI時代の追加判断軸:
1. AIエージェント普及度: 現地B2B購買にAIが介在しているか
   → Yes: API/ドキュメント/セルフサービス化が必須
   → No: 従来の人間向け営業を継続（ただし2-3年後に変化）

2. Trust Premium可能性: 現地でブランド信頼が価格プレミアムに転化するか
   → 文化的に「外資ブランド信頼」が高い市場 vs. 現地プレイヤー優位市場

3. データ主権リスク: データローカライゼーション規制がASI活用を制限するか
   → EU（GDPR）・中国（PIPL）・インド（PDPB）: 現地データ基盤が必須

4. 現地AI規制環境: AIを使った製品・サービスへの規制が参入コストを変えるか
   → EU AI Act（2024年〜）・各国の生成AI規制動向を必ずチェック
```

### グローバル展開のHuman Moat設計

```
ASI後も現地人材が担う機能:
✅ 文化的誤読の防止（「直訳では炎上する」の判断）
✅ 規制当局との関係構築（AIが代替できない人間の信頼）
✅ 現地パートナーとのディールメイキング（握手の文化）
✅ 予期しない政治・社会変化への即対応（現地感覚）

AIが代替する機能:
❌ 多言語コンテンツの量産翻訳
❌ 市場サイズ・競合の基礎データ収集
❌ 現地法規制の一次調査・サマリー
❌ 価格・為替シミュレーション
```

### ASI時代の展開優先市場スコアリング（追加軸）

```
従来スコアリングに以下を追加:

□ AIリテラシー（現地企業のAI活用度）:
  高い → AIネイティブな提案が刺さる / 低い → 人間向け営業が有効

□ データ主権規制の厳しさ:
  厳しい → 現地データ基盤への先行投資が必要 / 緩い → グローバル統合が容易

□ Trust Premiumの存在:
  外資ブランドへの信頼度 + 価格プレミアムの受容性をリサーチで確認

□ 現地AI規制リスク:
  参入後3年以内に規制強化される確率を評価し、退出コストをPLに含める
```
